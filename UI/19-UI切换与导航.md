# 《洪武立志传》UI切换与导航规范

> 版本：1.0.0  
> 继承自：《洪武立志传》整体 UI 设计规范 v1.0.0  
> 适用范围：全局界面跳转逻辑、返回栈管理、过渡动画定义

---

## 1. 设计目标与原则

### 1.1 设计目标
- **清晰可预测**：玩家在任何界面都能明确知道如何返回、如何前往其他核心模块。
- **沉浸式过渡**：界面切换动画统一为古风卷轴展开/收起风格，强化“纸墨王朝”的沉浸感。
- **低认知负担**：遵循主流游戏的导航习惯，返回按钮始终在左上角，关闭按钮在右上角。

### 1.2 核心原则
| 原则 | 说明 |
|------|------|
| **单向数据流式导航** | 界面跳转形成清晰的层级树，避免循环引用导致玩家迷失。 |
| **统一返回逻辑** | 所有全屏界面左上角「← 返回」按钮行为一致：返回上一界面，若已是最顶层则退出游戏/返回启动界面。 |
| **模态框独立于返回栈** | 模态框（设置、确认弹窗、消息事件）不压入返回栈，关闭后直接回到触发界面。 |
| **动效一致性** | 所有界面切换动画统一使用定义的过渡动画时长与缓动函数。 |

## 2. 界面层级架构

```
启动层 (Launch Layer)
├── 1.1 启动/登录界面
├── 1.2 存档/读档界面
└── 1.3 设置界面 (模态框)

核心玩法层 (Core Gameplay Layer)
├── 2.1 大明疆域图 (主地图) ◀── 游戏主枢纽
├── 2.2 城镇据点界面
├── 2.3 人物详情界面 (全屏/模态框)
└── 2.4 人物图鉴系统

子系统层 (Subsystem Layer)
├── 3.1 社交互动界面 (模态框)
├── 3.2 个人战界面
├── 3.3 合战界面
├── 3.4 政略经营界面
└── 3.5 任务与评定系统

辅助功能层 (Auxiliary Layer)
├── 4.1 小游戏界面
├── 4.2 消息/事件弹窗 (模态框)
├── 4.3 物品/道具栏
├── 4.4 商店/交易界面
├── 4.5 结局/成就界面
└── 4.6 加载/过渡界面 (临时)
```

**层级说明**：
- 玩家大部分时间在**核心玩法层**活动，其中「大明疆域图」是中枢。
- 从核心层可进入**子系统层**，子系统完成后通常返回触发它的核心层界面。
- **辅助功能层**可被多个上层界面调用。

## 3. 全局导航规则

### 3.1 通用组件行为

| 组件 | 位置 | 行为 |
|------|------|------|
| 「← 返回」按钮 | 界面左上角 | 返回上一界面（出栈）。若当前为启动界面，则退出游戏（二次确认）。 |
| 「✕ 关闭」按钮 | 界面右上角 | 关闭当前模态框/全屏界面，不保存未确认的更改。若为全屏界面且是返回栈底层，行为同返回按钮。 |
| 底部「出城」/「离开」 | 城镇/政略等界面 | 返回「大明疆域图」。 |
| 键盘 `ESC` 键 | 全局 | 行为同「返回」或「关闭」，优先关闭最上层的模态框。 |

### 3.2 返回栈管理

- **全屏界面**：每次进入一个新的全屏界面（如从地图进入城镇），当前界面**压入返回栈**。
- **模态框**：不压入返回栈，而是记录触发它的界面。关闭模态框时直接回到触发界面。
- **栈清空时机**：返回启动界面或退出游戏时清空。

## 5. 过渡动画规范

所有界面切换动画统一调用以下动画预设，确保体验一致。

| 动画名称 | 描述 | 时长 | 缓动函数 | 适用场景 |
|----------|------|------|----------|----------|
| **卷轴展开** | 新界面从中心由一条横线向上下展开，类似打开画卷 | 400ms | `cubic-bezier(0.4, 0, 0.2, 1)` | 进入核心子系统 (城镇、政略、图鉴) |
| **卷轴收起** | 当前界面由上下向中心收缩为一条横线后消失 | 350ms | `cubic-bezier(0.4, 0, 0.2, 1)` | 退出子系统返回上层 |
| **淡入淡出** | 透明度 0 ↔ 1 平滑过渡 | 250ms | `ease-in-out` | 启动界面、存档界面等轻量切换 |
| **模态框弹出** | 由小变大 + 淡入，带轻微上浮 | 250ms | `cubic-bezier(0.34, 1.56, 0.64, 1)` | 设置、消息、社交等模态框 |
| **镜头推进/拉远** | 模拟地图镜头聚焦到城镇，配合模糊与缩放 | 500ms | `ease-out` | 大地图 ↔ 城镇 |
| **水墨爆发/淡出** | 屏幕中心墨点扩散后显现新界面 (进入战斗)；战斗结束墨点收缩消失 | 450ms | `ease-in-out` | 进入/退出个人战、合战 |

**实现提示**：可使用 CSS `clip-path` 实现卷轴展开效果，使用 `transform: scale()` 配合 `opacity` 实现模态框弹出。

## 6. 特殊场景处理

### 6.1 游戏内菜单 (ESC 菜单)
- 在任何非战斗、非加载的全屏界面按下 `ESC`，弹出**游戏内菜单模态框**。
- 菜单选项：「继续游戏」「保存」「载入」「设置」「返回标题」「退出游戏」。
- 点击「保存」→ 进入存档界面 (模态框模式)。
- 点击「设置」→ 进入设置界面 (模态框模式)。
- 点击「返回标题」→ 二次确认后返回启动界面并清空返回栈。

### 6.2 战斗中的中断
- 个人战/合战中按下 `ESC`，仅弹出「是否认输？」确认框，不打开完整菜单。

### 6.3 加载与自动存档
- 任何界面切换若涉及大量资源加载，先显示**加载/过渡界面**，加载完成后再执行进入动画。
- 自动存档在以下时机触发 (无感知)：
  - 从城镇返回大地图时
  - 完成个人战/合战时
  - 完成评定任务时
  - 每日时辰更替时 (游戏内时间)

## 7. 对 AI 编程助手的导航实现提示

1. **路由/导航栈管理**：建议使用集中式的 `NavigationManager` 或状态管理库 (如 Vue Router / React Router 配合自定义栈) 维护界面栈。提供 `pushScreen(screenId, params)`、`popScreen()`、`replaceScreen(screenId)` 等 API。
2. **模态框全局控制**：使用事件总线或 `provide/inject` 控制全局模态框的显示隐藏，避免在每个组件重复引入。
3. **动画与路由钩子**：在路由切换的钩子函数中触发对应的 CSS 动画类，动画结束后再销毁旧界面。
4. **返回键拦截**：在移动端或桌面端，监听 `popstate` 或 `beforeunload` 事件，将浏览器返回键映射为游戏内返回逻辑。
5. **测试用例**：建议对每条跳转逻辑编写简单的测试，确保不会出现死循环或界面卡死。
