# 《洪武立志传》界面设计：合战界面

> 版本：1.0.0  
> 继承自：《洪武立志传》整体 UI 设计规范 v1.0.0  
> 对应模块：`BattleRenderer.js`（待新建）

---

## 1. 界面定位与功能概述

### 1.1 界面位置
合战界面是玩家指挥部队进行大规模战役的核心场景。借鉴《太阁立志传4》的**策略卡片对战**机制，将战术指令设计为可使用的“军略卡”，并结合阵型、兵种克制、士气等要素，呈现一场运筹帷幄的古代战争。

### 1.2 主要功能
- 展示敌我双方的部队配置、兵力、士气与阵型
- 提供可用的军略卡片供玩家选择（突击、齐射、火攻、计略、撤退等）
- 实时展示战场局势变化与战斗日志
- 支持加速、自动指挥、撤退选项

### 1.3 触发条件
- 在大地图选择敌对城池/部队并执行「进攻」
- 触发剧情战役（如鄱阳湖之战）
- 遭遇敌军伏击等事件

### 1.4 界面特点
- **全景战场视角**：背景为战场地形图（水墨风格），双方部队以“兵牌”形式陈列于战场两侧或上下。
- **阵型可视化**：以图形化方式展示当前阵型（如方圆阵、雁行阵），并显示兵种克制关系。
- **军略卡片**：借鉴太阁4的合战指令，将战术卡片化，消耗“士气”或“谋略值”使用。

## 3. 视觉继承与差异化说明

### 3.1 继承的全局令牌
| 元素 | 使用的令牌 |
|------|------------|
| 面板背景 | `var(--color-bg-secondary)` + 半透明 |
| 面板边框 | `var(--border-default)` |
| 字体 | `--font-serif` |
| 士气条 | 类似气势条，使用 `var(--color-accent-primary)` |
| 卡片 | 继承武技卡片样式，调整为横向或纵向军略卡 |

### 3.2 本界面的特殊处理
- **战场地形背景**：根据战斗地点（平原、山林、水战）切换对应的水墨地图，并影响兵种移动/攻击效果。
- **兵种图标**：采用传统纹样风格的图标（如马、弓、盾），区分骑兵、弓兵、步兵、水军等。
- **阵型指示器**：在双方面板中以小型图标展示当前阵型（方圆、雁行、长蛇等），悬停显示效果说明。
- **士气与兵力联动**：兵力损失时士气可能下降，士气归零则部队溃散。

## 4. 详细组件与状态

### 4.1 战场背景
- 根据战场类型加载不同背景图（如 `bg_battle_plain.webp`、`bg_battle_river.webp`）。
- 背景上方可叠加淡墨网格线（表示战场区域划分）。

### 4.2 双方军势面板
- **容器**：`.force-panel`，宽度 280px，内边距 `12px`。
- **主将名与兵力**：主将名 `--text-heading-sm`，兵力 `--text-body` 使用等宽数字。
- **士气条**：`height: 8px`，填充色 `var(--color-accent-primary)`。
- **阵型图标与名称**：置于士气条下方，字体 `--text-caption`，颜色 `var(--color-accent-blue)`。

### 4.3 部队兵种陈列
- 可使用简化方式：每个兵种显示图标 + 数量。
- 图标尺寸 32×32px，横向排列。
- 若实现完整兵牌，每个兵牌为一张小卡片，显示兵种、数量、状态（混乱、鼓舞等）。

### 4.4 战斗日志区
- **容器**：`.battle-log`，最大高度 80px，内边距 `8px 16px`。
- **文本**：字体 `--font-serif`，字号 `--text-body`。关键事件（如暴击、火攻成功）使用朱砂色。

### 4.5 军略卡片
与个人战的武技卡类似，但主题为战术指令。

**常见军略卡示例**：
| 名称 | 图标 | 消耗（士气/谋略） | 效果 |
|------|------|------------------|------|
| 突击 | 🏇 | 士气 2 | 骑兵冲锋，造成大量伤害，可能混乱 |
| 齐射 | 🏹 | 士气 1 | 弓兵远程攻击，敌方士气下降 |
| 火攻 | 🔥 | 谋略 3 | 造成持续伤害，特定地形（如山林）效果拔群 |
| 计略 | 📜 | 谋略 2 | 施放计策（如挑衅、伪报），影响敌方状态 |
| 坚守 | 🛡️ | 士气 1 | 本回合防御提升，恢复少量士气 |
| 撤退 | 🚪 | 0 | 尝试脱离战场，成功率与将领能力相关 |

- **卡片样式**：`.tactic-card`，宽度 100px，内边距 `8px`。
- **消耗显示**：士气消耗用火焰图标（朱砂），谋略消耗用卷轴图标（黛蓝）。
- **禁用态**：资源不足或条件不满足时置灰。

### 4.6 辅助按钮
与个人战界面共用相同组件（自动、加速、撤退）。

### 4.7 战斗结果弹窗
- **胜利**：「大捷！」，获得功勋、俘虏、物资。
- **失败**：「兵败…」，损失兵力、可能被俘或撤退。

## 5. 交互逻辑与动效

### 5.1 进入动画
- 战场背景由黑场淡入，双方面板由左右滑入，卡片区由下而上滑入。

### 5.2 回合流程
1. 玩家选择一张可用军略卡。
2. 确认后执行战术动画：兵牌移动/攻击、伤害数字弹出、日志更新。
3. 敌方 AI 行动，执行其军略。
4. 回合结束，士气自然变化，状态效果结算。
5. 重复直到一方兵力归零或主将溃逃。

### 5.3 特殊效果
- **兵种克制**：攻击时若克制对方，伤害数字旁显示「克制」印章，伤害增加。
- **阵型变换**：部分军略卡可改变阵型，面板上的阵型图标实时切换。

### 5.4 撤退
- 点击「撤退」弹出确认框，成功率基于将领智力与当前局势。成功则退出战斗返回大地图，失败则损失额外兵力。

## 6. 数据绑定说明

### 6.1 界面需要的数据
| 数据字段 | 类型 | 说明 |
|----------|------|------|
| `playerForce` | object | 玩家军势（主将、兵力、士气、阵型、兵种构成） |
| `enemyForce` | object | 敌方军势 |
| `availableTactics` | array | 当前可用的军略卡列表 |
| `battleLog` | array | 战斗日志 |
| `terrainType` | string | 当前战场地形 |

### 6.2 界面触发的操作
| 操作 | 目标 |
|------|------|
| 点击军略卡 | 执行玩家回合战术 |
| 点击「自动」 | 切换自动指挥 |
| 点击「加速」 | 切换速度倍率 |
| 点击「撤退」 | 尝试撤退 |

## 7. 对 AI 编程助手的生成提示

1. **卡片组件复用**：可复用个人战的 `<SkillCard>` 组件，调整数据源为军略。
2. **战场状态管理**：使用集中式状态存储双方数据，军略执行后更新兵力、士气。
3. **地形影响**：在军略效果计算时，根据 `terrainType` 调整伤害系数。
4. **兵种克制系统**：定义克制表（如骑兵克弓兵、弓兵克枪兵等），在攻击计算时应用。
5. **动画与节奏**：合战节奏比个人战慢，可加入“部队前进”的微动画。

## 9. 附录：太阁立志传4 合战参考要点

- 太阁4合战采用回合制，双方轮流使用战术卡片（如「突击」「射击」「计谋」等）。
- 阵型系统影响攻防，兵种（足轻、骑马、铁炮）有克制关系。
- 士气是重要资源，使用战术消耗士气。
- 本设计在保留经典合战玩法的基础上，强化了视觉表现与阵型可视化。
