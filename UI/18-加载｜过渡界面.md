# 《洪武立志传》界面设计：加载/过渡界面

> 版本：1.0.0  
> 继承自：《洪武立志传》整体 UI 设计规范 v1.0.0  
> 对应模块：`LoadingScreen.js`（待新建）

---

## 1. 界面定位与功能概述

### 1.1 界面位置
加载/过渡界面是游戏在加载资源、切换场景、等待数据时呈现的临时画面。它不仅是技术上的缓冲，更是沉浸式体验的一部分——让等待变得诗意而富有期待感。

### 1.2 主要功能
- 在资源加载或场景切换时覆盖屏幕，防止画面撕裂或空白
- 显示加载进度（进度条、百分比或动态水墨晕染）
- 展示古风诗句、游戏小贴士或随机历史典故
- 提供视觉上的舒缓过渡，减少等待焦虑

### 1.3 触发条件
- 游戏首次启动，加载初始资源时
- 大地图与城镇据点之间的场景切换
- 进入个人战、合战等大型子系统时
- 载入存档时

### 1.4 界面特点
- **水墨晕染动效**：加载进度以墨滴入水、逐渐晕开的动画形式呈现，取代生硬的进度条。
- **诗文小贴士**：屏幕中央或下方轮播随机诗文/小贴士（如「高筑墙，广积粮，缓称王」），配以作者名（如「—— 朱升」）。
- **极简元素**：仅有游戏 Logo、加载指示器、诗文，无多余按钮，保持画面干净。

## 3. 视觉继承与差异化说明

### 3.1 继承的全局令牌
| 元素 | 使用的令牌 |
|------|------------|
| 背景色 | `var(--color-bg-primary)` |
| 标题字体 | `--font-calligraphy` |
| 印章 | `.stamp` 类 |
| 正文字体 | `--font-serif` |

### 3.2 本界面的特殊处理
- **无按钮**：加载界面不提供任何交互按钮，避免玩家误操作。
- **水墨晕染进度**：使用 Canvas 绘制动态墨滴扩散效果，或使用 SVG/CSS 动画模拟笔触绘制。进度 0% 时为一点墨迹，100% 时铺满圆形区域。
- **诗文轮播**：若加载时间较长（超过 3 秒），可每隔几秒切换一条新的诗文。诗文从预设 JSON 库中随机选取。
- **过渡动画**：加载完成时，画面中心墨韵收缩为一点，然后淡出至下一场景。

## 4. 详细组件与状态

### 4.1 背景层
- **宣纸底**：`var(--color-bg-primary)`，可叠加低透明度宣纸纹理图片（`bg_paper_texture.png`）。
- **动态墨染**（可选）：根据加载进度，背景边缘逐渐出现淡墨晕染效果，仿佛画卷被墨色浸润。

### 4.2 标题区
- **游戏 Logo**：与启动界面共用同一资源（`title_hongwu_ligend_800x200.png`）。
- **加载印章**：位于 Logo 右下角，内容为「加载中」或「稍候」，使用 `.stamp` 样式，可带有旋转动画或呼吸效果。

### 4.3 水墨晕染进度（核心）
- **容器**：`.ink-progress`，尺寸 120×120px，居中。
- **实现方式**：
  - **Canvas 绘制**：根据 `progress` (0-1) 绘制从中心向外扩散的半透明墨色圆，边缘模糊。
  - **SVG/CSS**：使用带 `clip-path` 的圆形容器，内部填充墨色，`clip-path` 半径随进度变化。
- **百分比数字**：置于墨韵中央，字体 `--font-mono`，颜色 `var(--color-text-primary)`。
- **完成动画**：100% 时墨韵快速收缩为一点并消失。

### 4.4 诗文展示区
- **容器**：`.loading-quote`，最大宽度 600px，内边距 `16px`。
- **诗句**：字体 `--font-serif`，字号 `--text-heading-sm`（桌面端 22px），颜色 `var(--color-text-primary)`，居中对齐，行高 `1.6`。
- **作者**：字体 `--text-body-sm`，颜色 `var(--color-text-secondary)`，右对齐，前置破折号。
- **切换动画**：诗文切换时淡入淡出（`cross-fade`）。

### 4.5 状态说明
| 状态 | 表现 |
|------|------|
| 加载中 | 墨晕扩散，百分比增加，诗文轮播 |
| 加载完成 | 墨晕收缩，画面淡出，切换场景 |
| 加载失败 | 显示「加载失败，请重试」提示，并提供「重试」按钮（此时需打破无按钮原则） |

## 5. 交互逻辑与动效

### 5.1 进入动画
- 界面整体淡入（300ms）。
- 标题与印章由上而下轻微滑入。
- 墨晕从中心点开始扩散（`scale` 或 `clip-path` 动画）。

### 5.2 进度更新
- 监听资源加载事件，更新 `progress` 值。
- 进度值平滑过渡（使用 `requestAnimationFrame` 或 CSS `transition`）。

### 5.3 诗文轮播
- 若加载时间超过 3 秒，启动定时器，每 5 秒切换一条随机诗文（不与上一条重复）。
- 加载完成时停止轮播。

### 5.4 退出动画
- 进度达到 100% 时，触发退出动画：墨晕收缩至中心点（300ms），然后整体淡出（200ms）。
- 淡出完成后，触发 `onComplete` 回调，进入下一场景。

## 6. 数据绑定说明

### 6.1 界面需要的数据
| 数据字段 | 类型 | 说明 |
|----------|------|------|
| `progress` | number (0-1) | 当前加载进度 |
| `isLoading` | boolean | 是否处于加载状态 |
| `quotes` | array | 诗文库，每项包含 `text` 和 `author` |
| `currentQuote` | object | 当前显示的诗文 |

### 6.2 界面触发的操作
| 操作 | 目标 |
|------|------|
| 加载完成 | 触发退出动画，回调 `onComplete` |
| 加载失败 | 显示重试按钮，点击后重新加载 |

## 7. 对 AI 编程助手的生成提示

1. **Canvas 绘制墨晕**：使用 `ctx.createRadialGradient` 创建中心透明、边缘墨色的渐变，并根据 `progress` 调整渐变半径。
2. **进度平滑**：使用 `useEffect` + `requestAnimationFrame` 实现进度值从当前值向目标值的平滑过渡。
3. **诗文库管理**：将诗文存储为 JSON 数组，在组件中随机选取。
4. **无按钮原则**：正常情况下不渲染任何按钮；仅在错误状态显示「重试」。
5. **古风细节**：墨晕颜色使用 `var(--color-text-primary)` 并降低透明度；印章可添加 `@keyframes pulse` 呼吸动画。
6. **响应式**：移动端墨晕尺寸缩小至 80×80px，诗文字号适应。

## 9. 附录：太阁立志传4 加载过渡参考要点

- 太阁4在场景切换时有短暂的黑色淡入淡出，无进度条，简洁高效。
- 现代玩家对加载时长更敏感，故本设计加入水墨进度动画与诗文轮播，将等待转化为文化享受。
- 若加载极快（<1秒），可跳过加载界面直接切换，避免闪烁。
