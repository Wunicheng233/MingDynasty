# 《洪武立志传》界面设计：存档/读档界面

> 版本：1.0.0  
> 继承自：《洪武立志传》整体 UI 设计规范 v1.0.0  
> 对应模块：`SaveLoadUI.js`（待新建）

---

## 1. 界面定位与功能概述

### 1.1 界面位置
存档/读档界面是玩家管理游戏进度的核心面板，可从启动界面（「继续旧档」）或游戏内菜单（「保存/载入」）进入。

### 1.2 主要功能
- 展示所有存档位（建议 8-12 个）
- 显示每个存档的详细信息：游戏年份、玩家官品、持有银两、游戏进度截图（若有）
- 提供「保存」「载入」「删除」操作
- 支持退出返回上一界面

### 1.3 触发条件
- 启动界面点击「继续旧档」
- 游戏内菜单点击「保存游戏」或「载入游戏」

### 1.4 界面模式
本界面存在两种模式，布局相同但按钮行为有差异：
| 模式 | 入口 | 主要操作 |
|------|------|----------|
| **存档模式** | 游戏内菜单「保存游戏」 | 点击空存档位或已有存档位进行保存（需二次确认覆盖） |
| **读档模式** | 启动界面「继续旧档」/ 游戏内「载入游戏」 | 点击已有存档位进行载入 |

## 3. 视觉继承与差异化说明

### 3.1 继承的全局令牌
| 元素 | 使用的令牌 |
|------|------------|
| 背景色 | `var(--color-bg-primary)` |
| 卡片背景 | `var(--color-bg-secondary)` |
| 卡片边框 | `var(--border-default)` |
| 卡片阴影 | `var(--shadow-md)` |
| 标题字体 | `--font-serif`，字号 `--text-heading-sm` |
| 正文字体 | `--font-serif`，字号 `--text-body-sm` |
| 按钮 | `.btn-primary`、`.btn-secondary`、`.btn-text` |
| 翻页按钮 | `.btn-secondary` 小尺寸变体 |

### 3.2 本界面的特殊处理
- **存档卡片高度**：建议固定高度 280px（移动端 240px），内部截图区域占 60%。
- **空存档位**：背景色稍暗（`rgba(253,251,247,0.6)`），中央显示「空位」二字（书法体），下方仅显示「保存」按钮。
- **截图占位**：无截图时显示古风云纹图案或「暂无截图」文字，避免空白区域过于突兀。
- **当前选中态**：读档模式下，鼠标悬停或键盘聚焦的存档卡片边框变为朱砂色（`var(--border-highlight)`）。

## 4. 详细组件与状态

### 4.1 存档卡片
- **容器**：`.save-slot`，继承 `.card`，内边距 `12px`，宽度自适应（最小 280px）。
- **截图区域**：`height: 160px`，背景色 `var(--color-bg-dark)`（无截图时），`border-radius: var(--radius-sm)`，`overflow: hidden`。
- **信息区域**：
  - 年份：`--text-heading-sm`，颜色 `var(--color-text-primary)`
  - 官品：`--text-body-sm`，颜色 `var(--color-accent-gold)`
  - 银两：`--text-body-sm`，颜色 `var(--color-text-secondary)`，使用等宽数字
- **操作按钮区域**：三个按钮横向排列，间距 `var(--space-sm)`。

### 4.2 存档卡片状态
| 状态 | 截图区 | 信息区 | 按钮区 |
|------|--------|--------|--------|
| **有存档** | 显示游戏截图（若有） | 显示年份、官品、银两 | 读档模式：「载入」「删除」；存档模式：「覆盖」「删除」 |
| **空存档** | 显示「空位」书法字 + 云纹装饰 | 仅显示「无存档」文字 | 存档模式：「保存」；读档模式：无按钮（卡片置灰不可点击） |

### 4.3 顶部导航栏
- **返回按钮**：「← 返回」，使用 `.btn-text`，颜色 `var(--color-accent-blue)`。
- **界面标题**：居中，字体 `--font-serif`，字号 `--text-heading-md`，内容为「存档」或「读档」。
- **关闭按钮**：「✕」，使用 `.btn-text`，颜色 `var(--color-text-tertiary)`。

### 4.4 底部翻页栏
- **上一页/下一页**：使用 `.btn-secondary` 小尺寸（`padding: 4px 12px`）。
- **页码指示器**：`--text-body`，颜色 `var(--color-text-secondary)`。
- 仅当存档位总数超过一页容量时显示。

### 4.5 确认弹窗（模态框）
当执行「覆盖」「删除」等不可逆操作时，弹出确认对话框。
- 标题：「确认覆盖」或「确认删除」
- 内容：简要说明操作后果（如「此操作不可恢复」）
- 按钮：「确认」（朱砂红）与「取消」（墨色描边）

## 5. 交互逻辑与动效

### 5.1 进入动画
- 界面整体由透明渐显（`fade-in`，300ms）
- 存档卡片以轻微上浮 + 淡入方式交错出现（`stagger` 动画，每张卡片延迟 50ms）

### 5.2 卡片交互
- **悬停**：阴影加深至 `var(--shadow-lg)`，边框色变为 `var(--color-border-highlight)`（仅已有存档的卡片）
- **点击卡片**：若为读档模式且有存档，直接触发载入；若为存档模式且为空位，直接触发保存
- **点击按钮**：执行对应操作（保存/载入/删除），删除与覆盖需弹窗确认

### 5.3 翻页交互
- 点击「上一页」「下一页」时，当前页卡片淡出，新页卡片淡入（`cross-fade`，200ms）
- 页面切换时页码指示器同步更新

### 5.4 退出方式
- 点击「返回」或「关闭」按钮，界面淡出并返回上一界面
- 按下 `ESC` 键等同点击「返回」

## 6. 数据绑定说明

### 6.1 界面需要的数据
| 数据字段 | 来源 | 用途 |
|----------|------|------|
| `mode` | 入口参数 | 决定界面为「存档」或「读档」模式 |
| `saveSlots` | 本地存储 / 游戏状态 | 数组，每个元素包含：`id`、`year`、`rank`、`silver`、`screenshot`（可选）、`timestamp` |
| `currentPage` | 界面内部状态 | 当前显示的存档页索引 |
| `slotsPerPage` | 配置常量 | 每页显示的存档位数（建议 6 或 9） |

### 6.2 界面触发的操作
| 操作 | 目标 |
|------|------|
| 保存到空位 | 调用 `saveGame(slotId)`，更新对应存档位数据，刷新界面 |
| 覆盖已有存档 | 弹窗确认后调用 `saveGame(slotId)`，覆盖原数据 |
| 载入存档 | 调用 `loadGame(slotId)`，加载游戏状态并跳转至大地图 |
| 删除存档 | 弹窗确认后调用 `deleteSave(slotId)`，清空对应存档位，刷新界面 |

## 7. 对 AI 编程助手的生成提示

1. **组件拆分**：将存档卡片封装为 `<SaveSlot>` 组件，接收 `slotData`、`mode`、`onSave`、`onLoad`、`onDelete` 等 props。
2. **状态管理**：使用 React/Vue 的响应式状态管理当前页码与存档列表，确保操作后界面即时刷新。
3. **本地存储交互**：封装 `saveManager` 工具模块，统一处理存档的读写与序列化。
4. **截图功能**：若需要存档截图，可在保存时调用 `html2canvas` 或游戏引擎的截屏 API 生成缩略图并存入存档数据。
5. **空状态处理**：当没有任何存档时，可在网格中央显示「暂无存档」提示，并引导玩家开始新游戏。
6. **可访问性**：为每个存档卡片添加 `role="button"` 与 `aria-label`（如「存档1：洪武三年，从五品知州」），支持键盘导航。
7. **响应式适配**：移动端（<640px）改为单列网格，存档卡片宽度 100%。

## 8. 图片资源清单

本界面所需的图片素材明细如下。

### 8.1 存档截图
| 资源 ID | 文件名（建议） | 类型 | 尺寸 | 格式 | 说明 |
|---------|---------------|------|------|------|------|
| `SS_DEFAULT` | `ui_save_slot_placeholder_320x160.png` | 占位图 | 320×160px | PNG | 无截图时的默认占位图，可设计为云纹底 + 「暂无截图」文字。背景透明或与卡片背景融合。 |

### 8.2 空存档位装饰
| 资源 ID | 文件名（建议） | 类型 | 尺寸 | 格式 | 说明 |
|---------|---------------|------|------|------|------|
| `DECOR_EMPTY_SLOT` | `decor_empty_slot_200x120.png` | 装饰图案 | 200×120px | PNG | 空存档位中央显示的图案，可为一枚「空」字印章或云纹图案。背景透明。 |

### 8.3 翻页图标（可选）
| 资源 ID | 文件名（建议） | 类型 | 尺寸 | 格式 | 说明 |
|---------|---------------|------|------|------|------|
| `ICON_CHEVRON_LEFT` | `icon_chevron_left_24x24.svg` | 图标 | 24×24px | SVG | 上一页箭头图标 |
| `ICON_CHEVRON_RIGHT` | `icon_chevron_right_24x24.svg` | 图标 | 24×24px | SVG | 下一页箭头图标 |

> 注：箭头图标优先使用 SVG 或字体图标，若无资源，可用纯文本「〈」「〉」替代。

### 8.4 资源依赖关系
```
SaveLoadUI
├── SS_DEFAULT (推荐，可降级为纯色背景 + 文字)
├── DECOR_EMPTY_SLOT (推荐，可降级为纯文本「空位」)
└── ICON_CHEVRON_LEFT / RIGHT (可选)
```

### 8.5 对 AI 的图片使用提示
- 截图占位图应作为背景图应用于 `.save-slot .screenshot`，并设置 `background-size: cover`。
- 空位装饰图居中显示于 `.save-slot.empty .screenshot` 内。
- 若截图资源缺失，使用纯色背景（`var(--color-bg-dark)`）并显示文字提示。
