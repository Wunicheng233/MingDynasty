# 《洪武立志传》界面设计：个人战界面

> 版本：1.0.0  
> 继承自：《洪武立志传》整体 UI 设计规范 v1.0.0  
> 对应模块：`DuelRenderer.js`（待新建）

---

## 1. 界面定位与功能概述

### 1.1 界面位置
个人战界面是玩家与敌对人物进行一对一单挑、切磋或遭遇战斗的专属界面。它借鉴《太阁立志传4》的**卡片对战**机制，将攻击、防御、特技等动作设计为可收集与使用的“武技卡”。

### 1.2 主要功能
- 展示敌我双方状态（体力、气势、增益/减益效果）
- 提供可用的武技卡片供玩家选择（斩击、突刺、防御、秘技等）
- 实时展示战斗过程与结果反馈
- 支持战斗加速、自动战斗选项

### 1.3 触发条件
- 在城镇或大地图与 NPC 选择「交手」或「挑战」
- 遭遇盗匪、刺客等随机事件
- 武道大会、比武招亲等特殊场景

### 1.4 界面特点
- **卡片式指令**：借鉴太阁4，将战斗指令卡片化，每张卡片有独立的图标、名称、消耗与效果。
- **水墨特效**：攻击命中时屏幕边缘出现墨迹飞溅特效，受击方有震动反馈。
- **动态背景**：根据战斗地点（竹林、校场、街市）切换对应的水墨场景。

## 3. 视觉继承与差异化说明

### 3.1 继承的全局令牌
| 元素 | 使用的令牌 |
|------|------------|
| 状态栏背景 | `var(--color-bg-secondary)` + 半透明 |
| 状态栏边框 | `var(--border-default)` |
| 字体 | `--font-serif` 为主 |
| 体力条填充色 | `var(--color-accent-green)`（青碧） |
| 气势条填充色 | `var(--color-accent-primary)`（朱砂） |
| 卡片背景 | `var(--color-bg-secondary)` |
| 卡片边框 | `var(--border-default)` |

### 3.2 本界面的特殊处理
- **体力条与气势条**：体力为青碧色，气势为朱砂色。气势条影响武技卡的使用（某些秘技需要足够气势）。
- **武技卡片设计**：每张卡片纵向布局，包含图标（水墨刀剑）、名称、气势消耗（以小火焰或气团图标表示）。
- **水墨特效**：攻击命中时，受击方状态栏闪烁红色，屏幕边缘溅射墨点（通过 CSS 伪元素或 canvas 粒子实现）。
- **卡牌禁用态**：若当前气势不足以使用某张卡，卡片整体置灰并带有「气势不足」提示。

## 4. 详细组件与状态

### 4.1 战斗场景背景
- 根据战斗发生地点动态切换背景图。
- 建议尺寸：1920×1080px（桌面端），水墨风格，饱和度低。

### 4.2 双方状态栏
- **容器**：`.combatant-panel`，宽度 260px，内边距 `12px`，背景半透明宣纸，圆角 `var(--radius-md)`。
- **头像/势力图标**：尺寸 48×48px，置于面板左侧或上方。
- **姓名与势力**：姓名 `--text-heading-sm`，势力 `--text-caption` 颜色 `var(--color-accent-blue)`。
- **体力条**：`height: 10px`，背景 `var(--color-border-default)`，填充色 `var(--color-accent-green)`。右侧显示数值。
- **气势条**：`height: 8px`，背景 `var(--color-border-default)`，填充色 `var(--color-accent-primary)`。

### 4.3 战斗日志区
- **容器**：`.battle-log`，背景 `rgba(253,251,247,0.9)`，边框 `var(--border-double)`，内边距 `8px 16px`，圆角 `var(--radius-sm)`。
- **文本**：字体 `--font-serif`，字号 `--text-body`，颜色 `var(--color-text-primary)`。最多显示 3 行，自动滚动。
- **关键事件高亮**：暴击用朱砂色，格挡用黛蓝色。

### 4.4 武技卡片（手牌）
- **容器**：`.skill-card`，宽度 `100px`（桌面端），内边距 `8px`，背景 `var(--color-bg-secondary)`，边框 `var(--border-default)`，圆角 `var(--radius-md)`，阴影 `var(--shadow-sm)`。
- **悬停/选中**：边框 `var(--border-highlight)`，阴影 `var(--shadow-md)`，轻微上浮。
- **图标**：尺寸 `40×40px`，居中，使用水墨风格 SVG/PNG。
- **名称**：字体 `--font-serif`，字号 `--text-body-sm`，居中。
- **气势消耗**：火焰图标 + 数字，字体 `--text-caption`，颜色 `var(--color-accent-orange)`。
- **禁用态**：`opacity: 0.5`，`filter: grayscale(0.5)`，不可点击。

**常见武技卡示例**：
| 名称 | 图标 | 气势消耗 | 效果 |
|------|------|----------|------|
| 斩击 | ⚔️ | 1 | 基础攻击，造成 10-15 伤害 |
| 突刺 | 🗡️ | 2 | 穿透攻击，无视部分防御 |
| 防御 | 🛡️ | 1 | 本回合减伤 50%，恢复少量气势 |
| 横扫 | 🌀 | 3 | 高伤害，但命中略低 |
| 秘技·破军 | ✨ | 4 | 必定暴击，消耗所有气势 |

### 4.5 辅助按钮
| 按钮 | 样式 | 行为 |
|------|------|------|
| 自动 | `.btn-text` | 开启自动战斗，AI 自动出牌 |
| 加速 | `.btn-text` | 切换 2 倍速/正常速 |
| 认输 | `.btn-secondary`（小型） | 弹出确认框，认输后结束战斗 |

### 4.6 战斗结果弹窗
战斗结束后，弹出结果模态框：
- **胜利**：「大获全胜！」，获得经验、声望、可能掉落物品。
- **失败**：「技不如人…」，根据场景可能受伤、损失银两或结束游戏。
- **按钮**：「继续」（返回大地图/城镇）。

## 5. 交互逻辑与动效

### 5.1 进入动画
- 场景背景由黑场淡入，双方状态栏由左右两侧滑入（`slide-in`），手牌区由下而上滑入。

### 5.2 回合流程
1. 玩家点击一张可用武技卡。
2. 卡片高亮，播放选中音效（可选）。
3. 若需确认，显示确认按钮；若为即时制，直接执行。
4. 执行攻击动画：受击方晃动，屏幕边缘墨点飞溅，战斗日志更新。
5. 双方状态条更新（体力、气势变化）。
6. 敌方行动（若未击败）：AI 选择卡片并执行，更新日志与状态。
7. 重复直到一方体力归零。

### 5.3 特殊效果动画
- **暴击**：伤害数字放大、红色、震动。
- **格挡**：显示「格挡」文字，伤害减少。
- **气势攒满**：气势条闪烁金光，可使用秘技。

### 5.4 自动与加速
- 自动战斗：AI 接管玩家出牌，按钮变为「手动」。
- 加速：游戏内计时器/动画速度加倍。

## 6. 数据绑定说明

### 6.1 界面需要的数据
| 数据字段 | 类型 | 说明 |
|----------|------|------|
| `player` | object | 玩家战斗属性（体力、气势、武技列表） |
| `enemy` | object | 敌方战斗属性 |
| `availableSkills` | array | 当前手牌（武技卡）列表 |
| `battleLog` | array | 最近几条战斗日志文本 |
| `isAuto` | boolean | 是否自动战斗 |
| `speed` | number | 战斗速度倍率 |

### 6.2 界面触发的操作
| 操作 | 目标 |
|------|------|
| 点击武技卡 | 执行玩家回合行动 |
| 点击「自动」 | 切换自动战斗状态 |
| 点击「加速」 | 切换速度倍率 |
| 点击「认输」 | 结束战斗，判定失败 |

## 7. 对 AI 编程助手的生成提示

1. **卡片组件封装**：`<SkillCard>` 接收 `skill` 对象，处理禁用、点击事件。
2. **战斗状态机**：建议使用状态机管理回合（`PLAYER_TURN`、`ENEMY_TURN`、`ANIMATING`、`ENDED`），避免动画冲突。
3. **动画实现**：使用 CSS 类切换 + `setTimeout` 控制时序。墨点特效可通过绝对定位的 `div` 配合 `transform` 随机分布。
4. **气势系统**：每回合自然恢复少量气势，使用武技消耗气势。秘技可设置冷却。
5. **古风细节**：状态栏可添加云纹边框；手牌区背景可模拟皮革或锦缎纹理。

## 9. 附录：太阁立志传4 个人战参考要点

- 太阁4个人战以卡片形式进行，玩家手牌包含「斩击」「突刺」「防御」「秘技」等。
- 气势槽（游戏中称为“气合”）影响强力技能的使用。
- 战斗画面简洁，重点突出卡片与状态条。
- 本设计在保留经典卡片对战的基础上，加强了水墨视觉表现与手牌交互体验。
