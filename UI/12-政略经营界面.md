> 版本：1.0.0

> 对应模块：`ManageRenderer.js`（待新建）

## 1. 界面定位与功能概述

### 1.1 界面位置
政略经营界面是玩家管理所辖城池、发展内政、调配资源的核心面板。它借鉴《太阁立志传4》的城池管理逻辑，将「开发」「商业」「征兵」「人事」等指令以卡片形式呈现，并直观展示城池的三维发展度（农业、商业、军备）。

### 1.2 主要功能
- 展示当前管理城池的基本信息（名称、规模、所属势力、太守）
- 以进度条或数值展示城池三维：农业、商业、军备
- 提供内政指令卡片：开发、劝农、通商、征兵、训练、筑城、人事、计略
- 显示当前任务队列与执行进度
- 展示城池收入、存粮、兵力等资源摘要

### 1.3 触发条件
- 在城镇据点界面点击「官邸」
- 作为城主或太守进入所辖城池的政务厅
- 受君主之命管理某城

### 1.4 界面特点
- **仪表盘式概览**：上方以三个大卡片展示核心三维，一目了然。
- **指令卡片网格**：内政指令以网格形式排布，每个指令有图标、名称、消耗（银两/行动力）。
- **任务队列可视化**：下方显示当前正在执行的内政任务，带有进度条与剩余时间。

## 3. 视觉继承与差异化说明

### 3.1 继承的全局令牌
| 元素 | 使用的令牌 |
|------|------------|
| 面板背景 | `var(--color-bg-secondary)` + 半透明 |
| 进度条 | 农业（青碧）、商业（古铜金）、军备（朱砂） |
| 字体 | `--font-serif` |
| 卡片 | 继承通用卡片样式 |

### 3.2 本界面的特殊处理
- **三维配色**：农业使用青碧色（`var(--color-accent-green)`），商业使用古铜金（`var(--color-accent-gold)`），军备使用朱砂色（`var(--color-accent-primary)`），形成视觉区分。
- **内政指令卡片**：采用横向或纵向卡片，显示指令名、消耗（银两/行动力），悬停显示效果预览。
- **任务队列**：显示当前正在执行的任务，进度条动态增长，完成时弹出提示并更新三维数值。

## 4. 详细组件与状态

### 4.1 城池基本信息栏
- **内容**：太守姓名、城池规模（小城/中城/大都城）、人口数量。
- **样式**：内边距 `8px 16px`，字体 `--text-body`。

### 4.2 三维概览卡片
- **容器**：`.stat-card`，宽度 1:1:1 等分，内边距 `16px`，居中对齐。
- **标题**：图标 + 文字（「农业」「商业」「军备」），字体 `--text-heading-sm`。
- **进度条**：`height: 12px`，圆角 `6px`，背景 `var(--color-border-default)`。
- **数值**：进度条下方显示 `当前值 / 最大值`。

### 4.3 资源摘要栏
- **存粮**：🌾 图标 + 数值，颜色 `var(--color-accent-green)`。
- **银两**：🪙 图标 + 数值，颜色 `var(--color-accent-gold)`。
- **兵力**：⚔️ 图标 + 数值，颜色 `var(--color-accent-primary)`。

### 4.4 内政指令卡片
每个指令卡片包含：
- **图标**：尺寸 40×40px，居中。
- **指令名**：字体 `--font-serif`，字号 `--text-body`。
- **消耗**：银两图标 + 数值（或行动力消耗）。
- **悬停提示**：显示指令效果预览（如「农业 +3，消耗 50 两」）。

**常见内政指令**：
| 指令 | 图标 | 消耗 | 效果 |
|------|------|------|------|
| 开发 | 🏗️ | 银 50 | 农业 +2，商业 +1 |
| 劝农 | 🌾 | 银 30 | 农业 +3 |
| 通商 | 🛒 | 银 40 | 商业 +3，收入增加 |
| 征兵 | ⚔️ | 银 80，人口-500 | 兵力 +1000 |
| 训练 | 🎯 | 银 30 | 军备 +2，士气提升 |
| 筑城 | 🏯 | 银 200 | 城池规模提升（需条件） |
| 人事 | 👥 | 银 20 | 招募/登用武将 |
| 计略 | 📜 | 银 50 | 执行计略（破坏、流言等） |

### 4.5 任务队列区
- **容器**：`.task-queue`，内边距 `8px 16px`，背景半透明。
- **任务名称**：如「劝农」，字体 `--text-body`。
- **进度条**：`height: 8px`，填充色 `var(--color-accent-primary)`。
- **剩余回合**：如「剩余 2 回合」。
- **多任务**：若支持同时执行多个任务，以列表形式展示。

### 4.6 底部操作栏
- **评定**：进入评定界面，接受君主发布的任务（详见《任务与评定界面》）。
- **离开**：返回城镇据点界面。

## 5. 交互逻辑与动效

### 5.1 进入动画
- 界面整体淡入上浮（`fade-in-up`，300ms）。
- 三维进度条带有填充动画（由 0 增长至当前值，500ms）。

### 5.2 执行内政指令
1. 点击可用指令卡片。
2. 若资源足够，弹出确认框：「确定执行『劝农』？消耗银 30，预计农业 +3」。
3. 确认后，资源扣除，任务加入队列，进度条开始增长。
4. 任务完成时，三维数值更新，弹出完成提示。

### 5.3 任务队列管理
- 任务按顺序执行，每回合推进一次进度。
- 玩家可点击任务旁的「✕」取消任务（可能返还部分资源）。

### 5.4 评定与离开
- 「评定」：切换到评定界面。
- 「离开」：返回城镇界面。

## 6. 数据绑定说明

### 6.1 界面需要的数据
| 数据字段 | 类型 | 说明 |
|----------|------|------|
| `city` | object | 当前城池信息（名称、规模、人口、太守） |
| `stats` | object | 农业、商业、军备当前值及最大值 |
| `resources` | object | 存粮、银两、兵力 |
| `availableActions` | array | 当前可用的内政指令列表 |
| `taskQueue` | array | 当前进行中的任务列表 |

### 6.2 界面触发的操作
| 操作 | 目标 |
|------|------|
| 点击指令卡片 | 执行内政任务，加入队列 |
| 取消任务 | 从队列移除任务 |
| 点击「评定」 | 进入评定界面 |
| 点击「离开」 | 返回城镇界面 |

## 7. 对 AI 编程助手的生成提示

1. **三维进度条**：封装 `<StatProgress>` 组件，接收 `value`、`max`、`color`。
2. **指令卡片网格**：使用 CSS Grid 实现响应式网格（`display: grid; grid-template-columns: repeat(auto-fill, minmax(120px, 1fr))`）。
3. **任务队列系统**：使用数组存储任务，每回合（或玩家点击「下一回合」时）更新进度。
4. **资源检查**：执行指令前校验银两/存粮是否充足，不足时卡片置灰并显示提示。
5. **古风细节**：三维卡片可添加对应的传统纹样装饰（如麦穗、铜钱、刀剑纹）。

## 9. 附录：太阁立志传4 城池经营参考要点

- 太阁4中，成为城主后可管理城池，执行「开垦」「商业」「征兵」「筑城」等指令。
- 城池有石高（农业）、商业、兵力等数值，指令消耗行动力。
- 评定系统是接受主君任务的核心机制。
- 本设计在保留经典内政玩法的基础上，将信息可视化，并引入任务队列使经营节奏更清晰。
