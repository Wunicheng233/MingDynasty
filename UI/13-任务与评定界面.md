# 《洪武立志传》界面设计：任务与评定系统

> 版本：1.0.0  
> 继承自：《洪武立志传》整体 UI 设计规范 v1.0.0  
> 对应模块：`QuestRenderer.js`

---

## 1. 界面定位与功能概述

### 1.1 界面位置
任务与评定系统是玩家接受主君/上级指令、查看当前任务进度、获取功勋与奖励的核心面板。借鉴《太阁立志传4》的**评定会议**机制，定期召开评定，主君发布任务卡片供玩家选择，完成任务后获得功勋、晋升与赏赐。

### 1.2 主要功能
- 展示当前进行中的任务列表与进度
- 在评定会议中展示可选的任务卡片（主君发布）
- 查看已完成任务的历史记录
- 显示功勋值、官品晋升条件
- 领取任务奖励

### 1.3 触发条件
- 在官邸/评定间选择「评定」
- 特定时间自动触发（如每季度评定会议）
- 主君紧急召见（剧情事件）

### 1.4 界面形式
| 形式 | 触发场景 | 特点 |
|------|----------|------|
| **任务列表视图** | 日常查看任务进度 | 简洁列表，展示当前任务 |
| **评定会议视图** | 参加评定时 | 全屏/半屏，主君形象、任务卡片陈列 |

> 本文档以**评定会议视图**为主，任务列表可视为其简化版本。

## 3. 视觉继承与差异化说明

### 3.1 继承的全局令牌
| 元素 | 使用的令牌 |
|------|------------|
| 面板背景 | `var(--color-bg-secondary)` |
| 卡片边框 | `var(--border-default)` |
| 进度条 | 继承政略经营界面的进度条样式 |
| 字体 | `--font-serif` |
| 按钮 | `.btn-primary`（接受任务）、`.btn-secondary`（告辞） |

### 3.2 本界面的特殊处理
- **主君形象**：在评定会议中，顶部展示君主半身像与发言气泡，增强代入感。
- **任务卡片**：借鉴太阁4的任务选择，每张卡片标注任务名、目标、奖励功勋、期限。
- **期限倒计时**：任务卡片与当前任务条中显示剩余时间（月/日），逾期可能降低功勋或受罚。
- **功勋晋升条**：底部醒目展示当前功勋值与下一官品所需功勋，激励玩家完成任务。

## 4. 详细组件与状态

### 4.1 主君发言区（评定专用）
- **君主头像**：尺寸 80×80px，边框 `var(--border-gold)`。
- **发言气泡**：古籍引文框样式，内边距 `12px 20px`，最大宽度 600px。
- **文本**：字体 `--font-serif`，字号 `--text-body`，颜色 `var(--color-text-primary)`。

### 4.2 任务卡片
- **容器**：`.quest-card`，宽度 220px，内边距 `16px`，背景 `var(--color-bg-secondary)`，边框 `var(--border-default)`，圆角 `var(--radius-md)`。
- **任务名**：字体 `--font-serif` 加粗，字号 `--text-heading-sm`，颜色 `var(--color-text-primary)`，带小图标。
- **任务目标**：字体 `--text-body-sm`，颜色 `var(--color-text-secondary)`。
- **奖励功勋**：功勋图标 + 数值，颜色 `var(--color-accent-gold)`。
- **期限**：沙漏图标 + 「期限：X 个月」，字体 `--text-caption`，颜色 `var(--color-accent-orange)`（临近时变红）。
- **接受按钮**：`.btn-primary` 小尺寸，置于卡片底部。
- **已接受/不可选**：若已接受其他任务或条件不足，卡片置灰，按钮变为「不可接受」。

**任务类型示例**：
| 类型 | 图标 | 目标示例 |
|------|------|----------|
| 内政 | 🌾 | 将某城农业提升至 80 |
| 军备 | ⚔️ | 招募 3000 兵力 |
| 外交 | 🤝 | 与某势力亲善 |
| 讨伐 | 🏇 | 击败指定敌军 |
| 筑城 | 🏯 | 完成城池扩建 |

### 4.3 当前任务条
- **容器**：`.current-quest`，内边距 `8px 16px`，背景半透明，固定在底部（任务卡片下方）。
- **任务名**：字体 `--text-body`。
- **进度条**：`height: 8px`，宽度 200px，填充色 `var(--color-accent-primary)`。
- **剩余时间**：沙漏图标 + 剩余月数。

### 4.4 功勋与官品栏
- **容器**：`.rank-progress`，内边距 `8px 16px`。
- **当前功勋/下一级**：字体 `--text-body`，颜色 `var(--color-accent-gold)`。
- **晋升进度条**：`height: 8px`，背景 `var(--color-border-default)`，填充色 `var(--color-accent-gold)`。
- **官品显示**：当前官品（如「从五品」）→ 下一官品（如「正五品」）。

### 4.5 历史记录（任务列表视图）
在非评定状态下，玩家可查看「任务记录」：
- 已完成任务列表，显示任务名、完成时间、获得功勋。
- 使用纵向列表，每项带完成印章（「完」）。

## 5. 交互逻辑与动效

### 5.1 评定进入动画
- 朝堂背景由黑场淡入，君主形象由上而下出现（`slide-down`），任务卡片以交错淡入上浮出现。

### 5.2 接受任务
1. 点击任务卡片的「接受」按钮。
2. 若已有进行中任务，弹出确认框：「当前有未完成任务，是否放弃并接受新任务？」
3. 确认后，当前任务替换为新任务，卡片区关闭/收起，当前任务条更新。

### 5.3 任务进度更新
- 任务进度由游戏逻辑驱动（如完成一次开垦，进度 +10%）。
- 进度条实时更新，完成时弹出「任务完成！请回复主君领取赏赐」提示。

### 5.4 复命与领奖
- 在评定会议或官邸选择「复命」，若任务已完成，君主发言祝贺，功勋增加，可能晋升官品或获得赏赐。

### 5.5 告辞
- 点击「告辞」退出评定，返回城镇或大地图。

## 6. 数据绑定说明

### 6.1 界面需要的数据
| 数据字段 | 类型 | 说明 |
|----------|------|------|
| `availableQuests` | array | 本次评定可选的任务列表 |
| `currentQuest` | object | 当前进行中的任务（可为空） |
| `questProgress` | number | 当前任务完成进度 (0-100) |
| `merit` | number | 当前功勋值 |
| `nextRankMerit` | number | 晋升下一官品所需功勋 |
| `currentRank` | string | 当前官品名称 |
| `nextRank` | string | 下一官品名称 |
| `lordDialog` | string | 君主发言文本 |

### 6.2 界面触发的操作
| 操作 | 目标 |
|------|------|
| 接受任务 | 设置 `currentQuest`，扣除可选任务（或保留但标记已选） |
| 复命 | 检查 `questProgress`，若完成则结算奖励 |
| 告辞 | 退出评定界面 |

## 7. 对 AI 编程助手的生成提示

1. **任务数据结构**：定义 `Quest` 类型，包含 id、name、type、target、progress、rewardMerit、deadline 等字段。
2. **评定任务生成**：评定任务可由后端逻辑根据当前局势动态生成，或从预设任务池抽取。
3. **进度更新机制**：在相关游戏行为（如执行开垦）后，调用 `updateQuestProgress` 方法。
4. **期限系统**：每经过一个月，检查任务期限，逾期则触发惩罚逻辑（如功勋扣除、君主不满）。
5. **古风细节**：任务卡片可加盖「急」或「密」印章表示特殊性质；完成印章使用朱红色「完」字。

## 9. 附录：太阁立志传4 评定系统参考要点

- 太阁4中，定期参加评定会议，主君发布多个任务（如「开发新田」「调查岩村城」），玩家可择一接受。
- 任务有期限，完成后获得功勋，功勋积累可晋升官位。
- 评定界面简洁，以文字选项为主。
- 本设计将任务卡片化，并增加了君主立绘与发言气泡，强化沉浸感与策略选择。
