# 《洪武立志传》界面设计：商店/交易界面

> 版本：1.0.0  
> 继承自：《洪武立志传》整体 UI 设计规范 v1.0.0  
> 对应模块：`ShopRenderer.js`（待新建）

---

## 1. 界面定位与功能概述

### 1.1 界面位置
商店/交易界面是玩家在市集、商铺、行商等场所进行物品买卖、资源交换的核心交互面板。借鉴《太阁立志传4》的座（交易所）与米屋、马屋等交易逻辑，玩家可在此购入所需物资或出售闲置物品以获取银两。

### 1.2 主要功能
- 展示商人头像、名称与欢迎语
- 展示可购买的商品列表（武器、防具、书籍、消耗品、资源等）
- 展示玩家背包中的可出售物品
- 支持批量购买/出售，支持讨价还价（根据魅力/智力）
- 显示玩家持有银两，实时计算交易总额

### 1.3 触发条件
- 在城镇据点点击「市集」或「商铺」设施
- 与行商 NPC 对话选择「交易」
- 特定事件中的物资采购

### 1.4 界面形式
| 形式 | 触发场景 | 特点 |
|------|----------|------|
| **标准商店界面** | 城镇固定商铺 | 全屏或大模态框，商品列表固定 |
| **行商界面** | 偶遇行商 | 商品随机、限时，可能有好货或假货 |

> 本文档以**标准商店界面**为主要描述对象，行商版本可在此基础上增加随机性与限时标签。

## 3. 视觉继承与差异化说明

### 3.1 继承的全局令牌
| 元素 | 使用的令牌 |
|------|------------|
| 背景 | `var(--color-bg-primary)` |
| 列表面板背景 | `var(--color-bg-secondary)` |
| 列表项边框 | `var(--border-default)` |
| 字体 | `--font-serif` |
| 按钮 | `.btn-primary`（成交）、`.btn-secondary`（全部买入/卖出） |
| 银两显示 | 铜钱图标 + `--font-mono`，颜色 `var(--color-accent-gold)` |

### 3.2 本界面的特殊处理
- **商人气泡**：使用古籍引文框样式，左侧商人头像，右侧发言气泡。可随机切换欢迎语。
- **双列表对比**：左右列表对称，中间可有「→」「←」箭头图标表示买卖方向。
- **价格颜色**：玩家可出售物品的价格若高于商店售价，可用青碧色表示「推荐卖出」；若买入价格低于市场均价，可用朱砂色表示「划算」。
- **讨价还价**：点击「成交」前，可触发「还价」按钮（需魅力/智力检定），成功则价格浮动。

## 4. 详细组件与状态

### 4.1 商人信息栏
- **商人头像**：尺寸 64×64px，圆形或圆角，边框 `var(--border-default)`。
- **商人名称**：置于头像下方或右侧，字体 `--text-heading-sm`。
- **欢迎语气泡**：背景 `var(--color-bg-secondary)`，边框 `var(--border-double)`，内边距 `8px 16px`，圆角 `var(--radius-md)`。文本字体 `--text-body`。

### 4.2 商品/背包列表
- **列表容器**：`.shop-list`，最大高度 400px，`overflow-y: auto`，内边距 `8px`。
- **列表项**：`.shop-item`，内边距 `8px 12px`，边框底部 `1px solid var(--color-border-default)`，`display: flex`，`align-items: center`。
- **物品图标**：尺寸 32×32px，右边距 `12px`。
- **物品信息**：名称（`--text-body`）、价格（`--text-body-sm`，铜钱图标+数值）。
- **数量控制**：购买/出售时，列表项右侧显示数量选择器（`-` `数量` `+`）。
- **悬停**：背景 `rgba(158,42,43,0.03)`。

### 4.3 数量选择器
- **容器**：`.quantity-picker`，`display: flex`，`align-items: center`，`gap: 8px`。
- **按钮**：「-」「+」，尺寸 24×24px，背景透明，边框 `var(--border-default)`，圆角 `var(--radius-sm)`。
- **数量**：字体 `--font-mono`，最小宽度 30px，居中对齐。

### 4.4 交易摘要栏
- **持有银两**：左侧显示，铜钱图标 + 数值。
- **本次交易**：右侧显示「买入 X 两」「卖出 X 两」，差额合计。
- **差额正负**：若卖出总额 > 买入总额，差额用青碧色；反之用赭石色。

### 4.5 操作按钮
| 按钮 | 样式 | 行为 |
|------|------|------|
| 全部买入 | `.btn-secondary` | 将商店所有商品加入买入清单（数量最大化，受银两限制） |
| 全部卖出 | `.btn-secondary` | 将背包所有可出售物品加入卖出清单 |
| 成交 | `.btn-primary` | 执行交易，扣除/增加银两，转移物品 |
| 还价 | `.btn-text` | 触发讨价还价检定，成功则价格浮动 ±10%~20% |

### 4.6 讨价还价弹窗（可选）
- 点击「还价」后弹出小窗，显示「掌柜沉吟片刻...」，进行魅力/智力检定动画。
- 成功：「掌柜爽快答应，给你便宜了 X 两！」
- 失败：「掌柜摇头，分文不让。」

### 4.7 空状态
- 商店无商品：「今日货已售罄，客官改日再来。」
- 背包无可出售：「客官囊中空空，不如买点什么？」

## 5. 交互逻辑与动效

### 5.1 进入动画
- 界面淡入上浮（`fade-in-up`，300ms）。
- 商人气泡带有轻微弹跳动画。

### 5.2 选择物品
- 点击商店列表项：数量设为 1，更新买入总额。
- 点击背包列表项：数量设为 1，更新卖出总额。
- 使用「+」「-」调整数量，实时更新总额。

### 5.3 成交
- 点击「成交」后，播放银两飞入/飞出动画，物品图标从一侧移动到另一侧。
- 交易完成后弹出 Toast：「交易完成，花费/获得 X 两。」

### 5.4 全部买入/卖出
- 自动计算可买入/卖出的最大数量，更新列表中的数量，总额刷新。

## 6. 数据绑定说明

### 6.1 界面需要的数据
| 数据字段 | 类型 | 说明 |
|----------|------|------|
| `shopItems` | array | 商店出售的商品列表（含库存、价格） |
| `playerInventory` | array | 玩家背包物品 |
| `playerSilver` | number | 玩家持有银两 |
| `buyCart` | object | 买入清单 `{ [itemId]: quantity }` |
| `sellCart` | object | 卖出清单 `{ [itemId]: quantity }` |
| `merchant` | object | 商人信息（名称、头像、欢迎语） |
| `bargainMultiplier` | number | 当前讨价还价倍率（默认 1.0） |

### 6.2 界面触发的操作
| 操作 | 目标 |
|------|------|
| 修改数量 | 更新 `buyCart` / `sellCart` |
| 成交 | 调用 `executeTransaction(buyCart, sellCart)` |
| 全部买入/卖出 | 自动填充购物车 |
| 还价 | 触发检定，更新 `bargainMultiplier` |

## 7. 对 AI 编程助手的生成提示

1. **购物车状态管理**：使用 `useReducer` 或 Vuex 管理买入/卖出清单，便于撤销与批量操作。
2. **价格计算**：封装 `calculateTotal` 方法，结合 `bargainMultiplier` 计算最终价格。
3. **交易执行**：交易前校验银两、库存、容量，成功后更新全局物品与银两状态。
4. **讨价还价系统**：可设计为概率检定（魅力/智力影响），或小游戏（如快速点击）。
5. **古风细节**：列表项可添加小印章标识「新」「热卖」；银两显示使用铜钱图标。
6. **响应式**：移动端双列表改为上下布局或 Tab 切换。

## 9. 附录：太阁立志传4 交易系统参考要点

- 太阁4中，不同城镇的「座」出售不同商品，价格受供求影响。
- 米屋可买卖军粮，马屋可购买军马。
- 讨价还价是提升智力经验的一种方式。
- 本设计将买卖整合为统一的双列表界面，并保留讨价还价玩法，更符合现代 UI 习惯。
