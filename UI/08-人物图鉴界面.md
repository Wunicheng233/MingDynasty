# 《洪武立志传》界面设计：人物图鉴系统

> 版本：1.0.0  
> 继承自：《洪武立志传》整体 UI 设计规范 v1.0.0  
> 对应模块：`CollectionRenderer.js`

---

## 1. 界面定位与功能概述

### 1.1 界面位置
人物图鉴系统是玩家查看已收集人物、浏览人物资料、追踪收集进度的图鉴式界面。借鉴《太阁立志传4》的**相夹式卡片册**设计，将每个人物视为一张可收集的卡牌。

### 1.2 主要功能
- 以网格形式展示所有可收集的人物卡片
- 区分「已获得」与「未获得」状态（未获得显示剪影或「？」）
- 显示总体收集进度
- 提供分类筛选功能（全部/已获得/未获得/文臣/武将/其他）
- 提供排序功能（按姓名、品级、亲密度等）
- 点击人物卡片可进入人物详情界面

### 1.3 触发条件
- 从主菜单或城镇自宅选择「人物图鉴」
- 从某些事件中跳转（如获得新人物时提示「已收录入图鉴」）

### 1.4 界面特点
- **相夹式卡片册**：卡片以规整网格排列，背景模拟古籍书页，卡片带有「插卡槽」般的阴影与边框。
- **收集成就感**：顶部醒目展示收集进度（如「已收录 48/120 人」），并配以印章式成就标记。
- **状态区分明显**：已获得人物显示彩色立绘与完整信息；未获得人物显示灰色剪影，并标注「???」，激发玩家探索欲。

## 3. 视觉继承与差异化说明

### 3.1 继承的全局令牌
| 元素 | 使用的令牌 |
|------|------------|
| 背景色 | `var(--color-bg-primary)` |
| 卡片背景 | `var(--color-bg-secondary)` |
| 卡片边框 | `var(--border-default)` |
| 卡片阴影 | `var(--shadow-md)` |
| 字体 | `--font-serif` 为主 |
| 进度条填充色 | `var(--color-accent-primary)` |
| 标签按钮 | `.btn-secondary` 小尺寸变体 |

### 3.2 本界面的特殊处理
- **古籍书页背景**：在宣纸底基础上，叠加淡淡的古籍版式线（如天头地脚、鱼尾），模拟线装书对页展开的效果。
- **卡片「插卡槽」感**：每张卡片底部带有弧形阴影，仿佛插入相册卡槽。悬停时卡片轻微「抽出」效果（上浮）。
- **未获得卡片**：立绘区域显示深色剪影（人物轮廓），中央叠加「？」印章。品级显示「未获得」，不可点击进入详情。
- **收集进度条**：采用双行设计——进度条 + 数字百分比，进度条可分段变色（如 0-30% 赭石，30-70% 古铜，70-100% 朱砂）。
- **分类标签**：使用仿古籍版刻的标签页设计，选中项下方有朱砂色下划线。

## 4. 详细组件与状态

### 4.1 收集进度条
- **容器**：`.collection-progress`，内边距 `var(--space-md)`，背景 `var(--color-bg-secondary)`，边框 `var(--border-default)`，圆角 `var(--radius-md)`。
- **进度条**：`height: 12px`，背景 `var(--color-border-default)`，圆角 `6px`。填充色根据进度百分比动态变化。
- **数值文本**：右侧显示「已收录 X / Y」，字体 `--text-body`，颜色 `var(--color-text-primary)`。

### 4.2 分类筛选标签
- **容器**：`.filter-tabs`，`display: flex`，`gap: var(--space-sm)`。
- **标签按钮**：`.filter-tab`，内边距 `6px 16px`，背景透明，文字 `--text-body-sm`，颜色 `var(--color-text-secondary)`，圆角 `var(--radius-sm)`。
- **选中态**：`.filter-tab.active`，文字颜色 `var(--color-accent-primary)`，底部边框 `2px solid var(--color-accent-primary)`。
- **排序下拉**：置于最右侧，使用 `.dropdown` 组件（参见设置界面）。

### 4.3 人物卡片
卡片是图鉴的核心组件，分为「已获得」与「未获得」两种状态。

**卡片容器**（通用）：
- `.person-card`，宽度 `160px`（桌面端），内边距 `12px`，背景 `var(--color-bg-secondary)`，边框 `var(--border-default)`，圆角 `var(--radius-md)`，阴影 `var(--shadow-md)`。
- 悬停：`transform: translateY(-6px)`，阴影 `var(--shadow-lg)`，边框 `var(--border-highlight)`。
- 点击：进入人物详情界面（仅已获得）。

**已获得卡片**：
| 元素 | 样式 |
|------|------|
| 立绘区 | `height: 140px`，`background-color: var(--color-bg-dark)`（作为占位），居中显示人物头像/半身像。图片 `object-fit: cover` 或 `contain`。 |
| 姓名 | 字体 `--font-serif` 加粗，字号 `--text-heading-sm`，颜色 `var(--color-text-primary)`，居中对齐，上边距 `8px`。 |
| 品级 | 字体 `--text-caption`，颜色 `var(--color-accent-gold)`，居中对齐。 |
| 亲密度（可选） | 小进度条或心形图标，置于卡片底部。 |

**未获得卡片**：
| 元素 | 样式 |
|------|------|
| 立绘区 | 背景 `var(--color-bg-dark)`，中央显示「？」印章（可用 CSS 或图片），或人物剪影轮廓。 |
| 姓名 | 显示「？？？」（若有线索可显示部分信息），颜色 `var(--color-text-tertiary)`。 |
| 品级 | 显示「未获得」，颜色 `var(--color-text-tertiary)`。 |
| 交互 | 不可点击进入详情（可弹出提示「尚未结识此人」）。 |

### 4.4 翻页组件
- 与存档界面共用同一套翻页组件（`.pagination`）。
- 仅当总页数 > 1 时显示。

### 4.5 空状态
当筛选条件下无任何卡片时，显示空状态提示：
- 中央显示一枚印章图案（「暂无」）或古风云纹。
- 文字：「暂无符合条件的人物」，字体 `--text-body`，颜色 `var(--color-text-tertiary)`。

## 5. 交互逻辑与动效

### 5.1 进入动画
- 界面整体由下而上滑入（`slide-up`，300ms），伴随轻微淡入。
- 卡片以交错方式淡入上浮（`stagger`，每张延迟 20ms）。

### 5.2 筛选与排序
- 点击分类标签：立即过滤卡片网格，已获得/未获得卡片数量变化时，进度条同步更新。
- 切换排序：卡片网格重新排列，带有交叉淡入淡出效果（`cross-fade`，200ms）。

### 5.3 卡片交互
- 悬停已获得卡片：立绘轻微放大（`scale: 1.02`），显示更多信息（如亲密度）。
- 点击已获得卡片：进入人物详情界面（全屏或模态框）。
- 点击未获得卡片：弹出 Tooltip：「尚未结识此人，多在酒馆、市集等地打探消息吧」。

### 5.4 翻页交互
- 点击翻页：当前页卡片淡出，新页卡片淡入。

## 6. 数据绑定说明

### 6.1 界面需要的数据
| 数据字段 | 来源 | 用途 |
|----------|------|------|
| `allPersons` | 静态数据/存档 | 所有可收集人物的完整列表 |
| `collectedIds` | 游戏存档 | 已获得人物的 ID 数组 |
| `personDetails` | 游戏状态 | 每个人物的详细信息（品级、亲密度等） |
| `filterType` | 界面内部状态 | 当前筛选类型 |
| `sortBy` | 界面内部状态 | 当前排序字段 |
| `currentPage` | 界面内部状态 | 当前页码 |

### 6.2 界面触发的操作
| 操作 | 目标 |
|------|------|
| 点击已获得卡片 | 打开该人物的详情界面 |
| 切换筛选/排序 | 更新 `filterType`/`sortBy`，重新计算显示列表 |
| 翻页 | 更新 `currentPage` |

## 7. 对 AI 编程助手的生成提示

1. **数据过滤与排序**：
   - 使用 `useMemo` 或计算属性，根据 `filterType` 和 `sortBy` 动态生成显示列表。
   - 分页逻辑：`slice((currentPage-1)*pageSize, currentPage*pageSize)`。

2. **卡片组件**：
   - 封装 `<PersonCard>` 组件，接收 `person` 对象和 `isCollected` 布尔值，内部处理两种状态样式。
   - 立绘图片路径通过 `person.portrait` 获取，未获得时使用默认剪影图。

3. **收集进度计算**：
   - `progress = (collectedIds.length / allPersons.length) * 100`。
   - 进度条宽度绑定 `progress`，颜色可通过 CSS 类切换。

4. **古籍书页背景**：
   - 使用 `::before` 和 `::after` 伪元素绘制中缝（鱼尾、版心线）。
   - 可叠加极淡的纸张纹理图片。

5. **古风细节**：
   - 卡片底部阴影可模拟相册卡槽：`box-shadow: inset 0 -4px 0 rgba(0,0,0,0.05), var(--shadow-md)`。
   - 未获得卡片的「？」使用 `.stamp` 印章样式。

6. **响应式适配**：
   - 移动端（<640px）卡片宽度调整为 `calc(50% - 8px)`，每行 2 张。
   - 分类标签可横向滚动。
   - 进度条与统计信息可换行显示。

## 8. 图片资源清单

本界面所需的图片素材明细如下。

### 8.1 人物卡片立绘
| 资源 ID | 文件名（建议） | 类型 | 尺寸 | 格式 | 说明 |
|---------|---------------|------|------|------|------|
| `PORTRAIT_*` | 同人物详情界面 | 人物立绘 | 512×512px | PNG | 所有已有人物立绘，在图鉴卡片中缩放显示。 |
| `PORTRAIT_SILHOUETTE` | `portrait_silhouette_256x256.png` | 剪影占位 | 256×256px | PNG | 未获得人物的深色剪影，可带有「？」印章叠加。 |

### 8.2 装饰元素
| 资源 ID | 文件名（建议） | 类型 | 尺寸 | 格式 | 说明 |
|---------|---------------|------|------|------|------|
| `DECOR_BOOK_LINE` | `decor_book_line_1200x800.png` | 古籍版式线 | 1200×800px | PNG | 线装书对页的版式线条（鱼尾、象鼻等），作为背景叠加，透明背景。 |
| `STAMP_UNKNOWN` | `stamp_unknown_60x60.png` | 印章 | 60×60px | PNG | 未获得卡片中央的「？」印章，朱红底白字。 |

### 8.3 图标
| 资源 ID | 文件名（建议） | 类型 | 尺寸 | 格式 | 说明 |
|---------|---------------|------|------|------|------|
| `ICON_FILTER` | `icon_filter_16x16.svg` | 图标 | 16×16px | SVG | 筛选图标（可选）。 |
| `ICON_SORT` | `icon_sort_16x16.svg` | 图标 | 16×16px | SVG | 排序图标（可选）。 |

### 8.4 资源依赖关系
```
CollectionUI
├── PORTRAIT_* (必需，已获得人物)
├── PORTRAIT_SILHOUETTE (必需，未获得人物)
├── DECOR_BOOK_LINE (推荐，可降级为 CSS 伪元素)
├── STAMP_UNKNOWN (推荐，可降级为 CSS 印章)
└── ICON_FILTER / SORT (可选)
```

### 8.5 对 AI 的图片使用提示
- 立绘在卡片中使用 `object-fit: cover` 或 `contain`，确保不变形。
- 剪影图可与「？」印章合成为一张图，或分别叠加。
- 古籍版式线作为 `background-image`，设置 `opacity: 0.1`，`background-size: cover`。

## 9. 附录：太阁立志传4 图鉴系统参考要点

- 太阁4的「人物卡片」是游戏的核心收集要素之一，共有数百张。卡片收藏在「卡片夹」中，以相册形式展示。
- 未获得的卡片显示为黑色剪影，仅有轮廓，激发玩家收集欲。
- 卡片可按类型（人物、技能、称号等）分类查看。
- 本设计聚焦于**人物图鉴**，后续可扩展至技能图鉴、称号图鉴等。
