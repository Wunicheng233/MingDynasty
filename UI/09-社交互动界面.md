# 《洪武立志传》界面设计：社交互动界面

> 版本：1.0.0  
> 继承自：《洪武立志传》整体 UI 设计规范 v1.0.0  
> 对应模块：`SocialRenderer.js`

---

## 1. 界面定位与功能概述

### 1.1 界面位置
社交互动界面是玩家与 NPC 进行深度交流、提升亲密度、获取情报或触发事件的轻量交互面板。它通常在点击 NPC 并选择「交谈」「宴请」「赠与」等社交动作后，以**模态框**或**底部抽屉**形式呈现。

### 1.2 主要功能
- 展示当前互动的 NPC 基本信息（姓名、头像、亲密度）
- 提供多种社交动作选项（茶会、酒宴、赠礼、手谈、切磋等）
- 展示互动结果反馈（亲密度变化、获得物品、情报文本）
- 记录与该 NPC 的最近互动历史（可选）

### 1.3 触发条件
- 在城镇据点界面点击 NPC，并选择「交谈」或同类社交按钮
- 在大地图偶遇人物后自动触发
- 从人物详情界面点击「宴请」「赠与」等按钮

### 1.4 界面形式
本界面以**半屏模态框**为主，不遮挡过多背景，保持对话的沉浸感。特殊场景（如大型宴会）可扩展为全屏界面。

| 形式 | 触发场景 | 特点 |
|------|----------|------|
| **模态框（半屏）** | 日常交谈、赠礼、茶会 | 悬浮于当前界面之上，带有半透明遮罩，内容紧凑 |
| **全屏界面** | 重要宴会、大型社交事件 | 沉浸式背景，可展示多位 NPC |

> 本文档以**模态框**为主要描述对象。

## 3. 视觉继承与差异化说明

### 3.1 继承的全局令牌
| 元素 | 使用的令牌 |
|------|------------|
| 模态框背景 | `var(--color-bg-secondary)` |
| 模态框边框 | `var(--border-double)` |
| 模态框阴影 | `var(--shadow-xl)` |
| 文字 | `--font-serif` 为主 |
| 进度条 | 继承人物详情界面的亲密度进度条样式 |
| 按钮 | `.btn-primary`、`.btn-secondary`、`.btn-text` |
| 遮罩层 | `rgba(0,0,0,0.4)` + `backdrop-filter: blur(2px)` |

### 3.2 本界面的特殊处理
- **对话气泡**：NPC 的发言使用**古籍引文框**样式——左侧带有竖排的「曰」字印章，背景为牙色，边框淡墨。
- **选项卡片**：每个对话选项为一张微型卡片，悬停时边框高亮，并显示预期影响（如「亲密度 +5」）。
- **亲密度变化动画**：当亲密度数值变化时，进度条有短暂的发光或脉冲效果，数字跳动变化。
- **赠礼/宴请等子界面**：点击底部动作按钮后，可在当前模态框内展开二级面板（如下拉选择礼物），或弹出新的小模态框。

## 4. 详细组件与状态

### 4.1 NPC 信息区
- **头像**：尺寸 `48×48px` 或 `64×64px`，圆形或圆角 `var(--radius-sm)`，边框 `var(--border-default)`。
- **姓名与官职**：姓名使用 `--text-heading-sm`，官职使用 `--text-caption`，颜色 `var(--color-accent-gold)`。
- **亲密度**：进度条高度 `6px`，宽度 `120px`，背景 `var(--color-border-default)`，填充色 `var(--color-accent-primary)`。右侧显示数值。

### 4.2 对话/反馈区
- **容器**：`.dialogue-bubble`，背景 `var(--color-bg-secondary)`，边框 `var(--border-double)`，内边距 `var(--space-lg)`，圆角 `var(--radius-md)`。
- **装饰**：左侧使用 `::before` 伪元素添加「曰」字印章（竖排，朱红色）。
- **文本**：字体 `--font-serif`，字号 `--text-body`，行高 `1.6`，颜色 `var(--color-text-primary)`。
- **反馈文本**（如获得物品）：使用 `.feedback` 类，颜色 `var(--color-accent-green)` 或 `var(--color-accent-gold)`，带轻微入场动画。

### 4.3 选项区
- **选项卡片**：`.option-card`，内边距 `12px 16px`，背景 `var(--color-bg-secondary)`，边框 `var(--border-default)`，圆角 `var(--radius-sm)`，`margin-bottom: var(--space-sm)`。
- **悬停**：边框 `var(--border-highlight)`，背景 `rgba(158,42,43,0.03)`。
- **影响标签**：右侧显示预期亲密度变化，使用 `.badge` 样式（绿色 + ，红色 - ）。
- **不可用选项**：若某项因条件不足（如银两不够）不可选，置灰并显示提示。

### 4.4 底部动作栏
快捷社交动作按钮，横向排列，可触发赠礼、宴请、茶会、手谈（下棋）等。

| 动作 | 图标（建议） | 说明 |
|------|-------------|------|
| 赠礼 | 🎁 | 打开物品栏，选择礼物赠送，提升亲密度 |
| 宴请 | 🍷 | 消耗银两，宴请 NPC，大幅提升亲密度 |
| 茶会 | 🍵 | 消耗茶叶/银两，进行茶会小游戏 |
| 手谈 | ♟️ | 与 NPC 下棋，提升智力经验与亲密度 |
| 切磋 | ⚔️ | 进入个人战（切磋模式），提升武力经验 |

- **按钮样式**：使用 `.btn-text` 或 `.btn-secondary` 小尺寸变体，内边距 `6px 12px`。
- **展开二级面板**：点击「赠礼」时，在当前模态框下方展开物品网格（继承物品栏样式）。点击「宴请」则弹出确认框。

### 4.5 历史记录（可选）
在模态框底部或右侧，可设计一个折叠的「互动历史」区域，展示最近 3 条互动记录。
- 每条记录：「与徐达茶会，亲密度 +3」。
- 字体 `--text-caption`，颜色 `var(--color-text-tertiary)`。

## 5. 交互逻辑与动效

### 5.1 进入动画
- 遮罩层淡入（200ms）
- 模态框由上而下轻微滑入（`slide-down`，250ms）

### 5.2 对话推进
- 点击对话选项后，选项区淡出，反馈区更新文本，亲密度进度条更新（带动画）。
- 若对话有分支，选项区刷新为新选项。

### 5.3 社交动作执行
- 点击底部动作按钮（如「赠礼」），展开二级面板，同时其他按钮临时禁用。
- 选择礼物后，反馈区显示「赠送成功，亲密度 +X」，物品从背包扣除。
- 若亲密度达到阈值（如 100），可触发特殊对话或事件（如「徐达愿与您结为兄弟」）。

### 5.4 退出方式
- 点击关闭按钮或遮罩层，若处于对话中则弹出确认：「是否结束交谈？」
- 按下 `ESC` 键等同关闭。

## 6. 数据绑定说明

### 6.1 界面需要的数据
| 数据字段 | 类型 | 说明 |
|----------|------|------|
| `targetNPC` | object | 当前互动的 NPC 完整信息 |
| `relationship.favor` | number | 当前亲密度 |
| `dialogueState` | object | 当前对话节点 ID、文本、选项列表 |
| `inventory` | array | 玩家物品栏（用于赠礼） |
| `playerSilver` | number | 玩家银两（用于宴请） |

### 6.2 界面触发的操作
| 操作 | 目标 |
|------|------|
| 选择对话选项 | 推进对话，更新亲密度，可能触发事件 |
| 赠礼 | 扣除物品，提升亲密度 |
| 宴请 | 扣除银两，提升亲密度 |
| 茶会/手谈 | 触发小游戏，结束后根据结果更新亲密度与经验 |
| 切磋 | 进入个人战（切磋模式） |

## 7. 对 AI 编程助手的生成提示

1. **对话系统设计**：
   - 对话数据建议使用 JSON 树形结构，包含节点 ID、文本、选项及跳转逻辑。
   - 封装 `<DialogueBox>` 组件，接收 `node` 数据，渲染文本与选项。

2. **二级面板实现**：
   - 赠礼面板可复用物品栏的网格组件，通过 `v-if` 或条件渲染控制显隐。
   - 使用 `teleport` 或绝对定位将二级面板置于模态框内部。

3. **亲密度动画**：
   - 使用 CSS `transition` 实现进度条宽度变化。
   - 数字跳动可使用简单的 `requestAnimationFrame` 计数动画。

4. **古风细节**：
   - 对话气泡左侧的「曰」字印章可用 CSS 伪元素实现，或使用小型 PNG。
   - 选项卡片的影响标签使用 `.badge` 组件。
   - 反馈文本可添加淡入上浮动画。

5. **响应式适配**：
   - 移动端模态框宽度 90%，最大高度 80%，内部滚动。
   - 底部动作栏可换行显示。

## 9. 附录：太阁立志传4 社交系统参考要点

- 太阁4的社交主要通过「交谈」「赠与」「交手」等指令进行，互动后亲密度变化。
- 茶会、酒宴是提升亲密度的重要手段，有时需要消耗物品或金钱。
- 对话选项简洁明了，选项直接影响结果。
- 本设计在保留经典社交逻辑的基础上，将界面升级为更现代的卡片式交互，同时保留古风韵味。
