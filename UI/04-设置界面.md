# 《洪武立志传》界面设计：设置界面

> 版本：1.0.0  
> 继承自：《洪武立志传》整体 UI 设计规范 v1.0.0  
> 对应模块：`SettingsUI.js` 或模态框组件（待新建）

---

## 1. 界面定位与功能概述

### 1.1 界面位置
设置界面是玩家调整游戏参数的控制中心，可从启动界面（「游戏设置」）或游戏内菜单（「设置」）进入。

### 1.2 主要功能
- 调整音频设置（主音量、背景音乐、音效）
- 调整显示设置（画面质量、全屏/窗口模式、分辨率）
- 调整游戏性设置（文本速度、自动存档、战斗动画开关）
- 查看游戏版本与制作组信息
- 保存/取消/恢复默认设置

### 1.3 触发条件
- 启动界面点击「游戏设置」
- 游戏内菜单点击「设置」
- 游戏内按下 `ESC` 键（可配置）

### 1.4 界面形式
根据触发场景，本界面有两种呈现形式：
| 形式 | 触发场景 | 特点 |
|------|----------|------|
| **全屏界面** | 从启动界面进入 | 独立页面，有返回按钮，背景为宣纸底 + 水墨纹理 |
| **模态框** | 从游戏内菜单进入 | 悬浮于当前游戏画面之上，带半透明遮罩，尺寸较小，不遮挡过多游戏视野 |

> 本文档以**全屏界面**为主要描述对象，模态框版本可继承大部分组件样式，仅需调整容器尺寸与遮罩层。

## 3. 视觉继承与差异化说明

### 3.1 继承的全局令牌
| 元素 | 使用的令牌 |
|------|------------|
| 背景色 | `var(--color-bg-primary)` |
| 设置面板背景 | `var(--color-bg-secondary)` |
| 面板边框 | `var(--border-double)` |
| 面板阴影 | `var(--shadow-lg)` |
| 分组标题字体 | `--font-serif`，字号 `--text-heading-sm`，颜色 `var(--color-text-primary)` |
| 选项文字字体 | `--font-serif`，字号 `--text-body`，颜色 `var(--color-text-secondary)` |
| 按钮 | `.btn-primary`（保存）、`.btn-secondary`（取消/恢复默认） |
| 滑块/开关 | 自定义古风化组件（见下文） |

### 3.2 本界面的特殊处理
- **分组卡片**：每组设置项置于一个独立的半透明卡片中，带有双层边框（仿古籍栏线）。
- **滑块控件**：将原生 `<input type="range">` 重新设计为**墨滴滑块**：轨道为淡墨细线，滑块为一枚圆形墨迹或朱砂圆点。
- **分段选项**：使用仿古籍版刻风格的按钮组，选中项背景为朱砂红，未选中为牙色。
- **下拉选择**：分辨率选择使用自定义下拉，右侧箭头为向下展开的「▼」图标，菜单面板带宣纸底与阴影。
- **装饰元素**：面板四角可点缀小型云纹角花，提升精致感。

## 4. 详细组件与状态

### 4.1 分组卡片
- **容器**：`.settings-group`，背景 `var(--color-bg-secondary)`，边框 `var(--border-double)`，圆角 `var(--radius-md)`，内边距 `var(--space-xl)`，底部边距 `var(--space-2xl)`。
- **标题**：`.group-title`，字体 `--font-serif`，字号 `--text-heading-sm`，颜色 `var(--color-accent-primary)`，下方带有 `var(--border-default)` 分隔线，并可有小型印章装饰（如「音」「画」「戏」）。

### 4.2 滑块控件（Slider）
将原生滑块隐藏，使用自定义样式。

- **轨道**：`height: 2px`，背景 `var(--color-border-default)`，圆角 `1px`。
- **填充轨道**：`height: 2px`，背景 `var(--color-accent-primary)`，圆角 `1px`。
- **滑块（Thumb）**：`width: 16px`，`height: 16px`，背景 `var(--color-accent-primary)`，圆角 `50%`，边框 `1px solid #7A1F20`，阴影 `var(--shadow-sm)`。悬停时放大至 `18px`。
- **数值标签**（可选）：滑块右侧显示当前数值百分比（如「80%」），字体 `--text-body-sm`。

**实现提示**：使用 `<input type="range">` 配合 CSS 伪元素 `::-webkit-slider-thumb` 等实现。

### 4.3 分段选项（Segmented Control）
- **容器**：`.segmented`，`display: inline-flex`，边框 `var(--border-default)`，圆角 `var(--radius-sm)`，`overflow: hidden`。
- **选项按钮**：`.segmented-option`，内边距 `6px 16px`，背景 `var(--color-bg-secondary)`，文字颜色 `var(--color-text-secondary)`，字体 `--text-body-sm`。
- **选中态**：`.segmented-option.active`，背景 `var(--color-accent-primary)`，文字颜色 `var(--color-text-inverse)`。
- **分隔线**：选项之间用 `1px solid var(--color-border-default)` 分隔。

### 4.4 下拉选择（Dropdown）
- **触发器**：`.dropdown-trigger`，内边距 `8px 12px`，边框 `var(--border-default)`，圆角 `var(--radius-sm)`，背景 `var(--color-bg-secondary)`，文字 `var(--text-body)`。右侧显示向下箭头图标。
- **下拉菜单**：`.dropdown-menu`，背景 `var(--color-bg-secondary)`，边框 `var(--border-default)`，圆角 `var(--radius-sm)`，阴影 `var(--shadow-lg)`，内边距 `4px 0`。
- **菜单项**：`.dropdown-item`，内边距 `8px 16px`，文字 `var(--text-body)`。悬停时背景 `rgba(158,42,43,0.05)`，选中态文字变为 `var(--color-accent-primary)`。

### 4.5 开关控件（Toggle Switch）
用于「自动存档」等布尔选项。

- **容器**：`.toggle`，`width: 44px`，`height: 24px`，背景 `var(--color-border-default)`，圆角 `12px`，`padding: 2px`，`cursor: pointer`。
- **滑块**：`.toggle-thumb`，`width: 20px`，`height: 20px`，背景 `var(--color-bg-secondary)`，圆角 `50%`，阴影 `var(--shadow-sm)`，过渡 `var(--transition-fast)`。
- **开启态**：`.toggle.active`，背景 `var(--color-accent-primary)`，滑块右移 `translateX(20px)`。

### 4.6 底部操作按钮
| 按钮 | 类型 | 行为 |
|------|------|------|
| 恢复默认 | `.btn-secondary` | 将所有设置项重置为默认值，需二次确认 |
| 取消 | `.btn-secondary` | 放弃修改，返回上一界面 |
| 保存设置 | `.btn-primary` | 保存当前设置并应用，返回上一界面 |

### 4.7 状态说明
| 状态 | 表现 |
|------|------|
| 未修改 | 「保存设置」按钮可点击，但视觉上可稍作弱化（或保持正常） |
| 已修改 | 「保存设置」按钮正常高亮，「取消」按钮可用 |
| 保存中 | 「保存设置」按钮显示加载状态（墨滴旋转动画），禁止再次点击 |

## 5. 交互逻辑与动效

### 5.1 进入动画
- 全屏界面：整体淡入 + 轻微上浮（`fade-in-up`，300ms）
- 模态框：遮罩层淡入（200ms），内容面板由上而下轻微展开（`slide-down`，250ms）

### 5.2 控件交互反馈
- **滑块**：拖动时实时更新数值标签（若有），松开后触发设置变更事件（但不立即保存）。
- **分段选项**：点击后立即切换选中态，触发设置变更。
- **下拉选择**：点击触发器展开菜单，点击菜单项或点击外部关闭菜单。选中后触发器文字更新。
- **开关**：点击后切换开启/关闭态，伴随滑块滑动动画。

### 5.3 保存与取消
- **保存**：点击后应用所有设置（如音量、分辨率等），并写入本地存储/配置文件。若切换了全屏/窗口模式，需即时生效并可能伴随短暂黑屏过渡。
- **取消**：若设置已修改，点击「取消」时弹出确认框：「是否放弃修改？」。若无修改，直接返回。

### 5.4 恢复默认
- 点击「恢复默认」后弹出确认框：「恢复所有设置为默认值？」
- 确认后所有控件状态重置为默认值，但**不立即保存**，需用户再次点击「保存设置」。

### 5.5 退出方式
- 点击「返回」「关闭」按钮或按下 `ESC` 键，行为同「取消」按钮（检测修改状态）。

## 6. 数据绑定说明

### 6.1 界面需要的数据
| 数据字段 | 类型 | 默认值 | 说明 |
|----------|------|--------|------|
| `masterVolume` | number (0-100) | 80 | 主音量 |
| `bgmVolume` | number (0-100) | 70 | 背景音乐音量 |
| `sfxVolume` | number (0-100) | 90 | 音效音量 |
| `graphicsQuality` | string | 'high' | 画面质量：'low', 'medium', 'high', 'ultra' |
| `windowMode` | string | 'fullscreen' | 窗口模式：'windowed', 'fullscreen', 'borderless' |
| `resolution` | string | '1920x1080' | 当前分辨率 |
| `textSpeed` | number (1-5) | 3 | 文本显示速度，1 最慢，5 最快 |
| `autoSave` | boolean | true | 是否开启自动存档 |
| `battleAnim` | string | 'full' | 战斗动画：'full', 'simplified', 'off' |

### 6.2 界面触发的操作
| 操作 | 目标 |
|------|------|
| 修改任意控件 | 更新界面内部状态 `tempSettings`，标记 `isDirty = true` |
| 点击「保存设置」 | 将 `tempSettings` 写入全局 `gameSettings`，调用 `applySettings()` 应用更改，关闭界面 |
| 点击「恢复默认」 | 将 `tempSettings` 重置为 `defaultSettings`，刷新控件状态 |
| 点击「取消」 | 若 `isDirty` 为 true，弹窗确认；否则直接关闭界面 |

## 7. 对 AI 编程助手的生成提示

1. **组件化设计**：
   - 将滑块、分段选项、下拉、开关分别封装为独立组件（`<Slider>`、`<Segmented>`、`<Dropdown>`、`<Toggle>`）。
   - 这些组件应接收 `value` 与 `onChange` 回调，保持受控模式。

2. **自定义控件样式**：
   - 隐藏原生控件（如 `input[type=range]`），用 `<div>` 模拟，通过 JS 处理拖拽逻辑。
   - 或使用 CSS 伪元素美化原生控件（注意跨浏览器兼容性）。

3. **状态管理**：
   - 使用 `useState` 或 `ref` 维护临时设置对象 `tempSettings`。
   - 在组件卸载或关闭时，若 `isDirty` 为 true，应提示用户保存。

4. **设置持久化**：
   - 封装 `settingsManager` 模块，使用 `localStorage` 或 Tauri/Filesystem API 持久化设置。
   - 默认设置定义在常量文件中。

5. **古风细节**：
   - 为分组标题添加印章装饰（使用 `.stamp` 类或小型 PNG 图片）。
   - 滑块轨道可设计为细墨线，滑块为墨滴（圆形深色）。
   - 面板四角可添加 `::before`/`::after` 伪元素实现云纹角花。

6. **响应式适配**：
   - 移动端（<640px）时，分组卡片内边距减小至 `var(--space-lg)`。
   - 分段选项改为纵向堆叠或使用下拉替代。
   - 底部按钮改为纵向排列（保存在上，取消在下）。

7. **可访问性**：
   - 为所有交互控件添加 `aria-label`。
   - 滑块支持键盘左右键调整（步进 1 或 5）。
   - 焦点状态使用 `:focus-visible` 添加朱砂色外轮廓。
