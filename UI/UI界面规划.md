## 《洪武立志传》UI 界面设计文档 · 完整规划

### 规划原则
1. **全覆盖**：从启动到退出，玩家可能接触的每个界面都有对应的设计文档。
2. **分层级**：按界面在游戏流程中的位置，分为 **启动层 → 核心玩法层 → 子系统层 → 系统工具层**。
3. **优先级标注**：🔥🔥🔥 高（核心体验，优先完成） / 🔥🔥 中（重要辅助） / 🔥 低（完善体验，可后置）。
4. **文档格式统一**：每个界面设计文档遵循固定结构（下文有示例说明）。

### 📄 每个界面设计文档的固定结构示例

为保持一致性，每个 `界面设计-XXX.md` 将包含以下章节（类似你之前喜欢的 Stripe 格式，但针对游戏界面定制）：

```markdown
# 《洪武立志传》界面设计：XXX 界面

> 版本：1.0.0
> 继承自：《洪武立志传》整体 UI 设计规范 v1.0.0
> 对应模块：`xxx.js`

## 1. 界面定位与功能概述
- 界面在游戏流程中的位置
- 主要功能描述
- 触发条件 / 入口

## 2. 布局结构图
- 使用 ASCII 或文字描述区域划分（Header / Body / Footer / Sidebar 等）

## 3. 视觉继承与差异化说明
- 引用整体设计规范中的令牌
- 如有特殊处理（如深色背景），在此说明

## 4. 详细组件与状态
- 4.1 核心卡片/列表项样式
- 4.2 按钮与交互元素
- 4.3 动态数据展示区域
- 4.4 各种状态（加载中、空状态、错误状态）

## 5. 交互逻辑与动效
- 点击、悬停、拖拽等行为
- 过渡动画描述

## 6. 数据绑定说明
- 该界面需要从游戏状态获取哪些数据
- 哪些操作会修改数据

## 7. 对 AI 编程助手的生成提示
- 具体的组件实现建议
- 需要特别注意的古风细节
```
