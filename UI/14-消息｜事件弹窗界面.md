# 《洪武立志传》界面设计：消息/事件弹窗

> 版本：1.0.0  
> 继承自：《洪武立志传》整体 UI 设计规范 v1.0.0  
> 对应模块：`MessageRenderer.js` 或 `EventModal.js`（待新建）

---

## 1. 界面定位与功能概述

### 1.1 界面位置
消息与事件弹窗是游戏内传递信息、推动剧情、提供交互选择的核心轻量界面。它贯穿于整个游戏流程，用于展示人物对话、随机遭遇、系统提示、任务更新等。

### 1.2 主要功能
- 展示单条或多条信息（系统提示、人物对白、事件描述）
- 提供简单的交互选项（「继续」「确认」「取消」「选择」等）
- 展示事件插图或人物头像，增强沉浸感
- 自动消失的轻提示（Toast）或需手动关闭的模态框

### 1.3 触发条件
- 大地图移动时随机遭遇（遇盗匪、访名士、天灾）
- 与 NPC 对话推进剧情
- 系统操作反馈（保存成功、设置已应用、错误提示）
- 任务完成/失败通知

### 1.4 界面形式
根据信息重要程度与交互需求，分为三种形式：

| 形式 | 特点 | 应用场景 |
|------|------|----------|
| **轻提示（Toast）** | 自动消失，无交互，2-3秒后淡出 | 保存成功、获得物品、简单系统反馈 |
| **标准事件弹窗** | 半透明模态框，带人物头像/插图，需点击继续 | 人物对话、剧情推进、随机遭遇 |
| **选择事件弹窗** | 在标准弹窗基础上增加选项按钮 | 关键抉择、任务接受、风险决策 |

## 3. 视觉继承与差异化说明

### 3.1 继承的全局令牌
| 元素 | 使用的令牌 |
|------|------------|
| 弹窗背景 | `var(--color-bg-secondary)` |
| 弹窗边框 | `var(--border-double)` |
| 弹窗阴影 | `var(--shadow-xl)` |
| 文字 | `--font-serif` |
| 按钮 | `.btn-primary`、`.btn-secondary` |
| 遮罩层 | `rgba(0,0,0,0.4)` + `backdrop-filter: blur(2px)` |

### 3.2 本界面的特殊处理
- **人物头像**：若说话者为已知人物，显示其头像（64×64px），并配以姓名标签。无名 NPC 可使用通用剪影。
- **事件插图**：重要剧情事件可替换头像区域为全幅插图（如战场远景、灾荒场景），增强叙事。
- **文本逐字打印**：可选择开启「逐字显示」效果，模拟老式 RPG 对话感，配合文本速度设置。
- **选项高亮**：关键选项（如「迎战」）使用朱砂色按钮，「撤退」使用赭石色。

## 4. 详细组件与状态

### 4.1 轻提示（Toast）
- **容器**：`.toast`，固定定位，居中偏下，背景 `var(--color-bg-dark)`，文字 `var(--color-text-inverse)`，内边距 `8px 20px`，圆角 `var(--radius-sm)`。
- **图标**：可选前缀图标（✓ 成功、⚠ 警告、ℹ 信息）。
- **动画**：由下而上淡入，停留 2.5 秒后淡出。
- **堆叠**：多条 Toast 垂直排列，新消息在上方。

### 4.2 标准事件弹窗
- **容器**：`.event-modal`，宽度 500px（桌面端）/ 90%（移动端），内边距 `20px`。
- **说话人区域**：
  - 头像：`width: 64px`，`height: 64px`，圆形或圆角，边框 `var(--border-default)`。
  - 姓名：置于头像下方或右侧，字体 `--text-heading-sm`，颜色 `var(--color-text-primary)`。
- **文本气泡**：
  - 背景 `var(--color-bg-secondary)`，边框 `var(--border-double)`，内边距 `16px`，圆角 `var(--radius-md)`。
  - 左侧装饰：「曰」字印章（竖排，朱红）。
  - 字体 `--font-serif`，字号 `--text-body`，行高 `1.6`。
  - 最大高度 `200px`，`overflow-y: auto`。
- **继续按钮**：「继续」→ `.btn-primary`，点击后推进到下一条对话或关闭。

### 4.3 选择事件弹窗
在标准弹窗基础上，按钮区替换为选项列表。

- **选项按钮**：每个选项为 `.option-btn`，继承 `.btn-secondary` 样式，可附加影响标签（如「亲密度 +5」「银两 -50」）。
- **选项影响预览**：在按钮右侧显示预期结果，使用 `.badge` 组件。
- **不可选选项**：条件不足时置灰，并显示提示（如「银两不足」）。

### 4.4 事件插图模式
- 当事件具有强叙事性（如历史战役、人物登场），弹窗可切换为「插图模式」。
- **插图**：宽度 100%，高度 200px，`object-fit: cover`，圆角 `var(--radius-sm)`。
- **标题**：插图下方显示事件标题（如「鄱阳湖之战」），字体 `--font-calligraphy`，颜色 `var(--color-accent-primary)`。
- **描述文本**：位于插图下方或右侧。

### 4.5 状态与配置
| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `showAvatar` | 是否显示头像/插图 | true |
| `speakerName` | 说话人姓名 | 空（不显示） |
| `text` | 对话/事件文本 | 必填 |
| `options` | 选项列表（若为空则显示继续按钮） | [] |
| `closable` | 是否可点击遮罩关闭 | true |
| `typewriter` | 是否启用逐字打印 | false（可在设置中全局控制） |

## 5. 交互逻辑与动效

### 5.1 进入动画
- 遮罩层淡入（200ms）
- 弹窗由小变大 + 淡入（`scale-fade`，250ms）

### 5.2 对话推进
- 点击「继续」或按下空格键/回车，切换到下一条对话。
- 若为最后一条，关闭弹窗。
- 逐字打印模式下，点击「继续」可跳过打印直接显示全文。

### 5.3 选项交互
- 点击选项按钮后，按钮进入加载状态（避免重复点击），执行对应逻辑（扣资源、触发战斗等）。
- 选择后弹窗关闭或进入下一段对话。

### 5.4 关闭方式
- 点击「继续」（无选项时）
- 点击遮罩层（若 `closable` 为 true）
- 按下 `ESC` 键

## 6. 数据绑定说明

### 6.1 界面需要的数据
| 数据字段 | 类型 | 说明 |
|----------|------|------|
| `eventData` | object | 事件完整数据，包含说话人、文本序列、选项、插图等 |
| `currentIndex` | number | 当前显示的文本索引（用于多段对话） |
| `typewriterEnabled` | boolean | 是否启用逐字打印（从全局设置读取） |

### 6.2 界面触发的操作
| 操作 | 目标 |
|------|------|
| 点击「继续」 | 推进到下一段文本或关闭 |
| 点击选项 | 执行选项关联的游戏逻辑（如 `onSelect(optionId)`） |
| 关闭弹窗 | 触发 `onClose` 回调，恢复游戏控制 |

## 7. 对 AI 编程助手的生成提示

1. **事件队列管理**：封装 `EventManager`，支持顺序触发多个事件弹窗，避免重叠。
2. **逐字打印实现**：使用 `setInterval` 或 `requestAnimationFrame` 逐字更新文本，并提供跳过功能。
3. **组件复用**：将标准弹窗封装为 `<EventModal>`，接收 `eventData` 和回调。
4. **Toast 系统**：使用发布订阅模式，全局调用 `toast.success('保存成功')`。
5. **古风细节**：文本气泡左侧印章可用 CSS 伪元素实现；选项按钮的影响标签使用 `.badge`。
6. **响应式**：移动端弹窗宽度 90%，文本字号适当放大以保证可读。

## 9. 附录：太阁立志传4 消息事件参考要点

- 太阁4的对话以底部窗口形式呈现，左侧显示说话人头像，右侧为文本。
- 重要事件会有全屏插画与文字描述。
- 选项简洁明了，直接影响后续发展。
- 本设计将对话升级为居中弹窗，保留头像与文本气泡，更适合现代 UI 习惯，同时通过插图模式保留重要事件的叙事张力。
