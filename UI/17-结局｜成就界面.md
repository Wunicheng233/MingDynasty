# 《洪武立志传》界面设计：结局/成就界面

> 版本：1.0.0  
> 继承自：《洪武立志传》整体 UI 设计规范 v1.0.0  
> 对应模块：`EndingRenderer.js`（待新建）

---

## 1. 界面定位与功能概述

### 1.1 界面位置
结局/成就界面是玩家完成一个游戏周目后，或从主菜单查看成就进度时呈现的总结与荣誉展示面板。它借鉴《太阁立志传4》的多结局画廊设计，将玩家达成的结局、获得的称号、收集的成就以**画卷**或**勋章墙**形式呈现，给予玩家强烈的正反馈与重复游玩的动力。

### 1.2 主要功能
- 展示玩家达成的游戏结局（画面 + 标题 + 描述）
- 展示玩家获得的称号与成就（已解锁/未解锁）
- 显示整体完成度百分比
- 提供「返回标题」「继续旅程」（若未结束）等操作

### 1.3 触发条件
- 游戏通关后自动进入结局界面
- 在主菜单选择「成就/结局」
- 查看游戏统计时

### 1.4 界面形式
| 形式 | 触发场景 | 特点 |
|------|----------|------|
| **结局画廊视图** | 通关后 | 全屏沉浸式，展示达成结局的大幅插画与描述 |
| **成就勋章墙** | 主菜单进入 | 网格展示所有成就，区分已解锁/未解锁 |

> 本文档以**结局画廊视图**为主要描述对象，成就勋章墙可视为其简化/扩展版本。

## 3. 视觉继承与差异化说明

### 3.1 继承的全局令牌
| 元素 | 使用的令牌 |
|------|------------|
| 描述框背景 | `var(--color-bg-secondary)` + 半透明 |
| 描述框边框 | `var(--border-double)` |
| 字体 | `--font-serif` 为主，标题可用 `--font-calligraphy` |
| 按钮 | `.btn-primary`（继续旅程）、`.btn-secondary`（返回标题、查看成就） |
| 收集进度条 | 继承图鉴界面的进度条样式 |

### 3.2 本界面的特殊处理
- **结局插画**：每个结局有一幅专属水墨插画，营造史诗感或沧桑感。
- **结局文本排版**：模仿史书列传，可竖排或横排，首字下沉，行距疏朗。
- **称号展示**：称号以**烫金印章**形式呈现，置于摘要区。
- **收集进度**：显示结局收集数量与成就数量，激发玩家探索其他结局。

## 4. 详细组件与状态

### 4.1 结局插画区
- **容器**：`.ending-illustration`，宽度 100%，最大高度 400px，`overflow: hidden`，`border-radius: var(--radius-lg)`，边框 `var(--border-gold)`（辉煌结局）或 `var(--border-default)`。
- **图片**：`object-fit: cover`，居中。
- **标题**：置于图片下方或叠加于图片底部，字体 `--font-calligraphy`，字号 `--text-display`，颜色金色或白色，带文字阴影。

### 4.2 结局描述区
- **容器**：`.ending-text`，内边距 `24px`，背景半透明宣纸，边框双层，圆角 `var(--radius-md)`。
- **文本**：字体 `--font-serif`，字号 `--text-body`，行高 `1.8`，颜色 `var(--color-text-primary)`。可使用 `::first-letter` 首字下沉并染朱砂色。
- **装饰**：四周可点缀云纹或回纹边框。

### 4.3 摘要统计区
- **容器**：`.ending-summary`，内边距 `12px 20px`，`display: flex`，`justify-content: space-around`。
- **达成结局**：标签「达成结局」+ 结局名，字体 `--text-heading-sm`。
- **获得称号**：印章式展示（如「中山王」），背景古铜金，文字墨色。
- **收集进度**：🏆 图标 + `已解锁成就 / 总成就`，📜 图标 + `已达成结局 / 总结局`。

### 4.4 成就勋章墙（子界面）
点击「查看成就」后，可切换到勋章墙视图。

- **网格布局**：每行 4-6 个勋章，每个勋章为圆形或方形卡片。
- **已解锁勋章**：显示彩色图标 + 名称，带发光效果。
- **未解锁勋章**：显示灰色剪影 + 「???」，悬停显示解锁条件。
- **成就分类**：可分类（征战、内政、收集、社交、结局）。

### 4.5 底部操作栏
| 按钮 | 样式 | 行为 |
|------|------|------|
| 查看成就 | `.btn-secondary` | 切换到成就勋章墙视图 |
| 返回标题 | `.btn-secondary` | 返回启动界面 |
| 继续旅程 | `.btn-primary` | 若游戏未结束（如达成结局后仍可继续游玩），返回游戏；否则仅显示「返回标题」 |

### 4.6 结局列表（画廊模式）
在主菜单的「结局」入口，可展示所有已达成结局的缩略图列表，点击查看详情。未达成结局显示「???」占位。

## 5. 交互逻辑与动效

### 5.1 进入动画
- 背景由黑场淡入（1000ms，慢速营造仪式感）。
- 结局插画由模糊到清晰（`filter: blur(0)` 过渡）。
- 描述文本逐字打印或淡入上浮（`fade-in-up`，延迟 500ms）。

### 5.2 切换结局/成就
- 在画廊模式下，左右箭头可切换已解锁的结局。
- 成就勋章悬停有放大与发光效果。

### 5.3 返回标题/继续
- 点击按钮后，播放淡出动画，跳转至对应界面。

## 7. 对 AI 编程助手的生成提示

1. **结局数据配置**：使用 JSON 配置文件存储所有结局的数据，便于扩展。
2. **成就系统联动**：成就解锁可调用全局成就管理器，结局界面读取已解锁列表。
3. **动画序列**：使用 `async/await` 或 Promise 控制入场动画顺序。
4. **画廊模式**：封装 `<EndingGallery>` 组件，支持左右切换。
5. **古风细节**：结局文本建议使用文言文风格，可请文案同学撰写；称号印章使用 `.stamp` 类。
6. **响应式**：移动端插画高度减小，文本字号适应。

## 9. 附录：太阁立志传4 结局系统参考要点

- 太阁4拥有丰富的多结局，根据玩家职业、功勋、选择决定。
- 结局画面以静态插画 + 文字描述呈现，并展示获得的称号。
- 结局收集是游戏的重要重复游玩动力。
- 本设计在保留结局画廊的基础上，增加了成就勋章墙，进一步丰富收集要素。
