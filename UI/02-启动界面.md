# 《洪武立志传》界面设计：启动/登录界面

> 版本：1.0.0  
> 继承自：《洪武立志传》整体 UI 设计规范 v1.0.0  
> 对应模块：`StartScreen.js`（待新建）

---

## 1. 界面定位与功能概述

### 1.1 界面位置
玩家启动游戏后看到的第一个交互界面，位于所有游戏逻辑之前。

### 1.2 主要功能
- 展示游戏标题与品牌视觉
- 提供进入游戏的入口：「开启新档」「继续旧档」
- 提供系统配置入口：「游戏设置」
- （可选）展示制作组信息与版本号

### 1.3 触发条件
- 游戏首次加载完成
- 从游戏内「返回标题界面」菜单选项跳转而来

### 1.4 退出方式
- 点击「开启新档」→ 进入角色创建或序章
- 点击「继续旧档」→ 进入存档/读档界面
- 点击「游戏设置」→ 弹出设置模态框
- （桌面端）按下 `ESC` 键退出游戏（需二次确认）

## 3. 视觉继承与差异化说明

### 3.1 继承的全局令牌
| 元素 | 使用的令牌 |
|------|------------|
| 背景色 | `var(--color-bg-primary)` |
| 主按钮 | `.btn-primary`（朱砂红） |
| 字体 | 标题使用 `--font-calligraphy`，正文使用 `--font-serif` |
| 阴影 | 按钮悬停时使用 `var(--shadow-md)` |
| 过渡 | `var(--transition-base)` |

### 3.2 本界面的特殊处理
- **背景层**：除纯色宣纸底外，叠加一幅低透明度的**水墨江山图**（或皇城远眺图），营造「指点江山」的史诗感。
- **画轴装饰**：三个按钮放置在一个模拟**展开的卷轴**中央。卷轴可用 CSS 伪元素实现两侧深色边缘，或直接使用带透明度的 PNG 素材。
- **Logo 印章**：在标题文字右下方，添加一枚朱砂红印章，内容为「敕造」或「御览」，使用 `.stamp` 样式并旋转 -5 度。
- **入场动画**：界面加载时，标题和卷轴区域由透明渐显，同时伴随轻微的由上而下展开效果（`clip-path` 动画）。

## 4. 详细组件与状态

### 4.1 游戏 Logo
- **文字**：「洪武立志传」（可竖排或横排）
- **字体**：`--font-calligraphy`，字号 48px（移动端 36px）
- **颜色**：`var(--color-text-primary)`
- **装饰**：右侧/下方附带 `.stamp` 印章
- **入场动画**：`fade-in-down`，时长 `var(--transition-slow)`

### 4.2 主操作按钮（画轴内的三个按钮）
采用 `.btn-primary` 的变体，统一宽度 240px（移动端 200px），纵向间距 `var(--space-md)`。

| 按钮 | 文字 | 类型 | 交互行为 |
|------|------|------|----------|
| 开启新档 | 「开启新档」 | 主要按钮（朱砂） | 点击后进入角色创建或序章（具体跳转逻辑待定） |
| 继续旧档 | 「继续旧档」 | 次要按钮（墨色描边） | 点击后进入存档/读档界面 |
| 游戏设置 | 「游戏设置」 | 文字按钮变体（黛蓝色） | 点击后弹出设置模态框 |

**按钮样式细节**：
- 「开启新档」：使用 `.btn-primary`，字号 `--text-button`，内边距 `12px 32px`。
- 「继续旧档」：使用 `.btn-secondary`，字号 `--text-button`，内边距 `12px 32px`。
- 「游戏设置」：使用 `.btn-text`，字号 `--text-body`，颜色 `var(--color-accent-blue)`，悬停下划线。

### 4.3 卷轴装饰框
- 位于三个按钮外围，是一个**半透明的牙色圆角矩形**，带有内阴影和双层边框。
- 背景：`rgba(253,251,247,0.7)` + 轻微 `backdrop-filter: blur(4px)`
- 边框：`var(--border-double)`
- 圆角：`var(--radius-lg)`
- 内边距：`24px 40px`（移动端 `16px 24px`）
- 阴影：`var(--shadow-lg)`

### 4.4 底部信息
- 版本号：右下角，字体 `--font-mono`，字号 `--text-caption`，颜色 `var(--color-text-tertiary)`
- 制作组：左下角，一枚横向小印章，内容为「XX 工坊」或制作组名称，使用 `.stamp` 样式，颜色可设为古铜金。

### 4.5 状态说明
| 状态 | 表现 |
|------|------|
| 正常 | 按钮可点击，悬停有反馈 |
| 存档为空 | 「继续旧档」按钮置灰（透明度 0.5），不可点击，并显示提示「暂无存档」 |
| 加载中 | 点击「开启新档」或「继续旧档」后，按钮显示加载状态（可用旋转的墨滴图标代替文字） |

## 5. 交互逻辑与动效

### 5.1 入场动画序列
1. 背景水墨图淡入（`opacity 0 → 1`，时长 800ms）
2. 标题文字与印章由上而下淡入（`fade-in-down`，延迟 200ms，时长 500ms）
3. 卷轴区域由透明渐显，同时高度由 0 展开至完整高度（`clip-path` 或 `max-height` 动画，延迟 400ms，时长 600ms）
4. 底部信息淡入（延迟 800ms）

### 5.2 按钮交互
- **悬停**：主按钮阴影加深（`var(--shadow-md)`），次要按钮背景色轻微变化。
- **点击**：按钮有轻微下沉效果（`transform: scale(0.98)`）。
- **跳转过渡**：点击按钮后，整个界面淡出（`fade-out`，时长 300ms），随后加载下一界面。

### 5.3 键盘交互（桌面端）
- `↑` `↓` 键：在三个按钮间切换焦点（高亮边框）。
- `Enter` 键：触发当前聚焦按钮。
- `ESC` 键：弹出「是否退出游戏」确认框。

## 7. 对 AI 编程助手的生成提示

当基于本文档生成 `StartScreen.js` 组件时，请注意以下事项：

1. **优先使用全局 CSS 变量**：颜色、间距、字体等一律使用 `var(--token-name)` 形式。
2. **组件化实现**：
   - 将「卷轴按钮组」封装为独立组件 `<ScrollButtons />`。
   - 将「设置模态框」复用已有的全局设置组件（若存在）。
3. **动画实现**：
   - 使用 CSS `@keyframes` 定义 `fade-in-down` 和 `fade-out`。
   - 入场动画序列通过为不同元素添加递增的 `animation-delay` 实现。
4. **古风细节**：
   - 确保 `.stamp` 印章类已正确定义（背景朱砂红、白色书法字、旋转 -5deg）。
   - 背景图请使用一张低透明度（`opacity: 0.15`）的水墨 PNG，置于 `::before` 伪元素中。
5. **响应式处理**：
   - 移动端（<640px）时，卷轴内边距减小，按钮宽度自适应。
   - 标题字号从 48px 降至 36px。
6. **可访问性**：
   - 为按钮添加 `aria-label` 属性。
   - 确保键盘焦点样式明显（使用 `:focus-visible` 添加朱砂色外轮廓）。

## 9. 图片资源清单

本界面所需的图片素材明细如下。所有素材应遵循《整体 UI 设计规范》第11章的定义。

### 9.1 背景层
| 资源 ID | 文件名（建议） | 类型 | 尺寸 | 格式 | 说明 |
|---------|---------------|------|------|------|------|
| `BG_START_MAIN` | `bg_start_ink_mountain_1920x1080.webp` | 全屏背景 | 1920×1080px | WebP / JPG | 低透明度水墨江山图，叠加于宣纸底色之上。要求构图疏朗，留白较多，以突出中央的标题与按钮。 |

**使用方式**：
- 在 CSS 中作为背景图叠加：
  ```css
  .start-screen {
    background-color: var(--color-bg-primary);
    background-image: var(--bg-start-main);
    background-blend-mode: overlay;
    background-size: cover;
    background-position: center;
  }
  ```
- 若 `background-blend-mode` 兼容性不足，可将图片本身处理为低透明度（约 15%-20%）的 PNG，直接使用。

### 9.2 游戏标题与装饰
| 资源 ID | 文件名（建议） | 类型 | 尺寸 | 格式 | 说明 |
|---------|---------------|------|------|------|------|
| `TITLE_LOGO` | `title_hongwu_ligend_800x200.png` | 标题美术字 | 800×200px | PNG | 游戏标题「洪武立志传」的书法美术字，需经过设计处理（如添加墨迹飞白效果）。背景透明。 |
| `STAMP_IMPERIAL` | `stamp_chizao_80x80.png` | 印章 | 80×80px | PNG | 朱红底白字印章，内容为「敕造」或「御览」。背景透明。若不想使用图片，也可用 CSS 实现（参考 `.stamp` 类）。 |

**使用方式**：
- 标题美术字建议使用图片而非 Web 字体，以确保在所有设备上显示效果一致。
- 印章可置于标题右下角，`transform: rotate(-5deg)`。

### 9.3 卷轴装饰（可选）
| 资源 ID | 文件名（建议） | 类型 | 尺寸 | 格式 | 说明 |
|---------|---------------|------|------|------|------|
| `DECOR_SCROLL_VERTICAL` | `decor_scroll_vertical_600x400.png` | UI 装饰 | 宽度 600px，高度按比例 | PNG | 古画卷轴的边缘装饰（两侧深色轴杆），用于包裹三个主按钮。若不想使用图片，也可用 CSS 伪元素 + 渐变模拟。 |

**使用方式**：
- 作为 `.scroll-container` 的 `border-image` 或 `::before`/`::after` 背景图。
- 纯 CSS 替代方案：
  ```css
  .scroll-container {
    border-left: 8px solid #8B6F5C;
    border-right: 8px solid #8B6F5C;
    border-image: repeating-linear-gradient(...) 30;
  }
  ```

### 9.4 制作组印章（可选）
| 资源 ID | 文件名（建议） | 类型 | 尺寸 | 格式 | 说明 |
|---------|---------------|------|------|------|------|
| `STAMP_STUDIO` | `stamp_studio_name_120x40.png` | 印章 | 120×40px | PNG | 制作组名称的横向印章，可使用古铜金色调，背景透明。 |

**使用方式**：
- 置于界面左下角，不透明度 0.6。


### 9.6 对 AI 编程助手的图片使用提示
- 在 `StartScreen.js` 中，图片路径应通过环境变量或配置文件引入，避免硬编码。
- 当图片缺失时，应有降级样式（如纯色背景、CSS 绘制的印章），确保界面基本可用。
- 为 `<img>` 标签或背景图容器添加 `alt` 或 `aria-label` 属性，提升可访问性。
