# 《洪武立志传》界面设计：人物详情界面

> 版本：1.0.0  
> 继承自：《洪武立志传》整体 UI 设计规范 v1.0.0  
> 对应模块：`PersonRenderer.js`

---

## 1. 界面定位与功能概述

### 1.1 界面位置
人物详情界面是玩家查看任何人物（包括自己）详细信息的核心面板。它可能以**全屏界面**（查看主角或重要角色）或**模态框/侧边抽屉**（城镇内点击 NPC）的形式出现。

### 1.2 主要功能
- 展示人物头像/立绘、姓名、字号、品级、官位
- 以数值或雷达图展示五维属性（统率、武力、智力、政治、魅力）
- 展示人物列传（背景故事）
- 展示持有技能、物品、与玩家的关系（亲密度）
- 提供与该人物的交互操作按钮：「交谈」「赠与」「招募」「交手」「宴请」等

### 1.3 触发条件
- 在城镇据点界面点击 NPC
- 在人物图鉴中点击人物卡片
- 在大地图偶遇人物
- 查看主角自身状态（通过自宅或菜单）

### 1.4 界面形式
| 形式 | 触发场景 | 特点 |
|------|----------|------|
| **全屏详情页** | 查看主角、重要剧情人物、从图鉴进入 | 沉浸感强，信息完整，有返回按钮 |
| **侧边抽屉/模态框** | 城镇内快速查看 NPC | 轻量，不遮挡过多场景，可快速关闭 |

> 本文档以**全屏详情页**为主要描述对象，抽屉/模态框版本可复用大部分组件，仅需调整容器尺寸与布局密度。

## 3. 视觉继承与差异化说明

### 3.1 继承的全局令牌
| 元素 | 使用的令牌 |
|------|------------|
| 背景色 | `var(--color-bg-primary)` |
| 信息面板背景 | `var(--color-bg-secondary)` |
| 面板边框 | `var(--border-default)` |
| 标题字体 | `--font-serif`，字号 `--text-heading-md` |
| 正文字体 | `--font-serif`，字号 `--text-body` |
| 进度条填充色 | `var(--color-accent-primary)` |
| 按钮 | `.btn-primary`、`.btn-secondary` |

### 3.2 本界面的特殊处理
- **立绘展示**：人物立绘采用工笔重彩或水墨风格，背景透明，置于左侧，周围可有淡墨晕染装饰。
- **亲密度可视化**：使用分段式心形图标或进度条，颜色为朱砂红渐变。
- **属性展示**：可选择**横向进度条**（简洁）或**五维雷达图**（更直观），雷达图使用墨色线条与半透明填充。
- **列传区域**：模仿古籍中的小传排版，可带有首字下沉、竖排标点等古风细节。
- **技能标签**：使用 `.badge` 古铜金或黛蓝色变体。

## 4. 详细组件与状态

### 4.1 人物立绘
- **容器**：`.portrait-container`，内边距 `var(--space-lg)`，居中显示。
- **图片**：`max-width: 100%`，`max-height: 400px`，`object-fit: contain`。
- **装饰**：可在立绘后方添加圆形淡墨光晕（`radial-gradient`）。

### 4.2 基本信息区
- **姓名**：字体 `--font-calligraphy` 或 `--font-serif` 加粗，字号 `--text-heading-lg`，颜色 `var(--color-text-primary)`。
- **字号**（字）：置于姓名右侧，字号 `--text-body`，颜色 `var(--color-accent-blue)`。
- **品级/官职**：使用 `.badge` 朱砂或古铜变体。
- **亲密度**：
  - 标签：「亲密度」，字体 `--text-body-sm`。
  - 进度条：`height: 8px`，背景 `var(--color-border-default)`，圆角 `4px`。填充色 `var(--color-accent-primary)`。右侧显示数值（如 `85/100`）。

### 4.3 属性能力区
采用**五维属性**（统、武、智、政、魅）。

**方案 A：横向进度条**
| 属性 | 颜色建议 | 范围 |
|------|----------|------|
| 统率 | 黛蓝 `var(--color-accent-blue)` | 0-100 |
| 武力 | 朱砂 `var(--color-accent-primary)` | 0-100 |
| 智力 | 青碧 `var(--color-accent-green)` | 0-100 |
| 政治 | 古铜金 `var(--color-accent-gold)` | 0-100 |
| 魅力 | 赭石 `var(--color-accent-orange)` | 0-100 |

- 每行：属性名（如「统率」）+ 进度条 + 数值。
- 进度条高度 `6px`，圆角 `3px`。

**方案 B：雷达图**
- 使用 `<canvas>` 或 SVG 绘制五边形雷达图。
- 坐标轴：墨色细线（`var(--color-text-secondary)`）。
- 数据填充区：半透明朱砂色（`rgba(158,42,43,0.2)`），边框朱砂色。
- 雷达图尺寸：200×200px。

> 建议全屏详情页使用雷达图，抽屉/模态框使用进度条以节省空间。

### 4.4 列传区
- **容器**：`.biography`，背景 `var(--color-bg-secondary)`，边框 `var(--border-double)`，内边距 `var(--space-lg)`，圆角 `var(--radius-md)`。
- **标题**：「列传」，字体 `--font-serif`，字号 `--text-heading-sm`，颜色 `var(--color-accent-primary)`，右侧可有小印章。
- **正文**：字体 `--font-serif`，字号 `--text-body`，行高 `1.6`，颜色 `var(--color-text-primary)`。首字下沉（`::first-letter`，字号 3em，颜色 `var(--color-accent-primary)`）。
- **滚动**：最大高度 `120px`，`overflow-y: auto`。

### 4.5 技能/物品区
- **技能标签**：使用 `.badge` 组件，背景 `rgba(58,82,107,0.15)`，文字 `var(--color-accent-blue)`，边框 `1px solid rgba(58,82,107,0.3)`。
- **物品图标**：小尺寸图标 + 物品名，横向排列。

### 4.6 底部操作按钮
根据玩家与目标人物的关系、身份、场景，动态显示不同按钮。

| 按钮 | 显示条件 | 行为 |
|------|----------|------|
| 交谈 | 几乎所有人物 | 触发对话，可能提升亲密度或获取情报 |
| 赠与 | 可互动人物 | 打开物品栏，选择礼物赠送 |
| 招募 | 未仕官或可叛变的人物 | 尝试招募为部下，需满足条件（亲密度、相性等） |
| 交手 | 武力型人物 | 触发个人战 |
| 宴请 | 可互动人物 | 消耗银两，提升亲密度 |
| 学习 | 特定技能导师 | 学习技能，触发小游戏 |
| 挑战 | 敌对人物 | 触发个人战或舌战 |

- **按钮样式**：使用 `.btn-secondary` 或 `.btn-primary`（高亮推荐操作），间距 `var(--space-sm)`。
- **数量较多时**：可折叠为「更多」下拉菜单，或横向滚动。

## 5. 交互逻辑与动效

### 5.1 进入动画
- 全屏界面：由右向左滑入（`slide-in-right`，300ms）。
- 抽屉/模态框：由右向左滑入（`slide-in-right`，250ms），同时背景遮罩淡入。

### 5.2 属性雷达图交互（若实现）
- 悬停在数据点上时，显示具体数值的 Tooltip。
- 雷达图可带有轻微的入场绘制动画（线条由内向外生长）。

### 5.3 按钮交互
- 点击操作按钮后，根据功能触发相应子系统（如打开赠与界面、进入个人战）。
- 若操作需要确认（如「招募」），弹出确认模态框。

### 5.4 退出方式
- 点击「返回」「关闭」按钮或按下 `ESC` 键，界面以反向动画退出。

## 6. 数据绑定说明

### 6.1 界面需要的数据
| 数据字段 | 类型 | 说明 |
|----------|------|------|
| `person.id` | string | 人物唯一标识 |
| `person.name` | string | 姓名 |
| `person.courtesyName` | string | 字号（字） |
| `person.portrait` | string | 立绘图片路径 |
| `person.rank` | string | 品级（如「正一品」） |
| `person.position` | string | 官职（如「中书右丞相」） |
| `person.attributes` | object | 五维属性值（`lead`, `might`, `int`, `pol`, `cha`） |
| `person.biography` | string | 列传文本 |
| `person.skills` | array | 持有技能列表 |
| `person.items` | array | 持有物品列表 |
| `relationship.favor` | number | 与玩家的亲密度（0-100） |
| `availableActions` | array | 当前可执行的操作列表 |

### 6.2 界面触发的操作
| 操作 | 目标 |
|------|------|
| 点击「交谈」 | 触发对话系统，可能推进事件 |
| 点击「赠与」 | 打开物品选择界面，赠送后更新亲密度 |
| 点击「招募」 | 尝试招募，根据条件成功/失败 |
| 点击「交手」 | 进入个人战 |
| 点击「宴请」 | 消耗银两，提升亲密度 |

## 7. 对 AI 编程助手的生成提示

1. **组件化设计**：
   - 封装 `<AttributeBar>` 或 `<RadarChart>` 用于属性展示。
   - 封装 `<ActionButtons>` 根据 `availableActions` 动态渲染。

2. **雷达图实现**：
   - 使用 `<canvas>` 绘制，在 `useEffect` 或 `mounted` 中触发。
   - 数据更新时重绘。
   - 若需 Tooltip，监听 `mousemove` 事件计算最近数据点。

3. **立绘加载**：
   - 使用懒加载，并设置占位符（如墨色剪影）。
   - 图片缺失时显示默认立绘（如 `portrait_default.png`）。

4. **亲密度进度条**：
   - 使用 CSS `width` 百分比绑定 `favor` 值。

5. **古风细节**：
   - 列传首字下沉使用 `::first-letter`。
   - 品级/官职标签使用 `.badge` 样式。
   - 为人物姓名添加印章装饰（如「名将」「文臣」等）。

6. **响应式适配**：
   - 移动端（<640px）改为纵向布局：立绘在上方居中，信息区在下方。
   - 操作按钮可横向滚动或换行。
   - 雷达图尺寸缩小至 160×160px。

## 9. 附录：太阁立志传4 人物界面参考要点

- 太阁4的人物界面以**卡片形式**展示人物头像、姓名、能力值（条形图）。
- 能力值分为统率、武力、智力、政务、魅力五项，外加技能（如「军学」「剑术」等）。
- 下方提供「交谈」「赠与」「交手」等按钮。
- 本设计在保留卡片化信息呈现的基础上，加入了列传、雷达图等元素，更具沉浸感和收集价值。
