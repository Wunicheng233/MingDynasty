# 《洪武立志传》界面设计：城镇据点界面

> 版本：1.0.0  
> 继承自：《洪武立志传》整体 UI 设计规范 v1.0.0  
> 对应模块：`CityRenderer.js`（待新建）

---

## 1. 界面定位与功能概述

### 1.1 界面位置
城镇据点界面是玩家进入任意城市或据点后看到的主场景。这里是玩家进行社交、交易、接取任务、触发事件的核心交互枢纽。

### 1.2 主要功能
- 展示城镇的独特风貌（全屏场景背景）
- 提供主要功能入口卡片：「酒馆」「官邸」「市集」「城门」「自宅」等
- 显示城镇基本信息（名称、所属势力、发展度）
- 展示城镇内可互动的人物（如城主、商人、名士），以头像或小卡片形式呈现
- 提供返回大地图的入口（「出城」）

### 1.3 触发条件
- 从大地图点击城市并选择「进入」
- 完成特定事件后自动进入（如被传唤至官邸）
- 从其他子界面返回城镇主场景

### 1.4 界面特点
- **场景化沉浸**：每个城镇（或城镇类型）有独立的背景插画，描绘明代街景、府衙、市集等，风格统一为水墨淡彩或工笔。
- **卡片式入口**：主要功能设施以卡片形式横向排列于屏幕下方，点击进入对应子系统。
- **人物立绘点缀**：城镇内的重要 NPC 以小型立绘或头像悬浮于场景特定位置，点击可触发对话或互动。

## 3. 视觉继承与差异化说明

### 3.1 继承的全局令牌
| 元素 | 使用的令牌 |
|------|------------|
| 信息卡片背景 | `var(--color-bg-secondary)` + 半透明 |
| 卡片边框 | `var(--border-default)` |
| 卡片阴影 | `var(--shadow-md)` |
| 设施按钮 | 继承 `.card` 样式，结合 `.btn-secondary` 悬停效果 |
| 字体 | `--font-serif` 为主 |

### 3.2 本界面的特殊处理
- **全屏场景背景**：每个城镇使用独立的插画 PNG/WebP，风格统一为水墨淡彩或工笔，饱和度低，营造古画氛围。场景中预留 NPC 站位空间。
- **设施卡片**：采用**纵向卡片**设计（图标在上，文字在下），带有轻微悬浮感。选中或悬停时，卡片轻微上浮并显示朱砂色边框。
- **NPC 立绘交互**：人物以半身像或头像形式融入场景，带有名称标签。点击后弹出对话气泡或互动选项。
- **昼夜/天气变化**（可选）：根据游戏内时辰，场景可叠加半透明滤镜（如黄昏暖色、夜晚深蓝），增强沉浸感。

## 4. 详细组件与状态

### 4.1 场景背景
- **类型**：全屏背景图，尺寸建议 1920×1080px（桌面端），750×1334px（移动端）。
- **切换逻辑**：进入不同城镇时，动态加载对应的背景图。
- **叠加效果**：可随时辰叠加半透明渐变层（如 `.time-overlay`），不影响 UI 元素。

### 4.2 城镇信息卡片
- **容器**：`.city-info-card`，绝对定位，内边距 `12px 20px`，圆角 `var(--radius-md)`。
- **城镇名**：字体 `--font-calligraphy` 或 `--font-serif` 加粗，字号 `--text-heading-md`，颜色 `var(--color-text-primary)`。
- **所属势力**：字体 `--text-body-sm`，颜色 `var(--color-accent-blue)`。
- **发展度**：星级或数值条显示，使用古铜金色。

### 4.3 设施入口卡片
设施卡片是进入各子系统的入口。根据城镇类型（都城、府城、县城），可用的设施可能不同。

| 设施 | 图标（建议） | 功能说明 | 可用条件 |
|------|-------------|----------|----------|
| 酒馆 | 🍶 或 酒坛 | 打听消息、招募同伴、触发事件 | 所有城镇 |
| 官邸 | 🏛️ 或 衙门 | 接取任务、评定、拜见上级 | 有官署的城镇 |
| 市集 | 🧺 或 铜钱 | 买卖物品、交易粮食 | 商业城镇必有 |
| 城门 | 🚪 或 城门楼 | 出城、查看城防、移动至郊外 | 所有城镇 |
| 自宅 | 🏠 或 宅邸 | 休息、存档、管理家产 | 玩家拥有宅邸时 |
| 寺庙 | 卍 或 佛寺 | 参拜、学习技能、触发事件 | 有寺庙的城镇 |
| 医馆 | 🏥 或 药箱 | 治疗伤病、购买药品 | 较大城镇 |

**卡片样式**：
- 容器：`.facility-card`，宽度 100-120px，内边距 `12px 8px`，背景 `var(--color-bg-secondary)`，圆角 `var(--radius-md)`，阴影 `var(--shadow-md)`，`text-align: center`。
- 图标：尺寸 48×48px（PNG 或 SVG），居中显示。
- 文字：字体 `--font-serif`，字号 `--text-body-sm`，颜色 `var(--color-text-primary)`，上边距 `8px`。
- 悬停：`transform: translateY(-4px)`，阴影 `var(--shadow-lg)`，边框 `var(--border-highlight)`。
- 禁用态：若设施不可用（如无官邸），卡片置灰（`opacity: 0.5`），不可点击。

### 4.4 NPC 交互点
- **容器**：`.npc-interact`，绝对定位，使用百分比坐标定位。
- **立绘/头像**：尺寸约 80×100px（半身像）或 64×64px（头像），PNG 透明背景。
- **名称标签**：置于立绘下方或上方，背景 `var(--color-bg-dark)`，文字 `var(--color-text-inverse)`，字号 `--text-caption`，内边距 `2px 8px`，圆角 `var(--radius-sm)`。
- **交互反馈**：悬停时立绘轻微放大，点击后弹出互动菜单（「交谈」「赠与」「挑战」等）。

### 4.5 出城按钮
- **样式**：使用 `.btn-secondary` 大尺寸变体，置于设施栏下方中央或右下角。
- **图标**：城门图标 + 「出城」文字。
- **行为**：点击后返回大地图，并更新玩家位置为该城镇。

### 4.6 动态事件提示
当城镇内有特殊事件（如节日、名士来访、贼寇骚扰）时，在场景中显示浮动提示图标（如「！」或灯笼），点击可查看详情。

## 5. 交互逻辑与动效

### 5.1 场景进入动画
- 由黑场淡入（或从大地图镜头推近效果），时长 500ms。
- 城镇信息卡片与设施栏由下而上滑入（`slide-up`），延迟 100ms。

### 5.2 设施交互
- 点击设施卡片：卡片有轻微按下动画（`scale: 0.98`），然后切换到对应子系统界面（如酒馆、官邸）。切换时可配合卷轴展开效果。
- 返回城镇：从子系统返回时，城镇界面保持原状态，无重复入场动画。

### 5.3 NPC 交互
- 点击 NPC：弹出对话气泡或底部选项菜单（使用 `.modal` 或底部抽屉样式）。
- 对话气泡：仿古籍人物对话，背景为宣纸色，带三角指向 NPC。

### 5.4 出城
- 点击「出城」：界面淡出，返回大地图，并可能触发时间微推进（如半日）。

## 6. 数据绑定说明

### 6.1 界面需要的数据
| 数据字段 | 来源 | 用途 |
|----------|------|------|
| `currentCity` | 游戏状态 | 当前城镇 ID、名称、势力、发展度 |
| `cityFacilities` | 静态配置 | 该城镇可用的设施列表 |
| `npcsInCity` | 游戏状态 | 当前位于该城镇的 NPC 列表及其坐标/位置 |
| `playerHomeHere` | 游戏状态 | 玩家是否在此城镇拥有宅邸 |
| `specialEvents` | 游戏状态 | 当前城镇的特殊事件列表 |

### 6.2 界面触发的操作
| 操作 | 目标 |
|------|------|
| 点击设施卡片 | 进入对应子系统（酒馆 → `TavernUI`，官邸 → `OfficeUI` 等） |
| 点击 NPC | 打开与该 NPC 的互动菜单 |
| 点击「出城」 | 返回大地图（`MapUI`） |
| 点击事件提示 | 显示事件详情或直接触发事件 |

## 7. 对 AI 编程助手的生成提示

1. **场景背景管理**：
   - 建立 `cityBackgrounds` 映射表，根据 `cityId` 加载对应背景图。
   - 背景图使用 `background-size: cover`，并设置 `background-position: center`。
   - 若某城镇暂无专属背景，使用通用背景（如 `bg_city_generic.webp`）。

2. **设施卡片组件**：
   - 封装 `<FacilityCard>` 组件，接收 `icon`、`label`、`onClick`、`disabled` 等 props。
   - 根据 `cityFacilities` 数组动态渲染，避免硬编码。

3. **NPC 定位**：
   - 将 NPC 位置定义为相对于背景图容器的百分比（如 `{ left: '25%', top: '60%' }`），以便响应式适配。
   - 使用 `position: absolute` 渲染 NPC 交互点。

4. **状态保持**：
   - 从子系统返回时，城镇界面不应重新加载数据（使用 React 的 `useMemo` 或 Vue 的 `keep-alive`）。

5. **古风细节**：
   - 设施卡片可使用传统纹样装饰（如云纹边框）。
   - NPC 名称标签使用印章风格或竖向排版。
   - 出城按钮可添加淡墨笔触效果。

6. **响应式适配**：
   - 移动端设施卡片可横向滚动（`overflow-x: auto`），避免折行。
   - 城镇信息卡片在移动端可折叠为仅显示城镇名，点击展开详情。
   - NPC 交互点尺寸适当缩小，避免遮挡过多场景。

## 9. 附录：太阁立志传4 城镇界面参考要点

- 太阁4的城镇界面以全屏插画展示城镇风貌，不同城镇（如京、界、石山）有独立背景。
- 下方以文字按钮形式列出设施（「酒場」「宿屋」「米屋」「馬屋」等），无图标，但字体风格统一。
- 城镇内人物以 Q 版小人形式站在场景中，点击触发对话。
- 本设计在保留场景沉浸感的基础上，将设施升级为卡片式入口，更符合现代 UI 习惯，同时保留古风韵味。
