# 《洪武立志传》界面设计：大明疆域图（大地图）

> 版本：1.0.0  
> 继承自：《洪武立志传》整体 UI 设计规范 v1.0.0  
> 对应模块：`MapRenderer.js`（待新建）

---

## 1. 界面定位与功能概述

### 1.1 界面位置
大明疆域图是玩家进行战略移动、选择目的地、触发事件的核心主界面。玩家将在此规划行程、了解天下大势、并进入城镇或据点。

### 1.2 主要功能
- 展示大明两京十三省的宏观地图，标注主要城市、关隘、河流山川
- 提供极简移动指令：「回京述职」「传唤移动」「更衣歇息」
- 显示玩家当前位置、所属势力范围、目标追踪路径
- 点击城市/据点可查看详情并执行移动
- 显示游戏时间（年份、季节、时辰）与玩家基本状态

### 1.3 触发条件
- 完成角色创建/序章后首次进入
- 从城镇据点界面选择「出城」
- 载入存档后进入

### 1.4 界面特点
- **极简指令集**：借鉴《太阁立志传4》，屏幕下方仅保留三个核心按钮，将复杂操作收敛至子界面，保持地图视野开阔。
- **古地图美学**：地图本身采用明代风格手绘地图，山川河流用青绿山水笔触，城市以传统符号标注。
- **信息悬浮层**：玩家状态、时间信息、追踪目标以轻量面板悬浮于地图边缘，不遮挡主视野。

## 3. 视觉继承与差异化说明

### 3.1 继承的全局令牌
| 元素 | 使用的令牌 |
|------|------------|
| 面板背景 | `var(--color-bg-secondary)` + 半透明（约 0.85） |
| 面板边框 | `var(--border-default)` |
| 面板阴影 | `var(--shadow-md)` |
| 文字颜色 | 主文字 `var(--color-text-primary)`，次要 `var(--color-text-secondary)` |
| 按钮 | `.btn-primary`、`.btn-secondary`、`.btn-text` |
| 字体 | `--font-serif` 为主，数值用 `--font-mono` |

### 3.2 本界面的特殊处理
- **地图背景**：使用专门绘制的明代风格手绘地图 PNG，色彩以宣纸黄、青绿、淡墨为主，饱和度低。地图上标注城市、关隘等。
- **城市标记**：可用 CSS 绝对定位的圆点+文字，或直接绘制在地图图片上。若用 CSS 实现，便于后续动态更新（如势力变色）。
- **路径指示**：当追踪目标时，地图上显示一条**淡墨虚线**连接当前位置与目标城市，带有「行旅」动画（小点沿线移动）。
- **时辰更替**：左上角时间面板的背景色可随时辰轻微变化（如黄昏带暖色，夜晚带深蓝灰），增强沉浸感。
- **面板毛玻璃效果**：所有悬浮面板使用 `backdrop-filter: blur(4px)`，营造透过宣纸看地图的朦胧感。

## 4. 详细组件与状态

### 4.1 地图主体
- **类型**：大尺寸背景图（建议 2048×1536px 或更高），`background-size: cover` 或 `contain`，居中显示。
- **城市交互点**：若城市单独标记，每个城市为一个透明按钮，覆盖在城市坐标上。点击后弹出城市详情卡片。

### 4.2 左上时间面板
- **容器**：`.time-panel`，内边距 `8px 16px`，圆角 `var(--radius-md)`。
- **年份季节**：字体 `--font-serif`，字号 `--text-body`，颜色 `var(--color-text-primary)`。
- **时辰**：字体 `--font-serif`，字号 `--text-body-sm`，颜色 `var(--color-text-secondary)`。
- **时辰图标**（可选）：显示小图标（日、月、云），用 SVG 实现。

### 4.3 右上状态面板
- **容器**：`.status-panel`，内边距 `8px 16px`。
- **银两**：铜钱图标 + 数值，字体 `--font-mono`，颜色 `var(--color-accent-gold)`。
- **官品**：官帽/补子图标 + 文字，字体 `--font-serif`，颜色 `var(--color-text-primary)`。
- **头像**（可选）：圆形或方形小头像，边框 `var(--border-gold)`。

### 4.4 左下追踪面板
- **容器**：`.track-panel`，内边距 `8px 16px`。
- **当前位置**：图钉图标 + 城市名。
- **追踪目标**：旗帜图标 + 城市名，右侧带「✕」取消按钮（`.btn-text` 小尺寸）。
- **预计耗时**：沙漏图标 + 「尚需 X 日」，字体 `--text-body-sm`。

### 4.5 底部指令栏
借鉴太阁4的极简设计，三个大按钮横向排列，是整个地图交互的核心。

| 按钮 | 图标（建议） | 文字 | 类型 | 行为 |
|------|-------------|------|------|------|
| 回京述职 | 🏯 或 宫殿 | 回京述职 | `.btn-primary` 大尺寸 | 立即返回玩家所属的主城（如应天府），触发时间流逝并可能遭遇事件 |
| 传唤移动 | 🗺️ 或 马匹 | 传唤移动 | `.btn-secondary` 大尺寸 | 打开目标选择界面（列表或直接点选城市），确认后开始移动 |
| 更衣歇息 | 🌙 或 床榻 | 更衣歇息 | `.btn-secondary` 大尺寸 | 时间推进一个时辰/半日，恢复少量体力，可能触发夜间事件 |

- **按钮样式**：内边距 `12px 24px`（移动端 `8px 16px`），宽度自适应，最小宽度 120px。
- **禁用状态**：当玩家已在主城时，「回京述职」按钮置灰不可用。当已在移动中时，「传唤移动」变为「中止移动」（红色描边按钮）。

### 4.6 城市详情卡片（点击城市弹出）
- **容器**：`.city-detail-card`，绝对定位，靠近点击的城市。
- **内容**：城市名、所属势力、发展度简况、驻守武将（若有）、可进行的操作（「进入」「移动至此」「详情」）。
- **样式**：继承 `.card`，宽度 280px，阴影 `var(--shadow-lg)`。

### 4.7 移动路径指示
- 当存在追踪目标时，地图上显示一条连接当前位置与目标的虚线。
- 虚线颜色：`var(--color-accent-blue)`，`stroke-dasharray: 8 8`。
- 动画：使用 `stroke-dashoffset` 动画模拟行军流动感。

### 4.8 随机事件弹窗
移动或歇息过程中可能触发随机事件（遇盗匪、访名士、天灾等），以模态框形式展示，详见《界面设计-事件弹窗.md》。

## 5. 交互逻辑与动效

### 5.1 地图探索
- **拖拽/缩放**：若实现，支持鼠标拖拽平移地图，滚轮缩放。需限制缩放范围，避免过度失真。
- **点击城市**：高亮城市标记（外发光或放大），弹出详情卡片。点击卡片外区域关闭。
- **悬停城市**：显示城市名称浮层（`.tooltip`）。

### 5.2 指令执行流程
**传唤移动**：
1. 点击「传唤移动」→ 进入目标选择模式，地图上可移动的城市高亮（带光圈）。
2. 点击目标城市 → 若目标可达（同一势力或相邻区域），弹出确认框：「移驾苏州府？预计需 3 日」。
3. 确认后，玩家状态变为「移动中」，底部按钮变化，「传唤移动」变为「中止移动」。
4. 时间开始流逝（可逐日或逐时段推进），地图上玩家标记沿路径移动。

**回京述职**：
1. 点击后直接确认：「返回应天府述职？耗时约 X 日」。
2. 确认后播放移动动画（快速跳转或镜头移动至主城），进入城镇界面。

**更衣歇息**：
1. 点击后直接推进时间（如 2 个时辰），恢复少量体力。
2. 期间可触发夜间事件（弹窗）。

### 5.3 动画与过渡
- 弹出城市卡片：`scale-fade` 动画（由小变大 + 淡入，200ms）。
- 时间流逝：时间面板上的时辰文字切换时带轻微闪烁或翻页效果。
- 路径动画：小点沿线移动，速度与时间推进同步。

## 6. 数据绑定说明

### 6.1 界面需要的数据
| 数据字段 | 来源 | 用途 |
|----------|------|------|
| `currentCity` | 游戏状态 | 玩家当前所在城市 |
| `targetCity` | 游戏状态 | 当前追踪的目标城市（可为空） |
| `travelProgress` | 游戏状态 | 移动进度（0-1）或剩余天数 |
| `gameTime` | 游戏状态 | 年份、季节、时辰、日期 |
| `playerInfo` | 游戏状态 | 银两、官品、头像、所属势力 |
| `cities` | 静态数据/存档 | 所有城市坐标、名称、势力、发展度 |
| `isMoving` | 游戏状态 | 是否正在移动中 |

### 6.2 界面触发的操作
| 操作 | 目标 |
|------|------|
| 点击城市 | 显示 `cityDetail` |
| 执行「传唤移动」 | 进入目标选择模式，确认后调用 `startTravel(targetCity)` |
| 执行「回京述职」 | 调用 `returnToCapital()` |
| 执行「更衣歇息」 | 调用 `rest(amount)`，推进时间 |
| 执行「中止移动」 | 调用 `cancelTravel()`，停留在当前位置 |

## 7. 对 AI 编程助手的生成提示

1. **地图交互层**：
   - 若城市标记用 CSS 实现，建议将城市数据定义为数组，动态渲染 `<button>` 覆盖层。
   - 使用 `position: absolute` 与 `left/top` 百分比定位每个城市点。
   - 地图容器需设置 `position: relative` 与固定的宽高比。

2. **时间系统**：
   - 封装 `gameTime` 状态管理，使用 `setInterval` 或游戏循环驱动时间流逝（仅移动中或歇息时）。
   - 时间推进时触发事件检测（随机遭遇）。

3. **路径绘制**：
   - 可使用 `<canvas>` 或 SVG 绘制连接线。在 React 中可使用 `useRef` + `useEffect` 在 canvas 上绘制。
   - 动画可用 `requestAnimationFrame` 更新 `dashoffset`。

4. **响应式适配**：
   - 移动端地图支持手势缩放（使用 `touch-action: pan-x pan-y`）。
   - 底部按钮在移动端可缩小内边距，或改为纵向排列（若屏幕极窄）。
   - 悬浮面板在移动端可折叠或减小字号。

5. **古风细节**：
   - 地图背景本身是重中之重，需提供高质量手绘地图图片。
   - 时间面板添加时辰图标（太阳/月亮），增强氛围。
   - 底部按钮可使用图标 + 文字，图标采用线性古风风格。

6. **性能优化**：
   - 地图图片较大时，考虑使用 WebP 格式并压缩。
   - 避免在动画循环中进行大量 DOM 操作。

### 8.5 对 AI 的图片使用提示
- 地图背景作为 `<div>` 的 `background-image`，并设置 `background-size: cover` 或 `contain`。

- 面板图标使用 `<img>` 或 SVG 组件引入，与文字同行。
- 若无图标资源，可使用纯文本符号替代（如「💰」「👤」「📍」），但风格需统一。

## 9. 附录：太阁立志传4 地图界面参考要点

- 太阁4的地图界面极其简洁，下方仅有「本处归返」「行走指定」「待机」三个指令，本设计忠实地映射了这一交互逻辑。
- 地图上的城市以统一的小城堡图标表示，点击后弹出菜单。
- 移动路径以红色虚线表示，并有移动动画。
- 时间显示在地图左上角，包含年、月、日、时辰。
