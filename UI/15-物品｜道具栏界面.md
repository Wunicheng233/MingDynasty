# 《洪武立志传》界面设计：物品/道具栏

> 版本：1.0.0  
> 继承自：《洪武立志传》整体 UI 设计规范 v1.0.0  
> 对应模块：`InventoryRenderer.js`（待新建）

---

## 1. 界面定位与功能概述

### 1.1 界面位置
物品栏是玩家查看、管理持有物品（宝物、书籍、武器、消耗品、任务道具等）的核心面板。它通常以**全屏界面**或**半屏抽屉**形式呈现，是玩家在游戏中积累财富与实力的重要体现。

### 1.2 主要功能
- 以网格形式展示所有持有的物品
- 显示物品图标、名称、数量、简要说明
- 支持物品分类筛选（全部、武器、书籍、宝物、消耗品、任务道具）
- 支持物品排序（按获得时间、名称、价值、类型）
- 提供物品操作：「使用」「装备」「赠与」「丢弃」
- 显示背包容量（已用/上限）

### 1.3 触发条件
- 在城镇自宅选择「物品」
- 在赠礼、交易等界面作为子面板调用
- 快捷键（如 `I`）快速打开

### 1.4 界面形式
| 形式 | 触发场景 | 特点 |
|------|----------|------|
| **全屏物品栏** | 从自宅/菜单进入 | 沉浸感强，信息完整，带分类与搜索 |
| **半屏抽屉** | 赠礼、交易时调用 | 轻量，可快速选择物品后关闭 |

> 本文档以**全屏物品栏**为主要描述对象。

## 3. 视觉继承与差异化说明

### 3.1 继承的全局令牌
| 元素 | 使用的令牌 |
|------|------------|
| 背景 | `var(--color-bg-primary)` |
| 卡片背景 | `var(--color-bg-secondary)` |
| 字体 | `--font-serif` |
| 按钮 | `.btn-primary`、`.btn-secondary`、`.btn-text` |
| 分类标签 | 继承图鉴界面的 `.filter-tab` |

### 3.2 本界面的特殊处理
- **物品图标**：采用统一的图标底板（如锦缎纹样或古籍衬底），物品图标居中，品质用边框颜色区分（白、绿、蓝、紫、金对应不同品级）。
- **容量条**：使用类似进度条的视觉元素，填充色为古铜金。
- **装备中标记**：已装备的物品在卡片上显示「装备中」印章或角标。
- **详情区**：模仿古籍中的「器物志」排版，左侧可放置物品大图标，右侧为描述与属性。

## 4. 详细组件与状态

### 4.1 资源摘要栏
- **银两**：铜钱图标 + 数值，字体 `--font-mono`，颜色 `var(--color-accent-gold)`。
- **容量**：背包图标 + `已用 / 上限`，配合微型进度条（`width: 100px`，填充色 `var(--color-accent-gold)`）。

### 4.2 分类标签与搜索
- **分类标签**：复用 `.filter-tab` 组件，选中项有朱砂色下划线。
- **搜索框**：`.input` 样式，左侧放大镜图标，占位符「搜索物品...」。
- **排序下拉**：复用 `.dropdown` 组件，选项包括「获得时间」「名称」「价值」「类型」。

### 4.3 物品卡片
- **容器**：`.item-card`，宽度 `100px`（桌面端），内边距 `8px`，背景 `var(--color-bg-secondary)`，边框 `var(--border-default)`，圆角 `var(--radius-md)`。
- **图标**：尺寸 `48×48px`，居中，底板可有锦纹装饰。
- **名称**：字体 `--font-serif`，字号 `--text-body-sm`，居中对齐，单行截断。
- **数量**（可堆叠物品）：右下角显示「×3」，字体 `--text-caption`。
- **品质边框**：
  - 普通：`var(--border-default)`
  - 精良：绿色边框（`#2D6A4F`）
  - 稀有：蓝色边框（`#3A526B`）
  - 史诗：紫色边框（`#7A4F8B`）
  - 传说：金色边框（`var(--color-accent-gold)`）+ 微光动画
- **装备中标记**：卡片左上角显示「⚔️」或「装备」印章。
- **悬停**：边框高亮，阴影加深。
- **选中态**：边框朱砂色，外围光晕。

### 4.4 物品详情区
- **容器**：`.item-detail`，内边距 `16px`，背景 `var(--color-bg-secondary)`，边框 `var(--border-double)`，圆角 `var(--radius-md)`。
- **布局**：左侧大图标（64×64px），右侧信息。
- **名称**：字体 `--font-serif` 加粗，字号 `--text-heading-sm`，颜色 `var(--color-text-primary)`。
- **描述**：字体 `--text-body-sm`，颜色 `var(--color-text-secondary)`，多行。
- **属性**：如「攻击力 +15」「智力 +3」，使用 `.badge` 样式展示。
- **价值**：铜钱图标 + 数值，置于右下。

### 4.5 操作按钮
根据物品类型与场景，动态显示可用操作。

| 操作 | 显示条件 | 行为 |
|------|----------|------|
| 使用 | 消耗品、可阅读书籍 | 使用物品，触发效果（恢复体力、学习技能等） |
| 装备 | 武器、防具、宝物 | 装备到对应槽位，替换已有装备 |
| 赠与 | 几乎所有物品（任务道具除外） | 打开赠礼对象选择，赠出物品 |
| 丢弃 | 非关键物品 | 二次确认后销毁物品 |

- **按钮样式**：横向排列，间距 `var(--space-sm)`。

### 4.6 空状态
当筛选条件下无物品时，显示空状态：
- 中央显示古风空箱图案或印章「囊中羞涩」。
- 文字：「暂无此类物品」。

## 5. 交互逻辑与动效

### 5.1 进入动画
- 界面整体由右向左滑入（`slide-in-right`，300ms）。
- 物品卡片以交错淡入上浮出现（`stagger`）。

### 5.2 筛选与搜索
- 点击分类标签即时过滤物品网格。
- 搜索框输入时防抖过滤。
- 切换排序时物品重新排列，带交叉淡入淡出。

### 5.3 物品选中
- 点击物品卡片高亮，详情区更新。
- 双击物品可执行默认操作（如使用消耗品）。

### 5.4 物品操作
- 点击「使用」「装备」等按钮后，播放操作动画（如物品飞入角色），更新状态。
- 丢弃需二次确认：「确定丢弃『青釭剑』？此操作不可恢复。」

### 5.5 拖拽（可选）
- 支持将物品拖拽到角色装备槽或赠礼对象头像上，简化操作。

## 6. 数据绑定说明

### 6.1 界面需要的数据
| 数据字段 | 类型 | 说明 |
|----------|------|------|
| `inventory` | array | 玩家所有物品列表 |
| `capacity` | object | `{ used: number, max: number }` |
| `silver` | number | 玩家银两 |
| `selectedItem` | object | 当前选中的物品 |
| `filterType` | string | 当前分类 |
| `searchKeyword` | string | 搜索关键词 |
| `sortBy` | string | 排序字段 |

### 6.2 界面触发的操作
| 操作 | 目标 |
|------|------|
| 使用物品 | 调用 `useItem(itemId)` |
| 装备物品 | 调用 `equipItem(itemId, slot)` |
| 赠与物品 | 打开赠礼界面，选择对象后调用 `giftItem(itemId, targetId)` |
| 丢弃物品 | 调用 `discardItem(itemId)` |

## 7. 对 AI 编程助手的生成提示

1. **物品数据结构**：定义 `Item` 类型，包含 id、name、type、icon、quality、description、attributes、stackable、quantity、value 等字段。
2. **网格渲染**：使用 CSS Grid 实现响应式网格。
3. **筛选与排序**：使用计算属性（Vue）或 `useMemo`（React）生成过滤排序后的列表。
4. **品质边框**：通过 CSS 类 `.quality-common`、`.quality-rare` 等实现不同颜色边框。
5. **操作反馈**：操作后通过 Toast 提示结果（如「已装备青釭剑」）。
6. **古风细节**：详情区可添加竖排标点、首字下沉等古籍排版元素。

## 9. 附录：太阁立志传4 物品系统参考要点

- 太阁4中物品分为武器、防具、茶器、书籍、艺术品等，可收藏、赠礼、使用。
- 物品图标简洁，品质以稀有度区分。
- 赠礼是提升亲密度的重要手段。
- 本设计在保留经典物品分类与操作基础上，强化了物品品质可视化与详情展示，提升收集感。
