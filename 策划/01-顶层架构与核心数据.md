# 01 - 顶层架构与核心数据

## 一、整体架构设计

### 1.1 三层架构原则

游戏采用 **"核心数据层 → 系统模块层 → 玩法表现层"** 三层分离架构：

```
┌─────────────────────────────────────────────────────────────┐
│                      玩法表现层                              │
│   （文字界面：评定会、城镇场景、战斗界面、事件对话）          │
└─────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                      系统模块层                              │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌───────┐ │
│  │  合战  │ │ 个人战  │ │  主命  │ │  社交  │ │ 经济  │ │
│  └────┬────┘ └────┬────┘ └────┬────┘ └────┬────┘ └───┬───┘ │
│       └───────────┴───────────┴───────────┴──────────┘     │
│                           ▼                                  │
│                    ┌──────────────┐                          │
│                    │ 卡片收集系统 │（贯穿所有模块）            │
│                    └──────────────┘                          │
└─────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                      核心数据层                              │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐         │
│  │角色属性数据  │ │世界状态数据  │ │卡片库数据    │         │
│  │(玩家/NPC)    │ │(时间/城镇/   │ │(600+张卡片)  │         │
│  └──────────────┘ └──────────────┘ └──────────────┘         │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 核心设计原则（借鉴太阁立志传4）

1. **卡片驱动一切**：所有可收集内容（人物、技能、战术、称号、名所）统一卡片化
2. **评定循环节奏**：每两个月一次评定会，控制游戏推进节奏
3. **数据与逻辑分离**：静态定义与动态状态分离，便于扩展
4. **高自由度生涯**：支持武将、文官、商人、侠客、锦衣卫等多种人生路线

### 1.3 核心游戏循环（太阁式）

```
评定会（每两月）
    ↓
选择主命 → 执行主命（进入对应玩法：合战/个人战/小游戏/事件）
    ↓
结算奖励 → 功勋+技能经验+卡片
    ↓
返回评定会 → 功勋累积 → 身份晋升
    ↓
触发历史事件 → 推进剧情 → 最终统一/结局
```

## 二、核心数据层详细设计

### 2.1 设计原则

1. **卡片驱动**：所有可收集内容以 `card_id` 全局唯一标识
2. **最小冗余**：静态数据只存ID，不复制完整数据
3. **状态定义分离**：静态定义（游戏内置） vs 动态状态（玩家存档）分离
4. **版本兼容**：预留 `version` 字段便于存档迁移
5. **可读性优先**：字段命名清晰，注释详尽

### 2.2 静态库（游戏内置，不随存档改变）

| 静态库文件 | 内容说明 |
|------------|---------|
| `char_templates.json` | 所有NPC角色模板，基础属性、技能、初始卡片 |
| `card_library.json` | 全部648张卡片定义 |
| `city_defs.json` | 各城镇初始定义（规模、特产、设施） |
| `faction_defs.json` | 各势力初始数据（城池、将领、外交） |
| `skill_defs.json` | 16项技能定义，等级效果 |
| `mission_templates.json` | 所有主命模板参数 |
| `event_library.json` | 所有事件定义（触发条件、选项、效果） |

### 2.3 动态存档（玩家存档文件）

| 存档文件/数据块 | 内容说明 |
|----------------|---------|
| `player` | 玩家角色当前数据（属性、技能、卡片、位置、身份） |
| `npcs` | 已生成NPC的当前状态 |
| `factions` | 各势力当前状态（城池、将领、资金） |
| `cities` | 各城镇当前状态（防御、规模、投资、资金） |
| `relationships` | 角色间亲密度网络 |
| `global_time` | 当前时间（年/月/日） |
| `event_flags` | 已触发事件标记 |

```
┌────────────┐     引用      ┌────────────┐
│ 存档数据  │ ───────────► │ 静态库    │
│ player    │             │ char_tpl  │
│ factions │             │ cards     │
└────┬─────┘             └────┬───────┘
      │                       │
      │ 读写                   │ 提供定义
      ▼                       ▼
┌─────────────────────────────────────────────┐
│               系统模块                         │
│  合战 ← 读取 player.skills + owned_cards      │
│  个人战 ← 读取 player.skills + owned_cards    │
│  主命 ← 读取 player.identity + faction       │
│  社交 ← 读写 relationships                    │
└─────────────────────────────────────────────┘
```

## 三、初始化流程（新游戏）

1. **加载静态库**：所有 `*.json` 静态定义读入内存常量
2. **创建新存档**：
   - 从角色模板复制玩家初始数据
   - 设置初始时间（1353年3月，投军起点）
   - 初始化必要NPC（郭子兴、徐达、汤和、马秀英）
   - 根据势力模板初始化势力和城镇
   - 设置初始事件标记
   - 授予初始卡片
3. **存档/读档**：
   - 存档：序列化所有动态数据到JSON
   - 读档：反序列化覆盖内存动态数据
4. **数据校验**：读档时检查版本，必要时执行迁移

## 四、与太阁立志传4的对比

| 太阁4 | 洪武立志传 |
|------|------------|
| 日本战国 | 明初洪武 |
| 武士/忍者/海贼/商人 | 武将/文官/商人/锦衣卫/侠客 |
| 卡片收集600+ | 卡片收集648张 |
| 每一月评定 | 每两月评定（契合明朝军政节奏） |
| 16项技能 | 16项技能（本土化适配） |

本架构完全继承太阁4的"卡片驱动、评定循环"核心精髓，只是完全本土化到明朝背景。
