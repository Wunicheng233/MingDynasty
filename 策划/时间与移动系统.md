## 《洪武立志传》时间与移动系统详细设计

时间与移动系统是游戏世界的**骨架**。它决定了玩家行动的节奏、剧情推进的节点，以及所有动态事件的触发时机。在文字游戏中，清晰的时间反馈和合理的移动消耗，是营造真实感和策略性的关键。

以下从时间单位、日历系统、行动耗时、移动机制、时间驱动事件、界面设计、数据结构、与各系统耦合八个层面展开。


### 一、时间单位与推进机制

游戏采用**年-月-日**三级时间制，最小行动单位为“天”。

| 单位 | 说明 |
|------|------|
| **天** | 基本时间单位。执行绝大多数行动都会消耗天数。 |
| **月** | 每两个月触发一次**评定会**（奇数月1日）。每月30天，一年12个月。 |
| **年** | 用于记录历史年份和角色年龄。起始年份为1328年（朱元璋出生）或直接从1353年（投军）开始。 |

**时间推进方式**：
- **主动推进**：玩家选择执行一个消耗时间的行动（如移动、修行、执行主命），时间自动增加对应天数。
- **被动推进**：某些剧情事件触发后，时间可能直接跳跃（如“数月后……”）。
- **时间暂停**：在菜单界面、对话、战斗等非行动状态下，时间不流逝。


### 二、日历系统与明朝纪年

为增强沉浸感，日历同时显示**年号纪年**与**公历年份**。

**纪年规则**：
- 1356年以前，使用元朝年号（元统、至元、至正）。
- 1356年朱元璋攻占应天后，显示“吴国公元年”起算。
- 1368年称帝后，使用“洪武”年号。

**示例日历显示**：
```
═══════════════════════════════════════════
  至正十五年（1355年）三月十五日 · 春季
  应天府 · 未时
═══════════════════════════════════════════
```

**季节与天气**（可选，增加氛围）：
- 春季（1-3月）、夏季（4-6月）、秋季（7-9月）、冬季（10-12月）。
- 天气影响某些事件：雨天火器威力减半，大雪移动速度减慢等。


### 三、行动耗时规则

每种行动都有预设的**基础耗时**（单位：天），部分受技能、身份或天气影响。

| 行动类别 | 行动名称 | 基础耗时 | 影响因素 |
|---------|---------|---------|---------|
| **移动** | 相邻城镇间移动 | 2~5天 | 两地距离、骑战技能（加快）、天气 |
| **修行** | 师事学习技能 | 5天/次 | 无 |
| **任务** | 执行主命（如筹措军粮） | 15~30天 | 任务本身时限 |
| **社交** | 送礼、茶会、切磋 | 1天 | 无 |
| **小游戏** | 算术、口才等 | 1天/局 | 无 |
| **战斗** | 个人战 | 0天（瞬间） | 剧情战可能消耗天数 |
| **战斗** | 合战 | 视规模1~5天 | 攻城战通常更长 |
| **探索** | 访问名胜 | 1天 | 首次访问可能触发事件消耗更多 |
| **城镇内移动** | 在城内切换设施 | 0天 | 城内自由移动不消耗时间 |

**移动时间具体计算**：
- 预设一张`distance_matrix`表，定义城镇间的基础移动天数。
- 玩家“骑战”技能每等级减少10%移动时间（向下取整，最少1天）。
- 冬季移动时间+20%。


### 四、城镇与移动网络

游戏世界由若干城镇（City）和特殊地点（如名所、山寨）构成。城镇之间通过固定路线连接。

**城镇数据结构**：
```json
{
  "city_id": "YINGTIAN",
  "name": "应天府",
  "connections": [
    { "target": "FENGYANG", "base_days": 4 },
    { "target": "SUZHOU", "base_days": 3 },
    { "target": "ZHENJIANG", "base_days": 2 }
  ]
}
```

**移动界面示例**：
```
═══════════════════════════════════════════
            大地图 · 移动
═══════════════════════════════════════════
当前位置：应天府
可前往的地点：

1. 凤阳府（4天）
2. 苏州府（3天）
3. 镇江府（2天）
4. 返回（取消移动）

当前骑战Lv.2，移动时间减少20%。
请输入数字选择：
═══════════════════════════════════════════
```

**移动过程**：
- 玩家选择目的地后，时间流逝相应天数，期间可能触发随机事件（见下文）。
- 到达后自动进入目标城镇的主菜单。


### 五、时间驱动的事件系统

游戏世界不是静止的。随着时间流逝，会发生各种**时间驱动事件**。

**1. 定期事件**
- **评定会**：每逢奇数月1日自动触发（若玩家在主公居城）。
- **季节更替**：3/6/9/12月1日触发季节变化提示，可能影响物价、修行效果。
- **年始**：1月1日触发新年事件（拜年、领取俸禄、年龄增长）。

**2. 随机事件（移动/停留时）**
在移动过程中或城镇停留时，有一定概率触发：
- **遭遇山贼**：触发个人战，胜利可获得少量金钱。
- **行商叫卖**：可低价购买稀有宝物。
- **义士投奔**：魅力高的玩家可能吸引浪人加入。
- **天灾人祸**：影响城镇状态（如旱灾降低粮食产量）。

**3. 历史剧情事件**
根据当前日期和已触发标记，自动推进主线/支线剧情。
- 例如：至正十五年（1355年）六月，郭子兴病逝事件。
- 条件：当前日期≥1355.6.1，玩家在郭子兴势力，且郭子兴存活。

**事件触发优先级**：历史事件 > 定期事件 > 随机事件。


### 六、时间限制与紧迫感

为了增加策略性，游戏中有多种时间限制：

| 限制类型 | 说明 | 后果 |
|---------|------|------|
| **主命期限** | 评定会接受的任务有完成天数上限 | 超期未完成扣除功勋、降低主公亲密度 |
| **修行期限** | 某些师父只在一段时间内停留 | 错过需等待下次云游 |
| **历史人物寿命** | NPC有生卒年 | 到达卒年可能触发死亡事件（部分可避免） |
| **剧情窗口** | 某些事件只在特定时间段可触发 | 错过则本周目无法再触发 |

### 八、数据结构设计

**全局时间数据结构**（存档中）：
```json
{
  "global_time": {
    "year": 1355,
    "month": 3,
    "day": 15,
    "total_days": 492500
  }
}
```
- `total_days`：从游戏起始（如1328.1.1）累计的总天数，用于方便计算时间差和排序。

**时间推进函数（伪代码）**：
```python
def advance_time(days):
    global_time["day"] += days
    while global_time["day"] > 30:
        global_time["day"] -= 30
        global_time["month"] += 1
        if global_time["month"] > 12:
            global_time["month"] = 1
            global_time["year"] += 1
    # 触发各种时间检查
    check_regular_events()
    check_historical_events()
```

**移动距离矩阵（静态数据）**：
```json
{
  "distance_matrix": {
    "YINGTIAN": { "FENGYANG": 4, "SUZHOU": 3, "ZHENJIANG": 2 },
    "FENGYANG": { "YINGTIAN": 4, "XUZHOU": 5 }
  }
}
```


### 九、与各系统的耦合关系

| 系统 | 对时间的依赖 | 时间如何影响该系统 |
|------|-------------|-----------------|
| **评定会系统** | 每两月1日强制触发 | 若玩家不在主公居城则缺席惩罚 |
| **主命系统** | 任务有完成期限 | 超期判定失败；期限倒计时显示 |
| **修行系统** | 每次修行消耗固定天数 | 师父云游有时间窗口 |
| **社交系统** | 亲密度会随时间缓慢衰减 | 长时间不互动亲密度-1/半年 |
| **经济系统** | 物价每月波动 | 季节影响粮价；投资效果随时间显现 |
| **战斗系统** | 合战消耗天数 | 长途出征需考虑粮草消耗 |
| **角色成长** | 年龄增长影响某些事件 | 到达一定年龄可触发剧情（如弱冠、及冠） |
| **历史剧情** | 严格按日期触发 | 驱动主线前进 |


### 十、明朝特色时间事件举例

- **至正二十三年（1363年）七月**：鄱阳湖之战爆发。
- **至正二十四年（1364年）正月**：朱元璋自立为吴王。
- **洪武元年（1368年）正月初四**：朱元璋称帝，建立明朝。
- **每年八月**：武举乡试（需提前报名）。
- **每年二月**：会试（京城）。
- **每月初一、十五**：集市最热闹，商品最全。


### 十一、实现建议（文字游戏）

**1. 时间管理模块**：
编写一个`TimeManager`类，负责存储当前时间，提供`advance(days)`方法，并在推进后触发事件检查。

**2. 事件调度器**：
用一个列表存储所有待触发的事件（包括定期事件和一次性历史事件），每推进一天就检查是否有事件满足条件。

**3. 界面辅助**：
在玩家可能忽略时间限制的地方给予提示，例如主命倒计时用红色字体显示剩余天数。

**4. 存档兼容**：
时间数据必须随存档保存，读取存档时重建时间状态。


### 十二、总结

时间与移动系统是《洪武立志传》的脉搏。它让游戏世界有规律地运转，赋予玩家决策的紧迫感，同时为历史剧情的推进提供了时间轴。通过合理设计移动消耗和定期事件，玩家需要在“自由探索”与“履行职责”之间做出权衡——这正是《太阁立志传》系列时间管理的精髓。

有了这一系统，下一步你就可以将评定会、主命、修行等模块挂载到时间轴上，让游戏真正“活”起来。
