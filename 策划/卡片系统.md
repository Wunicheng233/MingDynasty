## 《洪武立志传》卡片系统完整设计

卡片系统是《洪武立志传》的**核心驱动引擎**，一切可收集、可解锁、可成长的内容均以“卡片”形态存在。玩家通过收集卡片来解锁新角色、习得新技能、获得战术招式、佩戴称号、记录足迹——这正是《太阁立志传》系列“卡片驱动”精神的完全继承。

以下从设计理念、卡片类型、数据结构、获取途径、收集图鉴、与各系统的交互、实现建议七个层面展开。


### 一、设计理念

**“一切皆卡片”**。在游戏中，玩家无论是学会了一招“太祖长拳”、在鄱阳湖击败了陈友谅、游览了秦淮河、获得了“策士”称号，还是与徐达结为生死之交——所有这些成就，最终都体现为一张张精美的卡片被收入“卡片图鉴”中。

**卡片系统的三大核心作用**：
1. **收集成就感**：600+卡片构成游戏的终极收集目标，驱动玩家反复游玩不同角色、尝试不同路线。
2. **玩法解锁器**：许多游戏内容需要先获得对应卡片才能体验。例如，你必须先获得“徐达”人物卡，才能在自由模式中选择扮演他；必须通过修行习得“火攻”战术卡，才能在合战中使用火攻指令。
3. **跨存档继承**：卡片图鉴是**跨存档共享**的（类似《太阁4》的卡片收集簿）。玩家在第一章通关后收集到的卡片，将永久保留，并可用于自由模式的新游戏。这是游戏耐玩度的核心来源。


### 二、卡片类型与数量规划

| 类型 | 预估数量 | 作用说明 |
|------|---------|---------|
| **人物卡** | 约300张 | 解锁可扮演角色。初期只可扮演朱元璋，收集后可选他人开局。 |
| **技能卡** | 48张 | 16项技能×3等级，记录玩家技能成长轨迹（实际技能等级存于角色属性，技能卡用于图鉴收集）。 |
| **合战战术卡** | 约60张 | 红色特殊卡，在合战中可主动使用（如火攻、三段击、伏兵等）。 |
| **个人战武技卡** | 约40张 | 个人战中的红色特殊技卡（如太祖长拳、四两拨千斤等）。 |
| **秘传卡** | 约30张 | 传奇名将专属金卡，个人战最强招式（如“常十万”、“北伐突进”）。 |
| **称号卡** | 约50张 | 达成特定成就后获得（如“乡勇”、“策士”、“武状元”），佩戴可提供属性加成。 |
| **名所卡** | 约30张 | 游览明朝版图内名胜古迹后获得（如秦淮河、鄱阳湖、嘉峪关）。 |
| **事件卡** | 约40张 | 记录重大历史事件（如“鄱阳湖之战”、“建立大明”），部分事件卡可解锁特殊玩法。 |
| **宝物卡** | 约50张 | 贵重物品（武器、书画、茶器等），可持有、送礼、或作为任务物品。 |
| **总计** | **约648张** | 丰富的收集要素支撑多周目游玩。 |


### 三、卡片数据结构

统一用`card_id`作为全局唯一标识。各类型卡片虽字段略有差异，但共享基础字段。

#### 3.1 基础字段（所有卡片共有）

```json
{
  "card_id": "CHAR_XU_DA",
  "type": "人物卡",
  "name": "徐达",
  "rarity": 5,
  "description": "明朝开国第一功臣，统兵如神，被誉为‘万里长城’。",
  "acquire_hint": "与徐达亲密度达到3心，或完成北伐剧情。",
  "is_hidden": false,
  "unlock_condition": null
}
```

- `card_id`：唯一标识，命名规则 `类型缩写_内容ID`，如`CHAR_`人物、`TACTIC_`合战战术、`MARTIAL_`武技、`SECRET_`秘传、`TITLE_`称号、`PLACE_`名所、`EVENT_`事件、`TREASURE_`宝物。
- `type`：卡片类型枚举。
- `rarity`：稀有度1-5星，影响收集成就感。
- `acquire_hint`：游戏内图鉴显示的获取提示文本，引导玩家。
- `is_hidden`：是否隐藏卡片信息（如某些隐藏人物、终极称号），获得前图鉴不显示。
- `unlock_condition`：某些卡片需要前置条件才能触发获取（如必须持有某张人物卡）。

#### 3.2 各类型卡片扩展字段

**（1）人物卡**

```json
{
  "card_id": "CHAR_CHANG_YUCHUN",
  "type": "人物卡",
  "name": "常遇春",
  "portrait": "changyuchun",
  "base_stats": { "统率": 92, "武力": 100, "内政": 40, "智力": 72, "魅力": 78 },
  "initial_skills": { "步战": 3, "骑战": 3, "武艺": 3 },
  "faction": "朱元璋军",
  "personality": "豪勇",
  "exclusive_secret": "SECRET_CHANG_100K",
  "unlock_playable": true
}
```

- `portrait`：立绘资源ID。
- `base_stats`、`initial_skills`：以该角色开始新游戏时的初始属性。
- `exclusive_secret`：指向其专属秘传卡ID。
- `unlock_playable`：获得此卡后是否可在新游戏中选择扮演（绝大部分人物卡为true，少数特殊角色可能不可扮演）。

**（2）合战战术卡**

```json
{
  "card_id": "TACTIC_FIRE_ATTACK",
  "type": "战术卡_合战",
  "name": "火攻",
  "color": "red",
  "duration": "instant",
  "effect": {
    "damage_multiplier": 1.5,
    "castle_defense_damage": 3
  },
  "acquire": {
    "method": "learn",
    "skill": "工政",
    "level": 1
  }
}
```

- `color`：`red`（特殊卡）或`green`（基础数字卡，但基础卡1-9为固定拥有，不作为收集卡片）。
- `duration`：`instant`（即时生效）或`permanent`（常驻）。
- `effect`：具体效果，由合战系统解析。
- `acquire`：获取方式定义，此处为向工政技能师父学习Lv1时获得。

**（3）个人战武技卡**

```json
{
  "card_id": "MARTIAL_TAIZU_CHANGQUAN",
  "type": "武技卡_个人战",
  "name": "太祖长拳",
  "priority": "B",
  "effect": {
    "hits": 2,
    "base_damage": 18
  },
  "acquire": {
    "method": "learn",
    "skill": "武艺",
    "level": 1,
    "location": "应天府武学"
  }
}
```

**（4）秘传卡**

```json
{
  "card_id": "SECRET_BEIFA_TUJIN",
  "type": "秘传卡",
  "name": "北伐突进",
  "exclusive_to": "CHAR_XU_DA",
  "priority": "S",
  "effect": {
    "hits": 3,
    "damage_scale": [1.0, 1.1, 1.2],
    "ignore_defense": true
  },
  "acquire": {
    "method": "duel_win_with_condition",
    "required_char_card": "CHAR_XU_DA",
    "required_event": "EVENT_NORTHERN_EXPEDITION",
    "min_skill": { "武艺": 3 }
  }
}
```

- `exclusive_to`：指向专属角色卡片ID。只有该角色可使用（玩家扮演该角色或获得其人物卡后习得）。
- `acquire`：复杂条件组合，需同时满足人物卡持有、事件触发、技能等级等。

**（5）称号卡**

```json
{
  "card_id": "TITLE_STRATEGIST",
  "type": "称号卡",
  "name": "策士",
  "effect": { "统率": 3 },
  "acquire": {
    "method": "battle_win_count",
    "count": 10
  }
}
```

- `effect`：佩戴后提供的属性加成。可同时佩戴多个称号？建议只允许佩戴一个主称号（效果可叠加但需设计平衡）。

**（6）名所卡**

```json
{
  "card_id": "PLACE_QINHUAI",
  "type": "名所卡",
  "name": "秦淮河",
  "location": "应天府",
  "description": "六朝金粉，十里秦淮。画舫凌波，笙歌彻夜。",
  "acquire": {
    "method": "visit",
    "city": "YINGTIAN",
    "specific_spot": "秦淮河畔"
  }
}
```

- 访问对应地点即可获得，是游戏中最简单的收集项。

**（7）事件卡**

### 四、卡片获取途径

| 获取方式 | 适用卡片类型 | 详细说明 |
|---------|-------------|---------|
| **初始拥有** | 部分基础武技、战术卡 | 如“太祖长拳”、“火攻”（通过教学关获得） |
| **师事修行** | 合战战术卡、个人战武技卡 | 向师父学习对应技能，通过小游戏后习得 |
| **战胜敌将** | 人物卡、秘传卡、战术卡 | 合战击败敌总大将或单挑胜利后，有几率获得其人物卡或习得其招式 |
| **亲密度达标** | 人物卡 | 与角色亲密度达到3心，交谈时对方赠予人物卡 |
| **完成主命/事件** | 事件卡、称号卡、宝物卡 | 推动历史剧情或完成特定主命后获得 |
| **访问名胜** | 名所卡 | 到达地图特定坐标点即可获得 |
| **购买/交易** | 宝物卡 | 在城镇市集或商帮会馆购买，或通过事件获得 |
| **达成成就** | 称号卡 | 如合战胜利10次获“策士”，个人战连胜20次获“乡勇” |
| **武举/科举** | 称号卡 | 通过乡试、会试、殿试获得对应称号 |
| **继承/传授** | 秘传卡 | 持有对应人物卡且满足条件后，通过切磋胜利习得 |


### 五、卡片收集图鉴系统

玩家在游戏中可随时打开“卡片图鉴”界面，查看已收集和未收集的卡片。未收集的卡片显示为黑影轮廓，鼠标悬停（或选中）可显示获取提示（`acquire_hint`），引导玩家。

**图鉴分类页签**：
- 人物（按势力或身份筛选）
- 战术·合战
- 武技·个人
- 秘传
- 称号
- 名所
- 事件
- 宝物

**收集率奖励**：当某一类卡片收集率达到特定百分比时，可解锁特殊奖励（如新的主人公、隐藏剧本、特殊宝物等），进一步激励收集。

**跨存档继承**：卡片图鉴数据独立于游戏存档，保存在玩家本地的`collection.json`文件中。每局游戏获得的卡片会实时更新到该文件。新游戏开始时，图鉴中已解锁的卡片效果（如人物卡可扮演）可直接使用。


### 六、卡片系统与各游戏系统的交互

卡片系统作为“奖励中枢”，与所有玩法模块紧密耦合。

| 游戏系统 | 如何产生卡片 | 如何消费/使用卡片 |
|---------|-------------|-----------------|
| **合战系统** | 胜利后可能获得敌将人物卡、战术卡；达成胜利次数获称号卡 | 读取玩家拥有的合战战术卡作为可选指令；阵法卡决定可用阵型 |
| **个人战系统** | 切磋胜利可习得对手武技/秘传；连胜获称号 | 读取玩家拥有的武技卡、秘传卡作为可选招式 |
| **修行系统** | 通过小游戏师事习得战术卡、武技卡 | 技能等级本身由属性存储，技能卡仅作收集证明 |
| **主命系统** | 完成任务奖励宝物卡、称号卡 | — |
| **社交系统** | 亲密度3心获人物卡 | 送礼时消耗宝物卡 |
| **经济系统** | 购买获得宝物卡 | 出售宝物换钱 |
| **探索系统** | 访问名胜获名所卡 | — |
| **剧情事件** | 触发事件获得事件卡 | 部分事件需要持有特定卡片才能触发 |
| **新游戏开局** | — | 已解锁的人物卡可用于选择扮演；称号卡可佩戴提供开局加成 |


### 七、实现建议（文字游戏向）

由于是单人文字游戏开发，可采取以下简化但高效的实现方式：

**1. 卡片数据存储**
将所有卡片定义在一个`cards.json`文件中，按类型分区。游戏启动时一次性加载到内存字典（`card_id` → `card_data`）。

**2. 收集状态管理**
用一个独立的`collection.json`文件记录已收集的`card_id`列表。结构如下：
```json
{
  "collected_cards": [
    "CHAR_ZHU_YUANZHANG",
    "CHAR_XU_DA",
    "TACTIC_FIRE_ATTACK",
    "MARTIAL_TAIZU_CHANGQUAN",
    "TITLE_STRATEGIST",
    "PLACE_QINHUAI"
  ],
  "collection_stats": {
    "total_collected": 126,
    "by_type": { "人物卡": 45, "战术卡_合战": 12, ... }
  }
}
```

**3. 卡片获取接口**
提供一个统一函数`acquire_card(card_id)`：
- 检查是否已拥有，若已有则忽略（或返回提示）。
- 将`card_id`加入`collected_cards`，更新统计。
- 根据卡片类型执行附加效果：如人物卡→解锁扮演；战术卡→加入玩家可用卡池（如果当前扮演角色）；称号卡→加入称号列表供佩戴。
- 触发图鉴更新事件。

**4. 卡片图鉴界面**
在文字界面中，用菜单形式展示：
```
═══════════════════════════════════════════
              卡片图鉴 · 人物卡
═══════════════════════════════════════════
已收集：45 / 300

1. 朱元璋 ★★★★★ （初始）
2. 徐达     ★★★★★ （已收集）
3. 常遇春   ★★★★★ （已收集）
4. 刘伯温   ★★★★★ （已收集）
5. 李善长   ★★★★   （已收集）
6. ？？？   ？？？   （未收集）
   └─提示：与刘伯温交谈时听说，此人正在浙东隐居……

请输入数字查看详情，或输入B返回。
═══════════════════════════════════════════
```

**5. 与战斗系统的集成**
合战/个人战开始时，从玩家存档的`owned_cards`中筛选出对应类型的卡片，构建当前可用卡组。秘策系统从`owned_cards`中读取已学会的所有战术卡供替换。

**6. 跨存档继承**
`collection.json`独立于存档，每次`acquire_card`时同步写入。新游戏开始时，读取该文件以决定哪些人物可选、哪些称号可佩戴。

### 九、总结

卡片系统是《洪武立志传》的**灵魂**。它将枯燥的数值成长转化为直观的收集乐趣，将零散的玩法模块统一在“收集卡片”这一共同目标之下。玩家每一次修行、每一场合战、每一段剧情，都在为这幅“明朝绘卷”增添新的卡片。

对单人开发者而言，卡片系统的另一大优势是**易于扩展**——你只需在`cards.json`中添加一条新记录，并为它设计一个获取途径，游戏内容便得到了丰富。无需修改核心代码，这也是“卡片驱动”架构的最大便利。

下一步，你可以着手编写`cards.json`的首批卡片数据（例如50张核心卡片），并实现`acquire_card`函数和简易图鉴界面。当第一张“徐达”人物卡被收集时，整个系统的框架便宣告跑通。
