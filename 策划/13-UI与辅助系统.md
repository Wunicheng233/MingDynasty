# 13 - UI与辅助系统

## 一、文字游戏UI布局

文字游戏采用**三段式布局**：

```
┌──────────────────────────────────────────────────┐
│  状态栏（常驻顶部）                                │
│  [洪武元年三月] 应天府 银两：320贯 体力：❤️❤️❤️❤️🤍 │
├──────────────────────────────────────────────────┤
│                                                  │
│                  主信息区                           │
│          场景描述、对话、战斗反馈...               │
│                                                  │
├──────────────────────────────────────────────────┤
│                  菜单区（底部）                     │
│          1. 选项一  2. 选项二  3. 选项三          │
└──────────────────────────────────────────────────┘
```

### 明朝风格装饰

- 用 `════` 双线做分隔，简洁大方
- 重要提示前加 `✦`
- 血条/进度条用 `■` 和 `□` 表示
- 标题可以用 `☯ 洪 武 立 志 传 ☯` 这样的装饰

## 二、交互规范

| 交互类型 | 规范 |
|----------|------|
| 菜单选择 | 输入数字，0 或 B 返回上级 |
| 确认操作 | 重要操作二次确认 (Y/N) |
| 数值输入 | 直接输入数字 |
| 长文本 | 分页显示，"按回车继续" |
| 全局快捷键 | S=存档，L=读档，H=帮助，Q=退出 |

## 三、辅助系统

### 3.1 存档读档

- 支持 **5个存档槽位**
- 每个存档显示：存档时间、玩家姓名、身份、游戏年份
- 自动存档：每季度初（1/4/7/10月1日）自动存到固定槽
- 存档数据结构：

```json
{
  "slot_id": 1,
  "save_time": "2025-04-08 22:15:30",
  "player_name": "朱元璋",
  "player_identity": "吴王",
  "game_year": 1367,
  "data": { /* 完整动态数据 */ }
}
```

### 3.2 设置系统

| 设置项 | 选项 | 说明 |
|--------|------|------|
| 文本滚动速度 | 快/中/慢 | 控制文本显示速度 |
| 战斗描述简化 | 开/关 | 开启后跳过详细描述，直接出结果 |
| 小游戏难度 | 简单/普通/困难 | 影响小游戏目标值 |
| 自动存档 | 开/关 | 是否季度自动存档 |

设置保存在 `config.json`，随游戏启动加载。

### 3.3 帮助系统

随时按 H 呼出**上下文帮助**，显示当前场景可用指令和玩法说明：

```
═══════════════════════════════════════════
              帮助 · 城镇指令
═══════════════════════════════════════════
· 在城镇中，你可以进入不同设施活动
· 评定间：每两月一次评定会，接受/发布主命
· 市集：买卖货物，投资城镇
· 武馆：修行武艺，切磋
· 酒馆：宴饮，打听情报
· 自宅：休息恢复体力，存档，家人互动
· 城门：移动到其他城镇（消耗天数）

输入任意键返回。
═══════════════════════════════════════════
```

### 3.4 成就系统（跨存档）

- 成就跨存档记录，存储在 `achievements.json`
- 完成成就奖励特殊卡片/称号

| 成就 | 条件 | 奖励 |
|------|------|------|
| 洪武开天 | 完成第一章主线 | 解锁自由模式所有人物 |
| 富可敌国 | 个人资产100万贯 | 解锁称号"豪商"（魅力+5）|
| 天下布武 | 收集全部秘传卡 | 解锁隐藏角色 |
| 游遍大明 | 收集全部名所卡 | 获得宝物"徐霞客游记"（移动消耗-10%）|
| 百战不殆 | 合战胜利100次 | 统率+5（永久）|

### 3.5 日志回顾

文字游戏容易滚太快漏掉信息，支持 **R 键回顾最近20条系统消息**。

## 五、文本与本地化管理

建议把所有文本（界面、事件、对话）从代码分离，存放在 `texts/zh_cn.json`：

```json
{
  "ui": {
    "title": "洪武立志传",
    "menu_back": "返回"
  },
  "events": {
    "E_MAIN_ZHU_TOUJUN": {
      "scene_1": "濠州城外，红巾军大营..."
    }
  }
}
```

好处：
1. 改文案不用碰代码
2. 方便以后多语言
3. 方便文案编辑协作

## 六、与太阁立志传4对比

| 太阁 | 洪武立志传 |
|------|------------|
| 图形界面 | 文字界面（可以封装为网页） |
| 多存档 | 多存档，一样 |
| 成就 | 成就，一样 |
| 帮助 | 上下文帮助，一样 |

适配文字游戏做了相应简化，更适合单人开发。
