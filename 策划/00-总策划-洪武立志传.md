# 《洪武立志传》完整游戏策划

## 项目概述

**《洪武立志传》** 是一个模仿《太阁立志传4》的文字游戏，以明朝初年洪武开国为历史背景，玩家可以扮演朱元璋及其他明初历史人物，体验从乞丐到皇帝的传奇人生。游戏继承了太阁系列"卡片驱动、高自由度、多路线生涯"的核心设计，并深度融入明朝历史特色。

### 核心理念

- **卡片驱动**：一切可收集内容（人物、技能、战术、称号、名所）均以卡片形式存在
- **评定循环**：每两个月一次评定会，接受主命→执行→结算功勋→身份晋升
- **历史沉浸**：所有玩法系统都融入明朝特色元素，让玩家亲历洪武开国
- **高自由度**：支持武将、文官、商人、侠客、锦衣卫等多种人生路线

### 目标玩家

- 喜欢《太阁立志传》系列的历史爱好者
- 对明朝开国历史感兴趣的玩家
- 喜欢文字冒险和策略收集的玩家

## 一、顶层系统架构

### 1.1 三层架构

游戏采用 **"核心数据层 → 系统模块层 → 玩法表现层"** 三层架构：

```
┌─────────────────────────────────────────────────────────────┐
│                      玩法表现层                              │
│   （文字界面：评定会、城镇场景、战斗界面、事件对话）          │
└─────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                      系统模块层                              │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌───────┐ │
│  │ 合战   │ │ 个人战  │ │ 主命   │ │ 社交   │ │ 经济  │ │
│  └────┬────┘ └────┬────┘ └────┬────┘ └────┬────┘ └───┬───┘ │
│       └───────────┴───────────┴───────────┴──────────┘     │
│                           ▼                                  │
│                    ┌──────────────┐                          │
│                    │ 卡片收集系统 │（贯穿所有模块）            │
│                    └──────────────┘                          │
└─────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                      核心数据层                              │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐         │
│  │角色属性数据  │ │世界状态数据  │ │卡片库数据    │         │
│  │(玩家/NPC)    │ │(时间/城镇/   │ │(600+张卡片)  │         │
│  └──────────────┘ └──────────────┘ └──────────────┘         │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 设计原则

- **卡片驱动**：一切可收集内容均卡片化，以 `card_id` 作为全局唯一标识
- **评定循环**：每两月一次评定会是游戏节奏控制器
- **数据解耦**：各系统通过核心数据层交互，避免直接相互调用
- **状态与定义分离**：静态定义与动态状态分开存储，便于存档扩展

### 1.3 核心游戏循环（以家臣阶段为例）

```
┌─────────────────────────────────────────────────────────────┐
│                    评定会（每两月）                          │
│  接受主命：攻城、练兵、开垦、外交、缉捕……                    │
└──────────────────────────┬──────────────────────────────────┘
                           │ 选择主命
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                    执行主命阶段                              │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐    │
│  │军务主命  │  │政务主命  │  │外交主命  │  │特殊主命  │    │
│  │→合战     │  │→小游戏   │  │→口才判定 │  │→个人战/  │    │
│  │→兵力消耗 │  │→技能经验 │  │→亲密度   │  │  小游戏   │    │
│  └────┬─────┘  └────┬─────┘  └────┬─────┘  └────┬─────┘    │
│       └─────────────┴───────────┴─────────────┘          │
│                             ▼                                │
│              功勋结算 / 技能经验 / 卡片获取                   │
└──────────────────────────┬──────────────────────────────────┘
                           │ 返回评定会
                           ▼
┌─────────────────────────────────────────────────────────────┐
│              评定会（功勋累积，身份晋升）                     │
│  放牛娃→士兵→小旗→总旗→百户→千户→指挥使→都督→五军都督       │
└─────────────────────────────────────────────────────────────┘
                           │
                           ▼
              触发历史事件（鄱阳湖、北伐等）
                           │
                           ▼
              第一章通关 → 解锁大量人物卡 → 自由沙盒模式
```

## 二、核心数据层设计

### 2.1 设计原则

1. **卡片驱动**：所有可收集内容统一为卡片结构，`card_id` 全局唯一
2. **最小冗余**：能通过ID引用绝不复制完整数据
3. **状态与定义分离**：静态数据（游戏内置）与动态数据（玩家存档）分离
4. **版本兼容**：预留 `version` 字段便于存档迁移
5. **可读性优先**：字段命名统一，注释详尽

### 2.2 数据表结构总览

```
┌────────────────────────────────────────────────────────────────┐
│                        核心数据层                               │
├────────────────────────────────────────────────────────────────┤
│  【静态库】（游戏内置，不随存档改变）                             │
│  ├── 角色模板库（char_templates）     所有NPC的基础属性/技能      │
│  ├── 卡片库（card_library）           600+卡片定义               │
│  ├── 城镇定义库（city_defs）          各城镇初始状态              │
│  ├── 势力定义库（faction_defs）       历史势力初始数据            │
│  ├── 技能定义库（skill_defs）         16项技能及效果              │
│  ├── 主命模板库（mission_templates）  所有主命的参数              │
│  └── 事件库（event_library）          剧情触发条件与对话           │
│                                                                  │
│  【动态存档】（玩家存档文件）                                     │
│  ├── 玩家角色数据（player）            当前能力/技能/卡片/称号     │
│  ├── NPC实例数据（npcs）              已生成NPC的当前状态          │
│  ├── 势力实例数据（factions）         当前各方势力状态             │
│  ├── 城镇实例数据（cities）           各城镇当前属性               │
│  ├── 关系网络（relationships）        角色间亲密度                 │
│  ├── 全局时间（global_time）          年/月/日                    │
│  └── 事件标记（event_flags）          已触发事件、开关             │
└────────────────────────────────────────────────────────────────┘
```

### 3.2 十六项技能

游戏共有 **16项技能**，每项技能分 **Lv0-Lv3** 四个等级：

1. **步战** - 步兵统领，影响合战步兵能力
2. **骑战** - 骑兵指挥，影响合战骑兵能力、减少移动消耗
3. **水战** - 水军指挥，影响合战水战能力
4. **火器** - 火器使用，影响神机营火器伤害
5. **兵法** - 军事谋略，影响合战秘策次数、阵型解锁
6. **武艺** - 个人武艺，影响个人战手牌数和伤害
7. **医术** - 医药知识，个人战恢复技、治疗伤病
8. **口才** - 辩说游说，影响劝降、社交、主命
9. **农政** - 农业开垦，影响开垦新田主命
10. **工政** - 工程建设，影响筑城、火器制造
11. **商政** - 商业算术，影响贸易、物价判断
12. **律政** - 法律知识，影响断案主命
13. **密探** - 潜入侦察，影响情报收集、破坏
14. **航海** - 航海技能，影响海外贸易、下西洋
15. **礼制** - 礼仪法度，影响外交、科举
16. **文墨** - 文化书法，影响科举、著书

## 五、体力系统

### 5.1 基础定义

- **当前体力**：0 ~ 最大体力，归零则昏迷
- **最大体力** = 基础体力 + 称号加成 + 宝物加成
- **基础体力** = 60 + (武力 × 0.8)

示例：常遇春武力100 → 基础体力 = 60 + 80 = 140

### 5.2 体力状态层级

| 体力百分比 | 状态 | 效果 |
|-----------|------|------|
| 100% ~ 75% | 精力充沛 | 无负面影响，修行效率 +10% |
| 75% ~ 40% | 略有倦意 | 移动速度 -10% |
| 40% ~ 20% | 疲惫不堪 | 移动-20%，经验-30%，个人战手牌-1 |
| 20% ~ 5% | 体力透支 | 无法修行出征，个人战武力-20% |
| < 5% | 危在旦夕 | 无法执行多数指令，10%/天概率昏迷 |

**生病状态**：独立异常状态，体力上限-30%，自然恢复停止，必须医治。

### 5.3 消耗规则

| 行动 | 基础消耗 | 说明 |
|------|---------|------|
| 城镇内移动 | 0 | 城内自由移动不耗 |
| 修行（5天） | 8 ~ 12 | Lv越高消耗越多 |
| 执行主命（全程） | 10 ~ 30 | 难度决定 |
| 跨省移动 | 5 | 骑战每级减1，最低2 |
| 武馆切磋 | 战后损失×50% | 实际扣除战斗损失的一半 |

### 5.4 恢复规则

| 方式 | 恢复量 | 消耗 | 限制 |
|------|-------|------|------|
| 旅店住宿 | 恢复满 | 50文×恢复百分比 | 消耗5天，生病不能全恢复 |
| 自宅休养 | 恢复满 | 无 | 需拥有自宅，消耗5天 |
| 医馆治疗 | 恢复满+解除生病 | 200文 | 消耗3天 |
| 使用药品 | 金疮药+30，续命丹+50 | 消耗物品 | 任何时候可用 |
| 与妻子互动 | +15 | 无 | 每天1次 |
| 自然恢复 | +2/天 | 无 | 非生病状态 |

### 5.5 伤病死亡机制

- **体力归零**：城镇内被救起，消耗7天恢复到20%；野外孤身有30%概率直接死亡
- **久病不愈**：生病超过30天，最大体力永久-5
- **角色死亡**：可由子嗣/结义兄弟继承继续游戏，无继承人则游戏结束

### 5.6 明朝特色设计

- **勤政过劳**：君主连续处理政务20+天，累积过劳值，满了触发"勤政猝晕"强制休息
- **丹药与丹毒**：丹药恢复体力但累积丹毒，过高会导致最大体力下降甚至暴毙
- **缠足与女性体力**：女性角色初始体力上限低10%，可通过事件"放足"改善

## 六、时间与移动系统

### 6.1 时间单位

- **天**：基本时间单位，绝大多数行动消耗天数
- **月**：每月30天，一年12个月，**奇数月1日自动召开评定会**
- **年**：记录历史年份和角色年龄

**纪年显示**：同时显示年号纪年和公历，如"洪武元年（1368年）正月"。

### 6.2 行动耗时规则

| 行动 | 基础耗时 | 说明 |
|------|---------|------|
| 相邻城镇移动 | 2~5天 | 距离+技能+天气影响 |
| 师事修行 | 5天/次 | 固定 |
| 执行主命 | 15~30天 | 任务决定 |
| 送礼/切磋/茶会 | 1天 | 固定 |
| 小游戏 | 1天 | 固定 |
| 个人战 | 0天 | 剧情战败北可能消耗天数 |
| 合战 | 1~5天 | 规模决定 |
| 城镇内切换设施 | 0天 | 不耗时间 |

**移动时间计算**：基础天数 × (1 - 骑战技能等级×10%)，冬季+20%。

### 6.3 城镇移动网络

城镇之间通过预设路线连接，每对城镇有基础移动天数。玩家选择目的地后，时间流逝对应天数，到达后进入目标城镇。

### 6.4 时间驱动事件

- **定期事件**：评定会（奇数月1日）、新年（1月1日）、科举（每年八月）
- **随机事件**：移动/停留时概率触发，如遭遇山贼、行商叫卖、路遇难民
- **历史剧情事件**：按日期自动触发，如郭子兴病逝、鄱阳湖之战

**优先级**：主线事件 > 支线事件 > 定期事件 > 随机事件

## 八、主命与评定系统

### 8.1 评定会框架

- **周期**：每隔两个月（奇数月1日）召开
- **缺席惩罚**：扣除功勋，多次缺席可能被驱逐
- **流程**：主公发布主命 → 玩家选择 → 执行 → 复命结算功勋

### 8.2 身份晋升（家臣阶段）

| 总旗 | 600 | 500 | 正七品 |
| 百户 | 1400 | 1000 | 正六品 |
| 千户 | 3000 | 2000 | 正五品 |
| 指挥佥事 | 5000 | 3000 | 正四品 |
| 指挥使 | 12000 | 5000 | 正三品 |
| 五军都督 | 35000 | 8000 | 正一品 |

**晋升城主**：功勋达到指挥使以上，受封城池后进入君主阶段。

### 8.3 主命类型（明朝特色）

**军务类**：
- 征兵 → 口才小游戏
- 训练 → 合战模拟
- 侦查敌城 → 密探小游戏
- 出征/攻城 → 触发合战

**政务类**：
- 征收赋税 → 算术
- 开垦新田 → 农政小游戏
- 修葺城墙 → 工政小游戏
- 审理案件 → 律政小游戏

**外交类**：
- 出使友邦 → 礼制+口才
- 劝降敌将 → 口才对决
- 进贡朝廷 → 礼制

**调略类**：
- 人才探访 → 寻找浪人
- 破坏敌城 → 密探潜入
- 流言离间 → 降低敌城士气

**特殊类**：
- 缉拿要犯 → 触发个人战
- 医治伤兵 → 医术配药
- 督建皇陵 → 工政+魅力

### 8.4 君主阶段（城主/国主/皇帝）

当玩家晋升为城主后，从**"接受任务"**变为**"发布任务"**：

- 可向家臣发布各类主命
- 管理城池军资金和税收
- 晋升家臣身份
- 成为皇帝后，按六部职能分流主命：
  - 吏部 → 人事考核
  - 户部 → 征税赈灾
  - 礼部 → 科举祭祀
  - 兵部 → 征兵出征
  - 刑部 → 断案修订律法
  - 工部 → 筑城水利

**明朝特色**：
- **科举武举**：定期举办，选拔人才
- **考成法**：主命有完成期限，政绩考核决定升降
- **开中法盐引**：商人路线专属任务

## 十、合战系统

### 10.1 基础机制

**触发场景**：执行军务主命、历史战役、敌方入侵。

**参战**：总大将+最多4名副将，每将带一支部队，可配置不同兵种。

**回合流程**（最多5轮）：
1. 手牌调整：可弃3张补到5张
2. 秘策发动：可换一张手牌为已学会的战术卡（兵法等级决定次数）
3. 出牌：最多3张基础卡 + 最多1张红色战术卡
4. 判定：比较组合强弱
5. 结算：计算伤害、士气、城防

**胜负**：野战/水战 - 敌方总大将兵力归零；攻城战 - 城防归零或敌方总大将败退。

### 10.2 兵种体系

| 兵种 | 特点 | 克制 | 受技能加成 |
|------|------|------|-----------|
| **步兵** | 基础防御稳定 | 被骑克，克弓 | 步战 |
| **骑兵** | 高机动高攻击 | 克步兵，被火器/弓克 | 骑战 |
| **弓兵** | 远程先手削弱 | 克骑兵，被步兵克 | 弓术 |
| **火器兵** | 远程高伤，雨天减半 | 克骑/步兵 | 火器 |
| **水军** | 水战专用 | - | 水战 |

**兵种相克**：克制方伤害×1.3，被克方×0.7。

### 10.3 卡片与组合

**基础卡组合强弱**：三同张（1.8倍） > 三连张（1.5倍） > 二同张（1.2倍） > 无组合（1.0倍）。

**数字大小影响**：三同张数字越小越强（111最强），三连张起始越小越强（123最强）。

**红色战术卡**（部分明朝特色）：
- **火攻** - 伤害×1.5，攻城城防-3（朱元璋火烧陈友谅）
- **三段击** - 火器3连击（沐英首创，明朝独有）
- **鸳鸯阵** - 火器步兵协同（戚继光）
- **火船突击** - 水战对敌船伤害×2.5（鄱阳湖之战）
- **高垒深沟** - 守城防御×1.5（朱元璋"高筑墙"）

### 10.4 阵型系统

战前选择阵型，阵型存在相克循环：

| 阵型 | 效果 | 习得条件 |
|------|------|---------|
| 鱼鳞阵 | 攻击+10% | 步战Lv1 |
| 雁行阵 | 弓/火器+20% | 兵法Lv1 |
| 锋矢阵 | 骑兵+25% | 骑战Lv2 |
| 衡轭阵 | 对骑兵+30% | 步战Lv3 |
| 鸳鸯阵 | 火器步兵协同攻防+10% | 火器Lv3+兵法Lv3 |
| 车悬阵 | 士气每回合+5，攻击+15%，克所有 | 击败徐达习得 |

**相克循环**：鱼鳞 → 雁行 → 锋矢 → 衡轭 → 方圆 → 长蛇 → 偃月 → 鹤翼 → 鱼鳞。克制方士气+10，被克方-10。

### 10.5 水战特殊规则

- 仅水军、弓兵、火器可用
- 火攻伤害×1.5
- 风向影响：上风方火攻伤害+20%
- 铁索连环（陈友谅专属）：防御+30%，但火攻伤害×2.5（还原历史）

### 10.6 历史战役还原

**鄱阳湖之战（1363）**：
- 陈友谅军铁索连环，火攻伤害×2.5
- 我方剧情获得"鄱阳湖火攻"特殊卡，伤害×3
- 风向影响关键，还原历史上火攻东风

**北伐大都（1368）**：
- 元军骑兵多，需用衡轭阵克制
- 攻克大都后触发明朝建立剧情

## 十二、社交与关系系统

### 12.1 亲密度体系

亲密度范围 **-4（死敌） ~ +5（生死相随）**：

| 心级 | 可解锁内容 |
|------|-----------|
| 0（中立） | 初次见面 |
| 1心（一面之缘） | 基础社交 |
| 2心（略有交情） | **可拜师修行、可劝诱** |
| 3心（推心置腹） | **获得人物卡、可结义** |
| 4心（莫逆之交） | 劝诱成功率极高 |
| 5心（生死相随） | 永不叛变（配偶/结义） |

### 12.2 互动方式

每次互动消耗1天：

1. **送礼**：赠送宝物卡，根据价值和喜好提升亲密度
2. **茶会/宴饮**：消耗金钱，检定礼制/口才，成功提升亲密度
3. **切磋**：个人战，胜负都增加亲密度，胜利更多
4. **交谈**：打听情报、请求指导、劝诱
5. **结义/婚姻**：4心以上可进行，亲密度锁定为5心

### 12.3 明朝特色关系网

- **淮西集团**：同乡之间天然亲密度+20
- **师生同年**：科举形成的关系网络，官场相互提携
- **姻亲政治**：婚姻结成政治联盟（朱元璋娶马秀英获得郭子兴信任）
- **厂卫监视**：得罪锦衣卫后，与某些官员交友受阻

### 12.4 婚姻与结义

**婚姻**：
- 条件：异性，4心以上，玩家身份千户以上
- 效果：亲密度锁定5心，配偶移居自宅，可互动恢复体力，偶尔赠礼

**结义**：
- 条件：同性，4心以上，最多2个结义兄弟
- 效果：合战同队士气+10，永不背叛

## 十三、经济系统

### 13.1 货币体系

| 单位 | 换算 | 用途 |
|------|------|------|
| **文** | 基本单位 | 小额交易 |
| **贯** | 1贯 = 1000文 | 日常买卖、军资金 |
| **两（银）** | 1两 ≈ 1.5贯 | 贵重宝物、大宗贸易 |

### 13.2 物价波动

物价受以下因素影响：
- 丰收年 → 粮价下跌，灾年 → 上涨
- 战乱 → 所有物价+20%
- 季节 → 秋收后粮价最低，春荒最高
- 商路阻断 → 特产价格飙升

### 13.3 基本交易与投资

- **市集交易**：低买高卖赚取差价，商人路线初期核心
- **城镇投资**：投入金钱提升投资等级，解锁高级特产，降低物价波动

### 13.4 明朝特色商贸

**开中法·盐引贸易**：
- 向边关运送粮食 → 换取盐引 → 到产盐区支取食盐 → 内地销售
- 暴利但需要本钱和时间，商人路线核心玩法

**海禁与走私**：
- 合法海商需市舶司许可，风险低利润低
- 走私风险高（被查获重罚）但利润极高

**宝钞通货膨胀**：
- 洪武中后期宝钞购买力逐年下降，玩家需及时兑换白银保值，增加策略深度

**商帮会馆**：
- 加入商帮可获取精准物价情报、借贷、承接专属主命
- 可晋升至会首，掌控商业网络

### 13.5 身份与经济权限

| 身份 | 可从事活动 |
|------|-----------|
| 平民 | 市集倒卖、打工 |
| 家臣（百户以下） | 领取俸禄、执行筹措主命 |
| 家臣（千户以上） | 领地税收、参与开中法 |
| 城主/国主 | 管理金库、调整税率、投资城镇 |
| 君主 | 全国财政、决定赋税政策、发行宝钞 |

## 十四、剧情与事件系统

### 14.1 事件类型

| 类型 | 说明 | 触发 | 可重复 |
|------|------|------|--------|
| **主线事件** | 驱动朱元璋从乞丐到皇帝的核心叙事 | 时间+条件自动 | 否 |
| **支线事件** | 特定角色/地点的剧情，不影响主线 | 条件触发 | 部分可 |
| **随机事件** | 移动停留时概率触发，增加生动性 | 概率 | 是 |
| **定期事件** | 固定日期发生 | 时间自动 | 每年重复 |

### 14.2 触发条件支持

- 时间条件：当前日期 ≥ XXXX年XX月XX日
- 地点条件：玩家位于特定城镇
- 身份条件：玩家特定身份
- 亲密度条件：与某角色亲密度达标
- 卡片条件：持有特定卡片
- 技能条件：技能等级达标
- 事件标记：特定事件已触发

**优先级**：主线 > 支线 > 定期 > 随机。

## 十六、开发路线图

### 16.1 开发顺序建议（由核心到外围）

1. **核心数据层**（1周）- JSON结构设计，存档读档
2. **卡片系统**（1周）- 卡片定义、收集、图鉴
3. **时间与移动系统**（1周）- 时间推进、城镇移动
4. **评定会与主命框架**（2周）- 评定会界面、任务生成、功勋结算
5. **小游戏系统**（2-3周）- 优先实现5个核心小游戏，搭建框架
6. **个人战系统**（2周）- 数字组合、特殊技、秘传卡
7. **合战系统**（3周）- 兵种、阵型、战术卡、水战特殊规则
8. **社交与经济系统**（2周）- 亲密度、送礼、交易、投资
9. **剧情与事件系统**（2-3周）- 事件调度、条件检测、第一章主线

### 16.2 MVP 最小可行版本范围

建议先完成以下内容即可体验核心乐趣：
- 朱元璋第一章主线
- 5项技能修行 + 对应小游戏
- 个人战基础机制
- 合战简化版（仅野战，5种战术卡）
- 评定会与三类主命

MVP跑通后，再逐步丰富内容。

## 结束语

《洪武立志传》在继承《太阁立志传4》"卡片驱动、评定循环、高自由度"核心精髓的基础上，通过全面本土化改造，将整个游戏放置在**明朝初年洪武开国**这一波澜壮阔的历史舞台上。

玩家既可亲历朱元璋从乞丐到皇帝的传奇一生，也可在自由模式中选择扮演徐达、常遇春、刘伯温、陈友谅等不同角色，走出属于自己的洪武传奇。

这份策划整合了所有分散内容，形成了一个结构清晰、逻辑完整、细节丰富的设计文档，可供开发按图索骥，逐步实现。
