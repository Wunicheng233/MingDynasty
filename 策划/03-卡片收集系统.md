# 03 - 卡片收集系统（太阁核心）

## 一、设计理念（完全继承太阁4）

**"一切皆卡片"** —— 这是太阁立志传系列的灵魂。在《洪武立志传》中：

- 学会一招太祖长拳 → 一张卡片
- 收服一员名将 → 一张人物卡
- 游览秦淮河 → 一张名所卡
- 达成百战百胜 → 一张称号卡

所有收集乐趣，全部通过卡片呈现。**卡片图鉴跨存档继承**，这是多周目游玩的核心动力。

### 核心作用

1. **收集成就感**：648张卡片构成终极收集目标
2. **玩法解锁器**：许多内容需要先获得卡片才能体验（人物卡解锁可扮演）
3. **跨存档继承**：收集的卡片永久保留，新游戏直接使用

## 二、卡片类型与数量规划

| 类型 | 数量 | 作用说明 |
|------|------|---------|
| **人物卡** | ~300 | 解锁可扮演角色，自由模式可选 |
| **技能卡** | 48 | 16技能×3等级，收集图鉴用 |
| **合战战术卡** | ~60 | 合战中可使用的特殊战术 |
| **个人战武技卡** | ~40 | 个人战中可使用的特殊武技 |
| **秘传卡** | ~30 | 名将专属金卡，个人战最强招式 |
| **称号卡** | ~50 | 达成成就，佩戴提供属性加成 |
| **名所卡** | ~30 | 游览名胜获得，收集解锁奖励 |
| **事件卡** | ~40 | 记录重大历史事件 |
| **宝物卡** | ~50 | 贵重物品，可装备送礼 |
| **总计** | **648** | 支撑多周目收集乐趣 |

## 三、卡片数据结构

### 3.1 基础字段（所有卡片共有）

```json
{
  "card_id": "CHAR_XU_DA",
  "type": "人物卡",
  "name": "徐达",
  "rarity": 5,
  "description": "明朝开国第一功臣，用兵如神，誉为万里长城。",
  "acquire_hint": "与徐达亲密度达到3心，或完成北伐剧情。",
  "is_hidden": false,
  "unlock_condition": null
}
```

**命名规范**：`类型缩写_内容`，例如：
- `CHAR_*` 人物卡
- `TACTIC_*` 合战战术卡
- `MARTIAL_*` 个人战武技卡
- `SECRET_*` 秘传卡
- `TITLE_*` 称号卡
- `PLACE_*` 名所卡
- `EVENT_*` 事件卡
- `TREASURE_*` 宝物卡

## 四、卡片获取途径

| 获取方式 | 适用卡片 | 说明 |
|---------|---------|---------|
| 初始拥有 | 基础武技/战术 | 新游戏开局获得 |
| 师事修行 | 战术卡/武技卡 | 小游戏完胜解锁 |
| 战胜敌将 | 人物卡/秘传卡 | 合战击败/单挑胜利 |
| 亲密度达标 | 人物卡 | 3心亲密度赠予 |
| 完成主命/事件 | 事件卡/称号卡 | 推动剧情获得 |
| 访问名胜 | 名所卡 | 到达即获得 |
| 购买/交易 | 宝物卡 | 市集购买 |
| 达成成就 | 称号卡 | 满足条件自动获得 |
| 继承传授 | 秘传卡 | 持有对应人物卡，满足条件切磋习得 |

## 五、卡片图鉴系统（借鉴太阁4）

### 5.1 图鉴界面

- 按类型分页显示（人物、战术、武技、秘传、称号、名所...）
- 未收集卡片显示黑影轮廓，只显示获取提示（`acquire_hint`）
- 已收集卡片显示完整信息和立绘（文字游戏可使用ASCII艺术或文字描述）

### 5.2 收集率奖励

| 收集率 | 奖励 |
|--------|------|
| 25% | 解锁1个隐藏人物 |
| 50% | 解锁称号"收集家"（魅力+3）|
| 75% | 解锁宝物"徐霞客游记"（移动不耗体力-10%）|
| 100% | 解锁隐藏剧本"靖难之变" |

### 5.3 跨存档继承

- 卡片收集状态存储在独立的 `collection.json` 中
- 不随存档删除而丢失
- 新游戏开局即可使用已解锁的人物卡

## 八、实现建议（文字游戏）

1. **卡片数据存储**：所有卡片定义放一个 `cards.json`，按类型分区
2. **收集状态管理**：`collection.json` 记录已收集 `card_id` 列表，独立于存档
3. **统一获取接口**：
   ```python
   def acquire_card(card_id):
       if card_id not in collected:
           collected.add(card_id)
           update_stats()
           # 根据类型执行解锁效果
   ```
4. **跳过机制**：不喜欢小游戏的玩家，技能达到Lv3后可跳过小游戏，但只能小胜，不能解锁卡片（保留"完胜解锁卡片"激励）
