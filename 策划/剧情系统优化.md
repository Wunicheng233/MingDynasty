# 剧情系统优化策划

## 现状分析

### 当前架构优势
- ✅ 数据驱动设计：所有事件定义在 `data/events.js`，新增事件无需修改系统代码
- ✅ 条件检查完备：支持时间、地点、身份、势力、标记、亲密度、技能等多种触发条件
- ✅ 效果执行模块化：每种效果独立处理，易于扩展

### 当前存在的耦合问题
1. **CHANGE_IDENTITY/CHANGE_FACTION 效果bug已修复**：之前错误修改静态模板，现在正确存储在 `GameState`
2. **奖励与其他系统耦合不足**：现有效果类型覆盖不全，部分系统无法通过剧情触发变化
3. **城市/势力经济互动不够**：剧情无法直接修改城池所有权、经济参数
4. **技能卡片解锁时机不够灵活**：现有 ADD_CARD 但缺乏条件解锁
5. **主命系统与剧情联动不够**：剧情选择无法开放/封锁特定主命分类

## 优化目标

让剧情系统成为真正的**全局事件驱动枢纽**，能够：
- 影响**所有**其他游戏系统
- 通过剧情选择解锁/封锁内容
- 给玩家带来真正的分支剧情体验
- 保持数据驱动设计，便于策划新增内容

### 效果参数设计

```javascript
// 1. 改变城池所有权
{
  type: 'change_city_owner',
  cityId: 2,      // 城池ID
  factionId: 1    // 新势力ID
}

// 2. 解锁城市设施
{
  type: 'unlock_facility',
  cityId: 2,      // 城池ID（可选，不填则给所有城池添加）
  facility: 'academy'  // 设施ID
}

// 3. 直接提升技能等级
{
  type: 'add_skill_level',
  skillId: 'agriculture',  // 技能ID
  levels: 1                 // 提升等级数
}

// 4. 开放主命任务分类
{
  type: 'unlock_mission_category',
  category: '内政'  // 分类名称
}

// 5. 增加势力声望
{
  type: 'add_faction_reputation',
  factionId: 1,   // 势力ID
  value: 10       // 声望增量
}

// 6. 更换玩家扮演角色
{
  type: 'set_player_character',
  characterId: 3   // 角色ID
}
```

## 与其他系统耦合设计

### 1. 与主命系统耦合
- **剧情开放主命**：完成特定剧情后，解锁对应分类的主命任务
- **剧情封锁主命**：选择特定分支后，封锁某些分类的主命
- **剧情给予功勋**：剧情关键选择直接给予功勋，加速身份晋升
- **外部完成机制**：主命任务要求玩家触发特定剧情，剧情完成后主命自动结算

### 2. 与设施系统耦合
- **剧情解锁设施**：完成剧情后，特定城池解锁新设施（如开科举后解锁国子监）
- **设施触发剧情**：进入特定设施有概率触发对应剧情（如酒馆随机剧情）
- **设施等级提升**：剧情可以提升设施等级，增加产出

### 3. 与技能系统耦合
- **剧情传授技能**：关键人物传授技能，直接提升等级
- **剧情给予经验**：完成剧情任务给予大量技能经验
- **技能条件触发**：特定技能等级才能触发某些剧情，获得更好奖励

### 4. 与社交系统耦合
- **剧情改变亲密度**：选择不同选项改变对不同NPC的亲密度
- **亲密度触发剧情**：亲密度达到一定值触发特殊剧情（如结义、求婚）
- **剧情解锁联姻**：满足条件后触发联姻剧情，解锁配偶buff

### 5. 与经济系统耦合
- **剧情改变城市经济**：剧情完成后提升城市繁荣度
- **税收增减**：政策选择改变税收率，影响后续收入
- **剧情给予金钱**：关键选择给予启动资金

### 6. 与卡片系统耦合
- **剧情解锁特殊卡片**：完成特定剧情解锁称号卡/事件卡/宝物卡
- **卡片条件触发剧情**：持有特定卡片才能触发隐藏剧情
- **剧情卡片跨存档继承**：事件卡片解锁后保持在收集进度中

### 7. 与合战系统耦合
- **剧情触发关键合战**：主线剧情关键节点触发历史著名战役（如采石矶之战）
- **合战结果影响剧情**：合战胜利/失败进入不同分支
- **剧情改变城池归属**：合战后通过剧情确认占领成果

## 数据驱动优势保持

所有扩展仍然保持**数据驱动**设计：
- 新增效果只需要在 `EventScheduler.executeEffect` 添加一个 case
- 新增条件只需要在 `EventScheduler.checkConditions` 添加一个检查
- 策划新增剧情只需要修改 `data/events.js`，不需要动系统代码

## 预期收益

1. **剧情真正影响游戏**：不同选择真正改变后续玩法，而不只是文字变化
2. **系统耦合更紧密**：剧情→系统→玩法形成闭环
3. **扩展性更强**：后续新增剧情只需要添加数据，不需要修改系统
4. **多结局多分支**：真正支持不同开局、不同选择、不同结局
