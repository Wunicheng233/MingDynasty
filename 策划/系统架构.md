## 《洪武立志传》完整系统框架设计

你的直觉很准——合战和个人战绝非孤立系统，它们与角色成长、卡片收集、任务评定、经济社交等模块深度交织。以下为完整的系统框架设计，兼顾“太阁式”高自由度与单人文字游戏的实现可行性。


### 🧩 一、顶层系统架构

游戏采用 **“核心数据层 → 系统模块层 → 玩法表现层”** 三层架构：

```
┌─────────────────────────────────────────────────────────────┐
│                      玩法表现层                              │
│   （文字界面：评定会、城镇场景、战斗界面、事件对话）          │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      系统模块层                              │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌───────┐ │
│  │合战系统 │ │个人战   │ │任务主命 │ │社交系统 │ │经济   │ │
│  │         │ │系统     │ │系统     │ │         │ │系统   │ │
│  └────┬────┘ └────┬────┘ └────┬────┘ └────┬────┘ └───┬───┘ │
│       └───────────┴───────┬────┴───────────┴──────────┘     │
│                           ▼                                  │
│                    ┌──────────────┐                          │
│                    │卡片收集系统  │（贯穿所有模块）            │
│                    └──────────────┘                          │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      核心数据层                              │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐         │
│  │角色属性数据  │ │世界状态数据  │ │卡片库数据    │         │
│  │(玩家/NPC)    │ │(时间/城镇/   │ │(600+张卡片)  │         │
│  │              │ │ 势力/关系)   │ │              │         │
│  └──────────────┘ └──────────────┘ └──────────────┘         │
└─────────────────────────────────────────────────────────────┘
```

**核心设计原则**：
- **卡片驱动**：一切可收集的内容（人物、技能、战术、称号、名所）均为“卡片”，统一数据结构
- **评定循环**：每两月一次的评定会是主要玩法推进器
- **数据解耦**：各系统通过核心数据层交互，避免直接相互调用

**（三）卡片库数据**

统一管理所有600+张卡片的定义，各系统通过`card_id`引用。卡片分为：
- **人物卡**：解锁可扮演角色
- **技能卡**：记录16项技能的等级（实际以角色技能数值为准，卡片作收集图鉴）
- **合战战术卡**：红色特殊效果卡
- **个人战武技卡**：红色特殊技卡
- **秘传卡**：名将专属金卡
- **名所卡**：名胜古迹
- **称号卡**：达成条件后获得
- **事件卡**：记录重大历史事件触发


### ⚙️ 三、系统模块层——各司其职又紧密联动

#### 1. 合战系统

**核心职责**：处理野战/攻城/水战的回合制卡片对战。

**输入数据**：
- 参战武将的属性（统率、武力、技能等级）
- 已习得的合战战术卡（从角色`owned_cards`读取）
- 当前阵型（从已解锁阵型中选择）
- 兵力、士气、城防等战场状态

**输出/影响**：
- 战斗结果 → 更新世界状态（势力城池归属、角色生死）
- 胜利次数 → 触发称号系统（“策士”→“军神”）
- 特定胜利 → 解锁秘传卡、人物卡（如击败徐达获得“车悬阵”）
- 功勋值 → 影响评定会晋升

**与其他系统的耦合点**：

| 关联系统 | 耦合方式 | 数据流向 |
|---------|---------|---------|
| **技能系统** | 步战/骑战/火器等技能等级决定可用阵型、秘策次数、战术卡解锁 | 技能 → 合战 |
| **卡片系统** | 合战战术卡作为卡片存储；胜利可获得新战术卡、人物卡 | 合战 → 卡片 |
| **任务主命** | 军务主命（攻城、练兵）触发合战；合战结果反馈任务完成度 | 任务 ↔ 合战 |
| **社交系统** | 与武将一同出征可提升亲密度；劝降敌将需口才技能 | 合战 → 社交 |
| **经济系统** | 出征消耗军资金、兵粮；战胜获得战利品 | 经济 ↔ 合战 |
| **称号系统** | 合战胜利累计次数授予称号；称号提供统率加成反哺合战 | 合战 ↔ 称号 |
| **世界状态** | 合战改变城池归属，影响势力版图 | 合战 → 世界 |

#### 2. 个人战系统

**核心职责**：处理单挑、切磋、武举考核的卡片对战。

**输入数据**：
- 角色武力、武艺/医术/密探技能等级
- 已习得的个人战武技卡、秘传卡
- 当前体力（由武力计算）

**输出/影响**：
- 胜负结果 → 影响剧情分支、任务完成
- 胜利次数 → 触发称号系统（“乡勇”→“天下第一”）
- 击败特定人物 → 获得人物卡、秘传卡
- 切磋胜利 → 提升亲密度，可能习得对手武技

**与其他系统的耦合点**：

| 关联系统 | 耦合方式 | 数据流向 |
|---------|---------|---------|
| **技能系统** | 武艺等级决定手牌数；医术提供恢复卡；密探提供控制卡 | 技能 → 个人战 |
| **卡片系统** | 个人战武技卡作为卡片存储；胜利可获新武技卡、秘传卡 | 个人战 → 卡片 |
| **任务主命** | 缉捕主命触发个人战；武举任务以个人战为核心 | 任务 ↔ 个人战 |
| **社交系统** | 切磋是提升亲密度主要手段之一；亲密度高才能向武将学艺 | 个人战 ↔ 社交 |
| **修行系统** | 武馆修行即进行个人战小游戏；胜利获得技能经验 | 修行 ↔ 个人战 |
| **称号系统** | 个人战胜利累计授予称号；称号提供武力加成反哺个人战 | 个人战 ↔ 称号 |

#### 3. 任务主命系统（评定会）

**核心职责**：每两月评定会生成可选任务，完成后积累功勋，推动身份晋升。

**任务类型**：
- **军务**：出征、练兵、侦查 → 触发合战/个人战
- **政务**：开垦、治水、断案、征税 → 触发小游戏，提升内政技能
- **外交**：出使、招降 → 触发口才小游戏，影响势力关系
- **特殊**：缉捕、医病、督建 → 触发个人战/小游戏

**耦合方式**：任务是所有玩法系统的“触发器”。玩家通过选择不同主命，自然进入合战、个人战、小游戏等模块。

**数据流**：
```
评定会选择主命 → 根据主命类型跳转对应系统
                → 军务类 → 合战系统
                → 缉捕类 → 个人战系统
                → 政务类 → 小游戏系统
                → 完成后返回评定会结算功勋
```

#### 4. 卡片收集系统

**核心职责**：统一管理所有可收集内容的获取、存储、查看。

**卡片获取途径**：
- **人物卡**：亲密度达3心；完成特定剧情；击败该角色
- **技能卡**：技能升级时自动获得对应等级卡（作收集图鉴）
- **合战战术卡**：师事修行；战胜敌将；事件习得
- **个人战武技卡**：武馆修行；切磋习得；秘笈
- **秘传卡**：拥有对应人物卡后，在特定条件下切磋胜利
- **名所卡**：访问特定地点
- **称号卡**：达成条件（合战/个人战胜利次数等）

**耦合方式**：卡片系统是“奖励中枢”。所有系统的产出最终都体现为卡片收集，而卡片又反哺各系统（习得新战术卡增强战斗力）。

#### 5. 技能与修行系统

**核心职责**：管理16项技能的升级，以及通过小游戏修行获取新卡片。

**技能作用**：
- **直接影响**：步战等级→可用阵型；火器等级→可带火器兵；兵法等级→秘策次数
- **解锁卡片**：技能达到一定等级可师事习得对应战术卡
- **小游戏表现**：技能等级越高，对应小游戏越容易达成高评分

**修行流程**：
```
找到对应师父（需亲密度≥2心）→ 选择修行技能 → 进入小游戏
→ 小游戏成功 → 技能经验+1 → 经验满则技能升级 → 可能习得新战术卡
```

**耦合点**：技能是连接“角色属性”与“战斗系统”的桥梁。没有技能，空有高统率也无法使用高级阵型。

#### 6. 社交系统

**核心职责**：管理角色间亲密度，处理送礼、茶会、切磋、婚姻等交互。

**亲密度等级**：
- 1心：一面之缘
- 2心：略有交情（可邀请修行、可劝诱）
- 3心：推心置腹（可获得人物卡）
- 4心：生死相随（极难叛变）

**耦合点**：
- 亲密度是**获取人物卡**的核心条件
- 亲密度是**拜师修行**的前提
- 亲密度影响**劝降成功率**（合战招降卡效果）
- 亲密度影响**婚姻、结义**等剧情

#### 7. 经济系统

**核心职责**：管理玩家个人资产、城镇物价、商帮贸易、投资。

**主要功能**：
- 买卖商品赚取差价（算术技能影响）
- 向城镇投资提升特产等级
- 参与开中法盐引贸易（商人路线核心）
- 购买武器、宝物送礼

**耦合点**：
- 资金用于**购买兵粮军备**（影响出征）
- 资金用于**购买礼物**（提升亲密度）
- 商人路线的**御用商人**身份需通过评定会任务晋升

#### 8. 世界状态与时间系统

**核心职责**：推进游戏时间，更新势力版图、城镇状态、角色位置。

**时间推进**：
- 移动消耗天数（城镇间移动）
- 执行任务消耗天数
- 修行小游戏消耗天数
- 每两月触发评定会

**耦合点**：时间是所有活动的“资源”。玩家需在有限时间内平衡修行、任务、社交、移动。


### 🔄 四、核心循环——玩家游戏流程

```
┌─────────────────────────────────────────────────────────────┐

└──────────────────────────┬──────────────────────────────────┘
                           │ 选择主命

┌─────────────────────────────────────────────────────────────┐
│                    执行主命阶段                              │

│  │军务主命  │  │政务主命  │  │外交主命  │  │特殊主命  │    │
│  │→合战     │  │→小游戏   │  │→口才判定 │  │→个人战/  │    │

```

### 💡 六、对单人开发的实现建议

**（一）模块开发顺序**（由核心到外围）

1. **核心数据层**（1周）
   - 设计角色、世界、卡片的JSON结构
   - 实现基础的存档/读档功能

2. **卡片系统**（1周）
   - 卡片数据结构定义
   - 卡片获取/检查/展示逻辑

3. **时间与移动系统**（1周）
   - 时间推进、日期显示
   - 城镇间移动消耗天数

4. **评定会与任务框架**（2周）
   - 评定会界面与任务生成
   - 任务完成与功勋结算
   - 身份晋升逻辑

5. **小游戏系统**（2-3周）
   - 优先实现2-3个简单小游戏（算术、口才、筑城）
   - 搭建小游戏→技能经验的接口

6. **个人战系统**（2周）
   - 数字组合判定逻辑
   - 特殊技卡与秘传卡机制
   - 与武馆修行的对接

7. **合战系统**（3周）
   - 阵型与兵种系统
   - 出牌与伤害计算
   - 攻城战/水战特殊规则

8. **社交与经济系统**（2周）
   - 亲密度与送礼
   - 买卖与投资

9. **剧情与事件系统**（2-3周）
   - 事件触发条件检测
   - 历史战役（鄱阳湖等）

**（二）技术选型建议**

- **语言**：Python（适合快速原型，文字游戏足够）
- **数据存储**：JSON文件（存档、卡片库、事件文本）
- **界面**：控制台输出即可，后期可封装为网页版（HTML+JS）
- **状态管理**：用一个全局`GameState`对象管理所有数据

**（三）MVP（最小可行版本）范围**

建议先完成以下内容即可体验核心乐趣：
- 朱元璋第一章主线剧情（线性推进）
- 3种技能修行（武艺、步战、算术）及对应小游戏
- 个人战基础机制（数字组合+太祖长拳+铁布衫）
- 合战简化版（仅野战，3种阵型，5种战术卡）
- 评定会与3类主命

MVP跑通后，再逐步添加水战、火器、密探、商帮等复杂系统。


### 📋 七、系统耦合关系速查表

| 系统A | 影响系统B的方式 | 数据流向 |
|-------|---------------|---------|
| **技能** | 技能等级决定合战可用阵型/秘策、个人战手牌数 | 技能 → 合战/个人战 |
| **合战** | 胜利获功勋→身份晋升；胜利次数→称号；胜利获人物卡/战术卡 | 合战 → 任务/称号/卡片 |
| **个人战** | 胜利获武技卡/秘传卡；切磋提升亲密度；胜利次数→称号 | 个人战 → 卡片/社交/称号 |
| **任务** | 选择主命类型决定进入哪个玩法系统 | 任务 → 合战/个人战/小游戏 |
| **卡片** | 习得新战术卡增强合战/个人战能力 | 卡片 → 合战/个人战 |
| **称号** | 提供属性加成反哺战斗 | 称号 → 合战/个人战 |
| **社交** | 亲密度决定拜师、获取人物卡、劝降成功率 | 社交 → 修行/卡片/合战 |
| **经济** | 资金用于军备、送礼、投资 | 经济 → 合战/社交/世界 |
| **世界** | 城池归属影响可执行任务类型 | 世界 → 任务 |


### ✅ 八、总结

《洪武立志传》的系统框架遵循 **“卡片驱动、评定循环、数据解耦”** 三大原则：

1. **卡片是通用货币**：一切收集物均卡片化，各系统通过卡片ID交互，避免硬编码
2. **评定会是节奏控制器**：每两月一次，玩家选择主命 → 进入对应玩法 → 结算成长，形成清晰循环
3. **核心数据层是唯一真相源**：所有系统读写同一份角色/世界数据，保证状态一致

合战与个人战看似复杂，实则只是“任务触发的一种玩法模块”。只要核心数据层设计得当，它们可像插件一样逐步添加，不影响其他系统。

单人开发时，**先跑通评定会→任务→小游戏/简单战斗→结算的闭环**，再逐步丰富卡片和战斗深度，是可行且稳健的路径。
