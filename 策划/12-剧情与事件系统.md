# 12 - 剧情与事件系统

## 一、事件类型与层级

| 类型 | 说明 | 触发 | 可重复 |
|------|------|------|--------|
| **主线事件** | 驱动朱元璋从乞丐到皇帝的核心叙事 | 时间+条件自动触发 | 否（单存档） |
| **支线事件** | 特定角色/地点的剧情，不影响主线 | 条件触发 | 部分可 |
| **随机事件** | 移动停留概率触发，增加世界生动性 | 概率 | 是 |
| **定期事件** | 固定日期发生 | 时间自动 | 每年重复 |

**触发优先级**：主线 > 支线 > 定期 > 随机。同一时间多个事件满足，只触发优先级最高的一个。

## 二、触发条件系统

支持多维度组合条件：

| 条件类别 | 示例 | 数据来源 |
|---------|------|---------|
| **时间条件** | 当前日期 ≥ 1356年3月1日；奇数月1日 | global_time |
| **地点条件** | 玩家位于应天府 | player.location |
| **身份条件** | 玩家身份为千户以上 |  player.identity |
| **势力条件** | 玩家所属朱元璋军；陈友谅已灭亡 | player.faction |
| **亲密度条件** | 与徐达亲密度 ≥ 3心 | relationships |
| **卡片条件** | 持有"刘伯温"人物卡 | player.owned_cards |
| **技能条件** | 武艺 ≥ Lv2 | player.skills |
| **标记条件** | "郭子兴病逝"已触发 | event_flags |
| **随机概率** | 移动时10% | 随机数 |

事件系统每次推进时间（每天）都会遍历所有未触发事件，检查条件，按优先级触发。

## 五、事件数据结构

```json
{
  "event_id": "E_MAIN_ZHU_TOUJUN",
  "type": "main",
  "name": "投身红巾",
  "trigger": {
    "time": {"year": 1353, "month": 3, "operator": ">="},
    "location": "HAOZHOU",
    "identity": "游方僧",
    "flags_forbidden": ["E_DONE"],
    "probability": 1.0
  },
  "scenes": [
    {
      "scene_id": 1,
      "text": "濠州城外，红巾军大营。你望着营门旗帜，想起汤和信...",
      "choices": [
        {
          "text": "大步走入军营，投效郭子兴！",
          "effects": [
            {"type": "change_identity", "value": "红巾军士兵"},
            {"type": "add_card", "card_id": "CHAR_GUO_ZIXING"},
            {"type": "set_flag", "flag": "E_DONE"}
          ],
          "next_scene": 2
        },
        {
          "text": "犹豫片刻，先在营外观察几日",
          "effects": [
            {"type": "advance_time", "days": 5},
            {"type": "set_flag", "flag": "ZHU_HESITATE"}
          ]
        }
      ]
    }
  ]
}
```

**效果指令类型**：
- `change_identity`：改变玩家身份
- `change_faction`：改变玩家势力
- `add_card`：授予卡片
- `set_flag`：设置事件标记
- `advance_time`：推进时间
- `trigger_battle`：触发合战/个人战
- `change_relation`：改变亲密度
- `add_gold`：增加金钱

## 七、与太阁立志传4对比

| 太阁4 | 洪武立志传 |
|------|------------|
| 主人公剧情 | 朱元璋开国剧情 |

核心事件架构完全继承太阁4，剧情完全替换为明朝开国历史，丰富了支线事件内容。

## 八、实现建议

1. **事件调度器**：每次时间推进后，检查所有未触发事件，按优先级选一个触发
2. **条件检测**：每个条件对应检测函数，支持组合条件
3. **效果执行**：每个效果指令对应执行函数，修改核心数据
4. **文本分离**：事件文本全存在JSON，不和代码混合，便于润色
5. **分支保存**：所有选择通过`event_flags`保存，读档后正确还原分支
